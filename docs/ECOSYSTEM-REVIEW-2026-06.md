## 2. Market changes, Jan–Jun 2026 (external)

### 2.1 Convergence on open standards

- **Agent Skills (`SKILL.md`)** became a cross-tool standard (agentskills.io). The neutral directory **`.agents/skills/` is read natively by Claude Code, Codex CLI, Gemini CLI, OpenCode (v1.16+), and Copilot CLI** — all five PAN runtimes. One skills tree can replace most of PAN's five-way command conversion.
- **AGENTS.md won** as the universal rules file — stewarded by the Agentic AI Foundation under the Linux Foundation (founders incl. Anthropic, OpenAI, Block); read natively by Codex, Copilot, OpenCode (primary), and the Gemini ecosystem. Claude Code still reads only CLAUDE.md; the sanctioned bridge is a CLAUDE.md containing `@AGENTS.md`. Nested AGENTS.md per subdirectory is normative for monorepos.
- **Plugin packaging everywhere**: Claude Code plugins (`.claude-plugin/`) bundle skills/agents/hooks/MCP/LSP/binaries — and **Copilot CLI consumes the Claude plugin format**. Codex has plugins + marketplaces (`.codex-plugin/plugin.json`, `marketplace.json` under `.agents/plugins/`); Gemini has extensions (`gemini-extension.json`).
- **Hooks now exist on every runtime**, not just Claude: Codex (`.codex/hooks.json`, Claude-like events/JSON, seconds timeouts), Gemini (`BeforeTool`-style names, settings.json, ms timeouts), Copilot (`.github/hooks/*.json` with **native `bash` + `powershell` dual keys**). The Copilot schema directly solves the recorded "hook commands must be cross-platform" feedback.
- **MCP**: the spec release of 2025-11-25 added async tasks and extensions; a stateless-core release candidate is locked for 2026-07-28 (no `initialize` handshake, plain load balancers work). Official registry still in preview — do not build distribution on it yet.

### 2.3 Competitive landscape

Spec-driven/workflow tooling is now an arms race: GitHub Spec Kit (~93k stars), **GSD (~61k — closest PAN analog, but destabilized by a governance collapse; development forked to a community redux)**, BMAD-METHOD (~47k), OpenSpec, Amazon Kiro, Taskmaster (delivered as an **MCP server**, not file installs). "Ralph loops" / context-rot discipline went mainstream — validating PAN's state-discipline thesis. Parallel/cloud agents (Copilot `/fleet` + agents panel, Claude agent teams + Routines, Codex cloud) mean `.planning/` will increasingly face concurrent readers/writers.

**Positioning:** sit above runtime-native orchestration as the planning/state/verification layer; move on multi-runtime messaging while GSD is destabilized.

## 3. Model alignment (most out-of-date area in the codebase)

Current lineup (verified 2026-06): **Claude Fable 5** ($10/$50 per MTok, 1M ctx, adaptive thinking only — explicit `thinking: disabled` is rejected), **Opus 4.8** ($5/$25, default in Claude Code), Opus 4.7/4.6, Sonnet 4.6 ($3/$15), Haiku 4.5 ($1/$5). `budget_tokens` returns 400 on Opus 4.7+; the cost/intelligence dial is **`effort`** (`low`/`medium`/`high`/`xhigh`/`max`). Codex: GPT-5.5/5.4/5.4-mini. Gemini: 3.1 Pro preview + auto-routing. Retirements 2026-06-15: `claude-sonnet-4-20250514`, `claude-opus-4-20250514`.

Concrete misalignments found:

1. **Wrong pricing** — `pan-wizard-core/bin/lib/cost.cjs` (DEFAULT_RATES) lists `claude-opus-4-7` at $15/$75; actual is $5/$25 (≈3× overstatement of Opus cost records). Missing `claude-opus-4-8`, `claude-fable-5`, GPT-5.x; `gemini-1.5-pro` is a fossil.
2. **Stale capability detection** — `bin/install-lib.cjs` `detectModelCapabilities()` keys on Opus 4.7 and Claude 3.5 Sonnet (retired Oct 2025). `bin/install.js` recommends "upgrade to claude-opus-4-7" (superseded twice).
3. **Deprecated thinking controls** — ADR-0023 paths emit `thinking: enabled` + `thinking_budget`; replacement is adaptive thinking + `effort` (now expressible directly in Claude Code skill/agent frontmatter).
4. **`model_profile` maps to the wrong dial** — quality/balanced/budget → tier swapping was right in 2025; in 2026 the primary within-model dial is `effort` (official guidance: `low` for subagents/scanners, `high` default, `xhigh` for coding/agentic).
5. **Prompts tuned for the wrong model generation** — Opus 4.8/Fable 5 shifts affecting shipped content: under-reach for subagents/file-memory/custom tools unless told *when* to use them; "CRITICAL/MUST" language now overtriggers; "only report high-severity" review prompts depress recall (pan-reviewer); narration verbose by default (remove forced progress-update scaffolding); Fable 5 is explicitly trained to exploit persistent file memory — `.planning/` and the learnings system should lean into this.

### P1 — Format migrations (v3.9)

5. ~~**Effort-aware model profiles**~~ — **DONE 2026-06-10.** `core.cjs` gained `AGENT_BASE_EFFORT` (all agents, mirroring shipped frontmatter) and `resolveEffortInternal()`: `budget` profile steps effort down one level (floor `low`), `quality`/`balanced` keep base, `config.json → effort_overrides` wins. `resolve-model` CLI now returns an `effort` field. 6. **Prompt re-tuning pass for Opus 4.8/Fable 5** — **surveyed + applied 2026-06-10; most items were no-ops because PAN didn't have the anti-patterns.** Findings from a full sweep of all shipped agents and workflows:
   - **pan-reviewer → report-everything-filter-downstream: DONE.** Reframed the finder's `FOCUS: report real issues, not style preferences` constraint to a `COVERAGE` constraint — report every finding (incl. uncertain/low-severity) tagged to the right tier; the severity tiers, verdict, and the existing **pan-meta-reviewer** stage (`/pan:review-deep`, which already disputes overstated findings and removes duplicates) are the downstream filter. This is the migration guide's recommended pattern, and PAN already had the two-stage finder→filter architecture, so the change is precise, low-risk, and model-agnostic (helps GPT-5.x/Gemini reviewers too).
   - **Soften CRITICAL/MUST: NO-OP (correctly).** Surveyed every CRITICAL/MUST in the agents (18 in pan-planner alone). They are legitimate correctness invariants — mandatory file reads, user-decision fidelity, requirement-ID integrity, TDD red/green discipline — **not** the tool-over-triggering anti-pattern the guide warns about. A mechanical softening sweep would have weakened genuine guardrails; deliberately not done.
   - **Remove forced progress-update scaffolding: NO-OP.** No "after every N tool calls, summarize" scaffolding exists anywhere.
   - **Silence-defaults / over-narration: NO-OP for the agent layer.** Leaf agents return tightly-structured `<output_format>` reports, not free narration.
   - **Explicit delegate/memory triggers: DEFERRED (lower-confidence).** PAN's leaf agents don't spawn subagents or manage file memory — delegation happens in the orchestration layer (workflows/commands read by the main session), which already encodes explicit delegation logic. Adding "delegate/check-memory more" nudges there is subjective and better done with human review than a mechanical pass. Left for a maintainer.

12. ~~**Native Claude Code workflows**~~ — **first slice SHIPPED 2026-06-11**: hand-authored native scripts for the genuinely deterministic fan-out protocols (pan-review-pipeline, pan-map-codebase) install to `.claude/workflows/`, generated by the pure `buildNativeWorkflowScripts()` builder (syntax-validated in tests, resume-safe — no Date.now/Math.random).
