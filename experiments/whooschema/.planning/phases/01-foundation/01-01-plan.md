Create `src/error-utils.js` exporting four functions:

1. **`makeError(path, rule, message, value, expected)`** — pure constructor returning an error object literal `{ path, rule, message, value, expected }`. If `expected` is `undefined`, OMIT the key from the object (so `JSON.stringify` output is byte-identical between calls). Use `const err = { path, rule, message, value }; if (expected !== undefined) err.expected = expected; return err;` — do NOT use object spread, do NOT include any extra keys.

2. **`appendKey(path, key)`** — returns `path + '.' + key`. No quoting, no special-character escaping (per research.md: dogfood targets do not have exotic keys; document the limitation in the file's leading comment).

3. **`appendIndex(path, i)`** — returns `path + '[' + i + ']'`. `i` is a non-negative integer.

4. **`sortErrors(errors)`** — returns a NEW array (does not mutate input), sorted by path with the numeric-aware comparator from research.md "Pattern: Numeric-Aware Path Sort":

```js
function comparePaths(a, b) {
  const segs = (s) => s.split(/(\.[^.\[]+|\[\d+\])/).filter(Boolean);
  const sa = segs(a), sb = segs(b);
  const n = Math.max(sa.length, sb.length);
  for (let i = 0; i < n; i++) {
    if (sa[i] === undefined) return -1;
    if (sb[i] === undefined) return 1;
    const ma = sa[i].match(/^\[(\d+)\]$/);
    const mb = sb[i].match(/^\[(\d+)\]$/);
    if (ma && mb) {
      const d = Number(ma[1]) - Number(mb[1]);
      if (d !== 0) return d;
    } else if (sa[i] !== sb[i]) {
      return sa[i] < sb[i] ? -1 : 1;
    }
  }
  return 0;
}
```

Apply to errors: `errors.slice().sort((a, b) => comparePaths(a.path, b.path))`. **Stable sort** is guaranteed by V8 since Node 12.

**Why this matters for API-03:** lex sort puts `'[10]' < '[9]'` (wrong). The numeric branch handles `[N]` segments correctly. For `.key` segments, lex sort is correct.

All exports are pure (no side effects). The file uses `export function` (named ESM exports). No default export. No `console.*` calls.

Code style: small file, ~50-70 lines including the comparator. Plain `function` declarations are fine (no need for arrow functions). Add a short module-level comment block referencing the spec contract.

**Create `src/schema-loader.js`** with:

```js
const KNOWN_KEYWORDS = new Set([
  // primitives & types (Phase 1)
  'type','enum','const',
  'minLength','maxLength','pattern',
  'minimum','maximum','exclusiveMinimum','exclusiveMaximum','multipleOf',
  'items','minItems','maxItems','uniqueItems',
  'properties','required','additionalProperties','patternProperties',
  // metadata (always permitted)
  '$schema','title','description','default','examples','definitions','$defs','$id','$comment',
  // composition (Phase 2 — listed so they don't warn)
  'oneOf','anyOf','allOf','not','$ref',
  // formats (Phase 3 — listed so they don't warn)
  'format'

```

2. A module-level `warned = new Set()` for dedup of unknown-keyword warnings (so the same unknown key doesn't print 100 times across recursive calls or across multiple loadSchema calls in a process).

3. **`export function loadSchema(schema)`** that walks the schema depth-first. For each object value visited:
   - For every key/value pair:
     - If `key === 'pattern'` AND `typeof value === 'string'`: try `new RegExp(value)`; on `SyntaxError`, throw `new Error('Invalid pattern regex "' + value + '" at schema-load time: ' + e.message)` — **DO NOT** swallow.
     - If `!KNOWN_KEYWORDS.has(key)` AND `!warned.has(key)`: `console.warn('whooschema: unknown keyword "' + key + '" — ignoring')`; `warned.add(key)`.
     - If the value is a non-null object (including arrays), recurse into it.
   - Return the schema unchanged.

Walk implementation note: only recurse into values that are objects. Skip primitive values. For arrays (e.g., `enum: [...]`, `required: [...]`, `oneOf: [...]`) recurse into each element since some are sub-schemas and some are primitives — the `typeof v === 'object'` check is sufficient (it matches both objects and arrays; primitives are skipped).

Do NOT track ref maps or detect cycles in Phase 1 — Phase 2 adds those. The walk is purely for `pattern` validation and unknown-keyword warning.
