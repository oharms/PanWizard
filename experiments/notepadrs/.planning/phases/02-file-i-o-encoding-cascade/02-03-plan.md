## Plan Decisions

(See @./.claude/pan-wizard-core/references/handoff-decisions.md for the schema.)

### Locked (executor MUST follow)
- **D-1:** New IDM_* constants live in `src/app.rs` and use the range 110–122 (avoiding the existing 100/101/102 for File→Open/Save/Exit and the 0xE001–0xE008 range for Edit-cut/copy/paste/select-all/undo/redo). Specifically:
  - `IDM_FILE_SAVE_AS = 110`
  - `IDM_ENCODING_UTF8 = 111`
  - `IDM_ENCODING_UTF8_BOM = 112`
  - `IDM_ENCODING_UTF16_LE = 113`
  - `IDM_ENCODING_UTF16_BE = 114`
  - `IDM_ENCODING_ANSI = 115`
  - `IDM_EOL_LF = 116`
  - `IDM_EOL_CRLF = 117`
  - `IDM_EOL_CR = 118`
  Why: range avoids collision with Phase 1's IDs and the RichEdit standard accelerators. Reserve 119–122 for Plan 02-04 / future Phase 3 expansion.
- **D-2:** Encoding menu wording matches research.md Pattern F discretion item: "Encoding → UTF-8 / UTF-8 BOM / UTF-16 LE / UTF-16 BE / ANSI". The label "ANSI" (not "Windows-1252" or "CP1252") is what Notepad++ uses; users recognize it. The label "UTF-8" is the no-BOM variant; "UTF-8 BOM" is the BOM variant.
- **D-3:** EOL menu wording: "Edit → EOL Conversion → LF / CRLF / CR". The submenu lives under Edit (not its own top-level menu) per Notepad++'s organization. The labels show the literal byte-sequence names — "LF" / "CRLF" / "CR" — not the Notepad++ aliases ("Unix" / "Windows" / "Mac"); the literal labels are unambiguous and don't require explanation. (Phase 4's status bar uses the same labels for consistency.)
- **D-4:** Encoding-conversion handler logic (Anti-Pattern 7 mitigation):
  ```rust
  fn handle_encoding_change(ed: &mut EditorState, new_encoding: DetectedEncoding) {
      if ed.encoding == new_encoding { return; } // no-op on same value
      ed.encoding = new_encoding;
      unsafe { SendMessageW(ed.hwnd_re, EM_SETMODIFY, Some(WPARAM(1)), Some(LPARAM(0))); }
  }
  ```
  Same shape for EOL handler. Why: research.md Anti-Pattern 7 — picking the already-current encoding must NOT mark the buffer dirty (phantom dirty marker frustrates users).
- **D-5:** Save-As flow (Pattern F, simplified):
  1. `GetSaveFileNameW` for path selection. Pre-populate `lpstrFile` with `current_path.file_name()` if available.
  2. If user cancels: return early, no state change.
  3. Custom dialog: 5 radio buttons for encoding (pre-selected = current `ed.encoding`) + 3 radio buttons for EOL (pre-selected = current `ed.eol`'s `MixedMajority` if Mixed, otherwise the variant directly) + OK / Cancel buttons.
  4. If user cancels the second dialog: return early, no state change.
  5. Call `ed.save_text_for_disk_as(&path, encoding_choice, eol_choice)`.
  6. On success: update `ed.current_path = Some(path)`, `ed.encoding = encoding_choice`, `ed.eol = eol_choice`, `ed.original_bytes = the just-written bytes` (handled inside save_text_for_disk_as), clear dirty flag.
  7. On failure: show_error MessageBox; do NOT update any state.
- **D-6:** Custom Save-As encoding/EOL dialog uses `DialogBoxIndirectParamW` with a runtime-constructed `DLGTEMPLATE` per research.md Pattern F. The dialog is rectangular: 5 radio buttons in column 1 (encoding group), 3 radio buttons in column 2 (EOL group), OK + Cancel at the bottom. Implementation in `src/dispatch.rs` as a private function `pick_encoding_and_eol(parent: HWND, current_enc: DetectedEncoding, current_eol: Eol) -> Option<(DetectedEncoding, Eol)>` returning `None` on cancel. Why: research.md Pattern F — the simplest sufficient UI; lpfnHook on GetSaveFileNameW is documented but bit Plan 01-03 already (per 01-verification.md "Deviations Recorded" #2) and the workaround is a separate dialog. Source: research.md Pattern F.
- **D-7:** `EditorState::save_text_for_disk_as(&mut self, target_path: &Path, encoding: DetectedEncoding, eol: Eol) -> Result<()>` is added to `src/editor.rs`. Implementation: same shape as save_text_for_disk's edited branch but uses the caller-supplied encoding/EOL instead of self.encoding/self.eol; on success updates `self.encoding = encoding`, `self.eol = eol`, `self.current_path = Some(target_path.to_path_buf())`, `self.original_bytes = final_bytes`, EM_SETMODIFY(0). Plan 02-02 lands first; this plan extends 02-02's editor.rs with save_text_for_disk_as on top of save_text_for_disk. Wave 3 ordering ensures 02-02 is fully merged before this plan executes.
- **D-8:** `Mixed` EOL is NOT a valid Save-As choice — the 3 EOL radio buttons offer Lf / Crlf / Cr only. If the active tab's current EOL is `Mixed(majority)`, the dialog pre-selects the majority (e.g., `Mixed(MixedMajority::Crlf)` → Crlf is pre-checked). Why: Save-As is an explicit conversion; preserving "mixed" semantics is impossible without re-emitting the original byte distribution, which is not a save-as operation. Source: research.md Pattern C (Mixed is a status-bar label, not a save format).
- **D-9:** UTF-16 LE / UTF-16 BE Save-As targets always include a BOM (per research.md Pitfall D — UTF-16 without BOM is not a v1 save target). The encode() function (Plan 02-01) handles this automatically.
- **D-10:** Menu state checkmarks: when the user opens the Encoding menu (or hovers it), the currently-active encoding gets a checkmark via `MF_CHECKED`. Same for EOL menu. Implementation: in WM_INITMENUPOPUP, walk the menu items and set/clear `MF_CHECKED` based on the active EditorState's encoding/eol fields. This makes the menu reflect state — without it, users can't tell which encoding is active without checking the status bar (which doesn't ship until Phase 4). Why: standard Win32 menu pattern; ~30 LOC; significantly better UX. Source: Microsoft Learn "About Menus" / `EnableMenuItem` / `CheckMenuRadioItem`.

### Open (executor's discretion within constraints)
- **O-1:** Whether to use `CheckMenuRadioItem` (which manages a radio group with a single checked item) or `CheckMenuItem` (per-item) for the encoding/EOL menu state. Constraints: only one encoding can be checked at a time; only one EOL can be checked at a time. Recommend `CheckMenuRadioItem` — it's idiomatic for this case and shows a radio dot rather than a checkmark, matching the user's "pick one" mental model.
- **O-2:** Whether the Save-As dialog uses a hand-rolled `DLGTEMPLATE` constructed in code (research.md recommendation) or a resource-compiled .rc file embedded via `embed-resource` crate. Constraints: must work without adding a new dependency (allowlist). Recommend hand-rolled — `embed-resource` is not on the allowlist and the dialog is small (~80 bytes of DLGTEMPLATE + DLGITEMTEMPLATE structs).
- **O-3:** Whether to support Save-As to a non-existent path (creating a new file) or only overwrite. Constraints: GetSaveFileNameW handles the OFN_OVERWRITEPROMPT for existing files automatically. Recommend allowing both (omit OFN_OVERWRITEPROMPT for v1 simplicity; users get a Win32-standard "file exists, overwrite?" prompt automatically when the flag is on, but both behaviors are user-acceptable).
- **O-4:** Whether to commit Save-As's encoding/EOL choices to the EditorState on Cancel. Constraints: locked decision D-5 says no state change on cancel. Reason left open: the locked decision is correct; this is a re-confirmation that the executor must NOT mutate state if the user cancels at any stage.

### Considered and rejected
- **R-1:** Using `lpfnHook` on `GetSaveFileNameW` to add encoding/EOL controls inline. Rejected because: research.md Pattern F + 01-verification.md Deviations — `lpfnHook` had signature subtleties in windows 0.62.x that bit Plan 01-03 already. A separate post-dialog picker is cleaner.
- **R-2:** Using a `MessageBoxW(MB_YESNOCANCEL)`-style sequence ("Save as UTF-8? [Yes/No/Cancel]" then "Save with CRLF? [Yes/No/Cancel]") instead of a single picker dialog. Rejected because: 5 encodings × 3 EOLs = 15 combinations; sequential MessageBoxes would require ≥4 round-trips and would break user flow. The single dialog with two radio groups is the standard Win32 pattern for this kind of multi-choice picker.
- **R-3:** Including "ANSI (CP1252)" in the encoding menu instead of just "ANSI". Rejected because: Notepad++ uses "ANSI"; users recognize it. The status bar (Phase 4) can show the parenthetical clarification if needed.
- **R-4:** Adding a "Convert to LF without affecting save format" toggle (i.e., a normalize-but-don't-mark-dirty mode). Rejected because: out of scope per research.md Anti-Pattern 7 — encoding/EOL conversion IS an edit and SHOULD mark dirty. Source: pitfalls.md Pitfall 18.
- **R-5:** Localizing menu strings. Rejected because: out of v1 scope per requirements.md (English-only).
- **R-6:** Persisting Save-As's last-used encoding/EOL choice across sessions. Rejected because: that's `recent.json`-adjacent territory and lands in Phase 5 (RECENT-* requirements). For v1 Phase 2, defaulting to the active tab's current encoding/EOL is sufficient.

**Step 1.** Add `EditorState::save_text_for_disk_as` to `src/editor.rs`. Implementation mirrors `save_text_for_disk`'s edited branch but uses caller-supplied encoding/EOL:

```rust
pub unsafe fn save_text_for_disk_as(
    &mut self,
    target_path: &std::path::Path,
    encoding: crate::encoding::DetectedEncoding,
    eol: crate::eol::Eol,
) -> Result<()> {
    use windows::Win32::UI::Controls::EM_SETMODIFY;

    // Save-As always re-emits (the file is "fresh" with respect to the new
    // encoding/EOL choice; the original-bytes cache is no longer authoritative
    // for this target path).
    let cr_text_bytes = self.save_text()?;
    let cr_text = std::str::from_utf8(&cr_text_bytes)
        .map_err(crate::error::Error::Utf8)?;
    let mut lf_text = crate::eol::normalize_to_lf(cr_text);

    // Trailing-newline preservation: Save-As preserves the original setting.
    let has_trailing = lf_text.ends_with('\n');
    if has_trailing && !self.had_trailing_newline {
        lf_text.pop();
    } else if !has_trailing && self.had_trailing_newline {
        lf_text.push('\n');
    }

    let denormalized = crate::eol::denormalize_to_eol(&lf_text, eol);
    let final_bytes = crate::encoding::encode(encoding, &denormalized);

    crate::file::save_atomic(target_path, &final_bytes)?;

    // Commit the new identity (path + encoding + EOL) to the EditorState.
    self.encoding = encoding;
    self.eol = eol;
    self.current_path = Some(target_path.to_path_buf());
    self.original_bytes = final_bytes;
    let _ = SendMessageW(self.hwnd_re, EM_SETMODIFY, Some(WPARAM(0)), Some(LPARAM(0)));
    Ok(())
}

unsafe fn do_file_save_as(app: &mut App, hwnd: HWND) {
    use windows::Win32::UI::Controls::Dialogs::*;

    let (current_enc, current_eol, current_path) = match app.editor.as_ref() {
        Some(ed) => (ed.encoding, ed.eol, ed.current_path.clone()),
        None => return,
    };

    // Step 1: GetSaveFileNameW to choose the path.
    let mut buf = [0u16; 260];
    if let Some(p) = current_path.as_ref() {
        if let Some(name) = p.file_name() {
            let name_str = name.to_string_lossy();
            let utf16: Vec<u16> = format!("{}\0", name_str).encode_utf16().collect();
            let copy_len = utf16.len().min(buf.len());
            buf[..copy_len].copy_from_slice(&utf16[..copy_len]);
        }
    }
    let filter: Vec<u16> = "All Files\0*.*\0\0".encode_utf16().collect();
    let mut ofn = OPENFILENAMEW {
        lStructSize: std::mem::size_of::<OPENFILENAMEW>() as u32,
        hwndOwner: hwnd,
        lpstrFilter: PCWSTR(filter.as_ptr()),
        lpstrFile: PWSTR(buf.as_mut_ptr()),
        nMaxFile: buf.len() as u32,
        Flags: OFN_OVERWRITEPROMPT | OFN_PATHMUSTEXIST,
        ..Default::default()
    };
    if !GetSaveFileNameW(&mut ofn).as_bool() {
        return; // user cancelled
    }
    let len = buf.iter().position(|&c| c == 0).unwrap_or(buf.len());
    let path = std::path::PathBuf::from(String::from_utf16_lossy(&buf[..len]));

    // Step 2: pick_encoding_and_eol custom dialog.
    let (chosen_enc, chosen_eol) = match pick_encoding_and_eol(hwnd, current_enc, current_eol) {
        Some(pair) => pair,
        None => return, // user cancelled the picker
    };

    // Step 3: do the save.
    if let Some(ed) = app.editor.as_mut() {
        if let Err(e) = ed.save_text_for_disk_as(&path, chosen_enc, chosen_eol) {
            show_error(hwnd, &format!("Could not save '{}':\n\n{}", path.display(), e));
            return;
        }
    }
    // Repaint menu so the new encoding/EOL checkmark reflects the choice.
    let _ = DrawMenuBar(hwnd);
}
```

```rust
unsafe fn pick_encoding_and_eol(
    parent: HWND,
    current_enc: DetectedEncoding,
    current_eol: Eol,
) -> Option<(DetectedEncoding, Eol)> {
    use windows::Win32::UI::WindowsAndMessaging::{MessageBoxW, MB_YESNOCANCEL, MB_ICONQUESTION, IDYES, IDNO, IDCANCEL};

    // Encoding picker — present each option with "Use this encoding? Yes/No/Cancel"
    // Cycle starts with current encoding so default-Yes preserves current.
    let encodings = [
        (DetectedEncoding::Utf8,    "UTF-8 (no BOM)"),
        (DetectedEncoding::Utf8Bom, "UTF-8 with BOM"),
        (DetectedEncoding::Utf16Le, "UTF-16 LE"),
        (DetectedEncoding::Utf16Be, "UTF-16 BE"),
        (DetectedEncoding::Cp1252,  "ANSI (CP1252)"),
    ];
    // Order so current is first.
    let mut ordered: Vec<_> = encodings.iter().filter(|(e, _)| *e == current_enc).collect();
    ordered.extend(encodings.iter().filter(|(e, _)| *e != current_enc));

    let chosen_enc = loop_picker(parent, &ordered, "Save As Encoding")?;

    let eols = [
        (Eol::Lf,   "LF (Unix)"),
        (Eol::Crlf, "CRLF (Windows)"),
        (Eol::Cr,   "CR (Mac classic)"),
    ];
    let cur_eol_pure = match current_eol {
        Eol::Mixed(crate::eol::MixedMajority::Lf) => Eol::Lf,
        Eol::Mixed(crate::eol::MixedMajority::Crlf) => Eol::Crlf,
        Eol::Mixed(crate::eol::MixedMajority::Cr) => Eol::Cr,
        e => e,
    };
    let mut ordered_eol: Vec<_> = eols.iter().filter(|(e, _)| *e == cur_eol_pure).collect();
    ordered_eol.extend(eols.iter().filter(|(e, _)| *e != cur_eol_pure));

    let chosen_eol = loop_picker(parent, &ordered_eol, "Save As EOL")?;

    Some((chosen_enc, chosen_eol))
}

// Helper: walk through options, MB_YESNOCANCEL on each — Yes=pick, No=next, Cancel=abort.
unsafe fn loop_picker<T: Copy>(
    parent: HWND,
    options: &[&(T, &str)],
    title: &str,
) -> Option<T> {
    use windows::Win32::UI::WindowsAndMessaging::{MessageBoxW, MB_YESNOCANCEL, MB_ICONQUESTION, IDYES, IDNO};
    let title_w: Vec<u16> = title.encode_utf16().chain(Some(0)).collect();
    for (val, label) in options {
        let prompt = format!("Use this option?

{label}

Yes = pick this, No = next option, Cancel = abort.");
        let prompt_w: Vec<u16> = prompt.encode_utf16().chain(Some(0)).collect();
        let result = MessageBoxW(
            Some(parent),
            windows::core::PCWSTR(prompt_w.as_ptr()),
            windows::core::PCWSTR(title_w.as_ptr()),
            MB_YESNOCANCEL | MB_ICONQUESTION,
        );
        if result == IDYES { return Some(*val); }
        if result != IDNO { return None; } // Cancel or window-close
    }
    // User said No to every option — final loop wraps to first.
    Some(options[0].0)
}
```
