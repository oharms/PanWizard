## Plan Decisions

(See @./.claude/pan-wizard-core/references/handoff-decisions.md for the schema.)

### Locked (executor MUST follow)
- **D-1:** The encoding detection cascade has exactly the 5 deterministic steps documented in research.md Pattern B: (1) `encoding_rs::Encoding::for_bom(&bytes[..min(3, len)])` for BOM, (2) `std::str::from_utf8(&bytes).is_ok() && bytes.iter().any(|b| *b >= 0x80)` for UTF-8 (no BOM, has non-ASCII), (3) NUL-parity heuristic on first 4096 bytes with thresholds 40%/5% (high-parity ≥40% AND low-parity ≤5% → pick UTF-16; else fall through), (4) pure ASCII → label as UTF-8 (no BOM), (5) `encoding_rs::WINDOWS_1252.decode_without_bom_handling(...)` fallback. Why: deterministic, fixture-testable, covers SC-6's literal encoding list. Source: research.md Pattern B + .planning/research/architecture.md Pattern 8.
- **D-2:** The `DetectedEncoding` enum has exactly 5 variants: `Utf8`, `Utf8Bom`, `Utf16Le`, `Utf16Be`, `Cp1252`. `Utf8` and `Utf8Bom` are distinct because the BOM must round-trip on save. UTF-16 LE/BE always include BOM in v1's encoding model — BOM-less UTF-16 detected via heuristic gets a BOM added on save (an acceptable side-effect documented in research.md Pitfall D). Why: locked in research.md FILE-05 row + Pitfall D. Source: research.md.
- **D-3:** The `Eol` enum has exactly 4 outer variants: `Lf`, `Crlf`, `Cr`, `Mixed(MixedMajority)`. `MixedMajority` is a separate enum with `Lf | Crlf | Cr` to record the dominant style for status-bar display. Why: locked in research.md Pattern C. The 95% threshold separates pure from mixed classification.
- **D-4:** Step 4 (pure-ASCII → UTF-8) is non-negotiable — never label pure-ASCII files as CP1252. Why: research.md Pitfall B; modern editors (Notepad++ 8+, VS Code) all treat pure-ASCII as UTF-8 by default. Bytes are byte-identical between UTF-8-ASCII and CP1252-ASCII; UTF-8 is the modern default.
- **D-5:** EOL detection runs on the **decoded UTF-8 string**, not on raw bytes. Why: research.md Anti-Pattern 4 — UTF-16 LE encodes `\r` as `0d 00`, so byte-level scan would falsely match the high byte of the next codepoint. Always decode first, then `detect_eol(&str)`.
- **D-6:** The encoding `encode()` function emits the BOM bytes manually (prepend `EF BB BF` for `Utf8Bom`, `FF FE` for `Utf16Le`, `FE FF` for `Utf16Be`, nothing for `Utf8` / `Cp1252`). `encoding_rs::Encoding::encode` does NOT emit a BOM — that is the caller's job. Why: research.md Pitfall D — round-trip fails byte-exact without explicit BOM emission.
- **D-7:** Wave 0 spike test for `encoding_rs::UTF_16LE.encode("hello")`: assert the output is exactly 10 bytes (`[h, 0, e, 0, l, 0, l, 0, o, 0]`). If it is — use `encoding_rs::UTF_16LE.encode()` directly in `src/encoding.rs::encode`. If it isn't (the LOW-confidence case from research.md Open Question 1) — implement hand-rolled `encode_utf16_le(text: &str) -> Vec<u8>` and `encode_utf16_be(text: &str) -> Vec<u8>` (~30 LOC each, mechanical UTF-8 → UTF-16 with surrogate-pair handling for non-BMP) and use those. Document the resolution in 02-01-summary.md. Why: locked Wave 0 gap from research.md.
- **D-8:** Fixtures are generated by `tools/generate-fixtures.rs` (NOT hand-edited via a text editor — `core.autocrlf` would mangle the LF / CR / CRLF distinctions). The tool writes byte-exact content via `std::fs::write` and is committed alongside the fixtures so anyone can regenerate. Why: research.md Pitfall E + Test Fixture Strategy. Source: research.md.
- **D-9:** All fixtures live under `tests/fixtures/` which is already covered by `.gitattributes: tests/fixtures/* binary` (verified in Phase 1; no .gitattributes change required). The encoding_cascade.rs and eol_detection.rs tests include defensive in-test guards that assert each fixture's first bytes (BOM / content) match expectation — catching any future autocrlf mangling at test time. Why: research.md Pitfall E.
- **D-10:** `encoding_rs = "0.8"` is added to `[dependencies]` in Cargo.toml. The crate is already on the project allowlist (per .planning/idea.md and .planning/research/stack.md); this plan is the first to actually use it. The version `0.8` matches research.md (`0.8.35` is current; using `^0.8` is the standard caret-versioning approach). Why: SC-6 fully needs encoding_rs for transcoding; deferring it would just push the same diff to plan 02-02. Source: research.md "Phase 2 windows crate features" + idea.md crate allowlist.
- **D-11:** `Error::EncodingDetectFailed(String)` is added to `src/error.rs::Error`. This variant is rarely constructed (the cascade's CP1252 fallback never fails for byte content) but is reserved for future use cases (e.g., a UTF-8 BOM file that decodes to invalid UTF-8 — extremely rare but possible). The variant exists now so plans 02-02 and 02-04 don't need to extend Error themselves. Why: convergent design — every error path Phase 2 needs is enumerated up front.

### Open (executor's discretion within constraints)
- **O-1:** Whether to derive `Default` for `DetectedEncoding` and `Eol`. Constraints: must not affect the cascade behavior. Reason left open: a `Default` makes the test code marginally cleaner but isn't required. Recommend skipping unless it simplifies a specific test.
- **O-2:** Whether `tools/generate-fixtures.rs` is a `[[bin]]` target in Cargo.toml or a freestanding `.rs` script run via `rustc tools/generate-fixtures.rs && ./generate-fixtures`. Constraints: it must be re-runnable by anyone with rustup; the fixtures must be reproducible. Reason left open: a `[[bin]]` target adds a Cargo.toml entry but makes `cargo run --bin generate-fixtures` work; a freestanding `.rs` is simpler but has friction. Recommend the `[[bin]]` target since the project already has `[dependencies.windows]` etc. — adding `[[bin]] name = "generate-fixtures" path = "tools/generate-fixtures.rs"` is a 3-line cost.
- **O-3:** Exact non-ASCII content of the fixtures. Constraints: each non-empty UTF-8/UTF-8 BOM/UTF-16 fixture must contain at least one non-ASCII codepoint (so step 2 of the cascade can distinguish from pure-ASCII); CP1252 must use a byte that's invalid UTF-8 (e.g., `0xF6` for 'ö'). Reason left open: cosmetic. Recommend "Hello\nWörld\n" pattern (with appropriate EOL) — short, easy to inspect with a hex viewer, and the 'ö' provides the disambiguating byte.
- **O-4:** Whether `Eol::Mixed` carries `MixedMajority` (the dominant style) or just a flag. Constraints: status-bar display in Phase 4 needs the dominant style. Recommend `MixedMajority` enum carried inside `Mixed` (research.md's recommendation).

### Considered and rejected
- **R-1:** Adding `chardetng` to the allowlist for legacy code-page detection. Rejected because: research.md "Don't Add" + idea.md crate allowlist constraint. The 5-step deterministic cascade covers SC-6's explicit list (UTF-8 / UTF-16 / CP1252) without the +150KB binary-size cost. Re-evaluate post-v1 only if real-world testing surfaces detection failures.
- **R-2:** Storing fixtures as `include_bytes!` byte-arrays inside Rust source. Rejected because: research.md "Construction approach" — separate binary files under `.gitattributes binary` are cleaner, hex-inspectable in the source tree, and let `cargo test` exercise the actual file-IO path (matching what users hit). Source: research.md Test Fixture Strategy.
- **R-3:** Using `from_utf8_lossy` in step 2 of the cascade. Rejected because: research.md Anti-Pattern 3 — silently substitutes invalid bytes with `\u{FFFD}`. We need exact byte preservation; `is_ok()` is the boolean we want.
- **R-4:** A 90% or 99% EOL-majority threshold. Rejected because: research.md Pattern C / Open Question 3 — 95% is the documented compromise. Round-trip preservation is independent of this threshold (cache-emit on unedited save) — only status-bar display is affected. 95% can be tuned later if real-world files surface false-mixed classifications.
- **R-5:** Implementing `chardet`-style statistical detection in the cascade's step 5. Rejected because: deterministic CP1252 fallback (always succeeds for any byte content) is the spec-compliant minimum; statistics belong in `chardetng`. R-1 is the parent rejection.
- **R-6:** Generating fixtures inline inside each test (no `tools/generate-fixtures.rs`). Rejected because: each test would duplicate the byte-exact construction logic (8+ fixtures × N tests = combinatorial duplication); a single generator binary is the spec.

```rust
//! Regenerates Phase 2 test fixtures byte-exactly. Run via:
//!   cargo run --bin generate-fixtures --target x86_64-pc-windows-msvc
//! Idempotent — overwrites existing fixtures.

use std::fs;
use std::path::Path;

fn main() {
    let dir = Path::new("tests/fixtures");
    fs::create_dir_all(dir).unwrap_or_else(|e| {
        eprintln!("create_dir_all failed: {e}");
        std::process::exit(1);
    });

    // 1. UTF-8 (no BOM) + LF — "Hello\nWörld\n" — has 'ö' (0xC3 0xB6) for non-ASCII disambiguation.
    fs::write(dir.join("utf8_no_bom_lf.txt"),
        b"Hello\nW\xC3\xB6rld\n").unwrap();

    // 2. UTF-8 BOM + CRLF.
    let mut buf = vec![0xEF, 0xBB, 0xBF];
    buf.extend_from_slice(b"Hello\r\nW\xC3\xB6rld\r\n");
    fs::write(dir.join("utf8_bom_crlf.txt"), &buf).unwrap();

    // 3. UTF-16 LE + BOM + CRLF — encoded code-unit-by-code-unit.
    let mut buf = vec![0xFF, 0xFE]; // BOM
    for u in "Hello\r\nWörld\r\n".encode_utf16() {
        buf.extend_from_slice(&u.to_le_bytes());
    }
    fs::write(dir.join("utf16le_bom_crlf.txt"), &buf).unwrap();

    // 4. UTF-16 BE + BOM + LF.
    let mut buf = vec![0xFE, 0xFF]; // BOM
    for u in "Hello\nWörld\n".encode_utf16() {
        buf.extend_from_slice(&u.to_be_bytes());
    }
    fs::write(dir.join("utf16be_bom_lf.txt"), &buf).unwrap();

    // 5. CP1252 + CRLF — uses 0xF6 for 'ö' (the byte that's invalid UTF-8).
    fs::write(dir.join("cp1252_crlf.txt"),
        b"Hello\r\nW\xF6rld\r\n").unwrap();

    // 6. UTF-8 + CR-only line endings (classic Mac).
    fs::write(dir.join("cr_only.txt"),
        b"Hello\rWorld\r").unwrap();

    // 7. UTF-8 + Mixed (CRLF + LF; CRLF majority).
    fs::write(dir.join("mixed_eol.txt"),
        b"lineA\r\nlineB\nlineC\r\nlineD\n").unwrap();

    // 8. Empty file (0 bytes).
    fs::write(dir.join("empty.txt"), b"").unwrap();

    // 9. UTF-8 LF, last line has no trailing newline.
    fs::write(dir.join("no_trailing_newline.txt"),
        b"line1\nline2").unwrap();

    println!("10 fixtures regenerated under tests/fixtures/");
}
```
