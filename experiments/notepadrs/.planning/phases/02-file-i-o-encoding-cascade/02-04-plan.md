- **D-2:** `read_with_timeout<F>(error_path: &Path, timeout: Duration, work: F) -> Result<Vec<u8>> where F: FnOnce() -> io::Result<Vec<u8>> + Send + 'static` implements Pattern E. Generic over the work closure so deterministic tests can inject a slow synthetic worker (W-4). Spawns `std::thread::spawn` running the closure, sends the result through `mpsc::Sender`, the caller waits with `recv_timeout(timeout)`. On RecvTimeoutError, returns `Err(Error::OpenTimeout(path_string))`; the worker thread is leaked (acceptable per research.md — bounded leak, one thread per timed-out call). The production caller `open_any_encoding` passes `move || std::fs::read(&path_buf)` as the closure. Why: research.md Pattern E + Pitfall 12 + W-4 deterministic-timeout-test requirement.
- **D-3:** Default timeout is 5 seconds. Why: research.md Pattern E — "5s is a guess based on the spec." Long enough to cover slow-but-eventually-responsive shares; short enough that the user perceives "this isn't going to work" rather than "the editor is broken." Document in code comments that this is tuneable; revisit if real-world testing surfaces problems. Source: research.md Open Question (timeout value).
- **D-4:** `open_any_encoding(path: &Path)` (Plan 02-02's function) is updated to call `read_with_timeout(path.to_path_buf(), Duration::from_secs(5))` instead of bare `std::fs::read`. The detection cascade and decoding logic remain unchanged. Why: minimum-viable change; the timeout protection is orthogonal to encoding detection.
- **D-5:** Error categorization happens at the dispatch layer, not in the file layer. `src/file.rs` returns `Error::FileNotFound` / `Error::PermissionDenied` only when it can deterministically classify (e.g., `std::io::ErrorKind::NotFound` → FileNotFound, `PermissionDenied` → PermissionDenied). For ambiguous errors (other ErrorKinds, raw OS errors), it returns `Error::Io(std::io::Error)` and `dispatch::categorize_open_error` decides whether to render via FormatMessageW or the generic Display. Why: keeps the file layer's logic simple and the dispatch layer's UI logic testable.
- **D-6:** `dispatch::categorize_open_error(err: &Error, path: &Path) -> String` returns the user-facing dialog text for each variant:
  - `FileNotFound(p)` → `"File not found:\n\n{p}"` (path is from the variant; the helper just formats).
  - `PermissionDenied(p)` → `"Access denied: {p}\n\n{localized OS message via FormatMessageW(GetLastError) — fall back to generic if FormatMessageW fails}"`. Note: by the time we're in dispatch, GetLastError may have been overwritten by intermediate calls; the OS message is best-effort. If FormatMessageW returns nothing, fall back to "Check that the file is not open in another program and that you have read permission".
  - `OpenTimeout(p)` → `"Open timed out (5s): {p}\n\nThe file may be on an unresponsive network share. Try opening from a local copy."`
  - `EncodingDetectFailed(_)` → `"Could not detect encoding for:\n\n{path.display()}\n\nThe file may be binary or corrupted."`
  - `Io(io_err)` → falls through to the Phase 1 generic message: `format!("Could not open '{path.display()}':\n\n{io_err}")`. This is the catch-all for ambiguous errors.
  - All other variants (Windows, Utf8, Misc) → format with `Display` (existing Phase 1 behavior).
- **D-7:** `Win32_System_Diagnostics_Debug` is already present in the `windows` crate features list in Cargo.toml (added in Phase 1; this plan does not modify Cargo.toml). FormatMessageW signature is `FormatMessageW(dwFlags, lpSource, dwMessageId, dwLanguageId, lpBuffer, nSize, args)`; we use `FORMAT_MESSAGE_FROM_SYSTEM | FORMAT_MESSAGE_IGNORE_INSERTS`, source NULL, language 0 (default user language), buffer is a Vec<u16> with manual length. **`src/lib.rs` requires NO change in this plan:** Phase 1 already exposes `pub mod file;` and `pub mod error;`, and Plan 02-03 already adds `pub mod dispatch_pure;`. The new `categorize_open_error_with_os_msg` function added in this plan lives inside the existing `dispatch_pure` module (extending Plan 02-03's file), so no new module declaration is needed. Source: research.md "Phase 2 windows crate features to add" + Microsoft Learn + 02-03-summary.md.
- **D-8:** The CLI path argument (`std::env::args().nth(1)`) goes through `dispatch::open_path_external` — the SAME function called from File→Open. This means the FILE-11 categorized error UX automatically applies to CLI invocations: `notepadrs.exe nonexistent.txt` shows the FileNotFound MessageBox with the main window already visible (the CLI arg is processed AFTER ShowWindow per src/main.rs). Why: Phase 1 already wires this; this plan just inherits.
- **D-9:** Worker-thread cancellation is NOT implemented. The hung worker thread is leaked (research.md Pattern E "the leak is bounded"). Why: `CancelSynchronousIo` would require switching from `std::fs::read` to `CreateFileW` + `ReadFile` overlapped I/O — significant complexity for marginal gain. Document the leak in code comments; revisit only if real-world usage surfaces problems. Worker thread leak is bounded under expected user behavior; future-work backlog item if a regression appears.
- **D-10:** Tests for the timeout path do NOT actually wait 5 seconds — that would slow the test suite unacceptably. Instead, the test calls `read_with_timeout` with a deliberately small timeout (e.g., 100ms) on a path that triggers a slow read. **Realistic approach:** test the fast path (local file, expect Ok in <100ms) and unit-test the categorization logic against synthetic Error variants. The actual timeout mechanism is exercised by manual smoke (a paused VM mounting a network share) — documented in 02-04-summary.md.

### Open (executor's discretion within constraints)
- **O-1:** Whether `read_with_timeout` is `pub` in the file module or `pub(crate)`. Constraints: must be reachable from `tests/open_errors.rs`. Recommend `pub` — keeps the API surface symmetric with `open_any_encoding`.
- **O-2:** Whether the categorization logic in dispatch.rs uses `match` on the Error variants or polymorphic dispatch via a helper. Constraints: must be readable. Reason left open: `match` is idiomatic Rust; the helper-method approach is over-engineered for 5 variants. Recommend `match`.
- **O-3:** Whether to add a "Cancel" button to the open dialog while the timeout is in flight. Constraints: would require posting a custom WM_USER+N message from a UI button to interrupt the recv_timeout. Reason left open: complexity disproportionate to benefit at v1; users can wait the 5s or kill the app. Recommend deferring to v1.x. Document the design path in summary.
- **O-4:** Whether the FormatMessageW fallback (when the API itself fails or returns empty) is "Check that the file..." or a generic "OS error: {GetLastError() raw value}". Reason left open: cosmetic. Recommend the actionable English message; users who want raw error codes can read the kernel debugger.

### Considered and rejected
- **R-1:** Implementing actual cancellation via `CancelSynchronousIo`. Rejected because: research.md Pattern E + D-9 — doubles complexity for marginal gain; the v1 leak is bounded. Re-evaluate post-v1 if real-world usage shows the leak is a problem.
- **R-2:** Using `tokio::time::timeout` for the network-share hang. Rejected because: locked decision (no async runtime); std::thread + mpsc::recv_timeout is the documented pattern.
- **R-3:** Showing all four error categories with identical text "Failed to open file". Rejected because: research.md FILE-11 row + Pitfall 12 explicitly require categorized messages so the user knows whether to retry, check permissions, or move the file locally.
- **R-4:** Implementing the "open empty buffer with the filename pre-set, marked dirty" pattern from `pitfalls.md` Pitfall 12 (Notepad++/VS Code behavior for missing files). Rejected because: out of v1 Phase-2 scope — would require Save As to default to the missing path, which is a multi-touchpoint refactor that touches the Save-As dialog (Plan 02-03). The current plan's behavior — show error + leave existing buffer — is the SC-1 minimum and matches the "existing tabs are untouched" success criterion verbatim. Add to v1.x backlog as a UX polish item.
- **R-5:** Configurable timeout via Cargo feature flag or runtime arg. Rejected because: 5s is a reasonable default; configurability is YAGNI for v1.
- **R-6:** Using `Result::map_err` chains in `read_with_timeout` to convert `RecvTimeoutError` to `Error::OpenTimeout`. Rejected because: `recv_timeout` returns `Result<T, RecvTimeoutError>` and the std::sync::mpsc::RecvTimeoutError doesn't implement `Into<Error>` for our crate's Error type. Direct match-on-variant is clearer.

```rust
use std::sync::mpsc;
use std::time::Duration;

/// Pattern E — wraps an arbitrary read closure in a worker thread with a timeout.
/// Generic over the work function so deterministic tests can inject a slow closure.
/// On timeout, the worker thread is leaked (bounded — one per timed-out call).
///
/// `error_path` is used only to construct the path string for FileNotFound /
/// PermissionDenied / OpenTimeout error variants.
pub fn read_with_timeout<F>(error_path: &std::path::Path, timeout: Duration, work: F) -> Result<Vec<u8>>
where
    F: FnOnce() -> std::io::Result<Vec<u8>> + Send + 'static,
{
    let (tx, rx) = mpsc::channel();
    std::thread::spawn(move || {
        let result = work();
        let _ = tx.send(result); // best-effort send — ok if receiver dropped
    });
    let path_str = error_path.display().to_string();
    match rx.recv_timeout(timeout) {
        Ok(Ok(bytes)) => Ok(bytes),
        Ok(Err(io_err)) => {
            // Categorize at the file layer for the kinds we can detect deterministically.
            match io_err.kind() {
                std::io::ErrorKind::NotFound => Err(Error::FileNotFound(path_str)),
                std::io::ErrorKind::PermissionDenied => Err(Error::PermissionDenied(path_str)),
                _ => Err(Error::Io(io_err)),
            }
        }
        Err(_recv_timeout_err) => Err(Error::OpenTimeout(path_str)),
    }
}

/// Updated to use read_with_timeout (5s default).
pub fn open_any_encoding(path: &std::path::Path) -> Result<OpenedFile> {
    let path_buf = path.to_path_buf();
    let bytes = read_with_timeout(path, Duration::from_secs(5), move || std::fs::read(&path_buf))?;
    let (encoding, bom_len) = crate::encoding::detect_encoding(&bytes);
    let text = crate::encoding::decode(encoding, &bytes[bom_len..]);
    let eol = crate::eol::detect_eol(&text);
    let had_trailing_newline = bytes.last()
        .map(|b| *b == b'\n' || *b == b'\r')
        .unwrap_or(false);
    Ok(OpenedFile { bytes, text, encoding, eol, had_trailing_newline })
}
```

```rust
/// Render a localized OS message for the current GetLastError() value via FormatMessageW.
/// Returns None if FormatMessageW returns 0 or any error.
unsafe fn format_last_error() -> Option<String> {
    use windows::Win32::Foundation::GetLastError;
    use windows::Win32::System::Diagnostics::Debug::*;

    let err_code = GetLastError().0;
    if err_code == 0 { return None; }

    let mut buf = vec![0u16; 512];
    let len = FormatMessageW(
        FORMAT_MESSAGE_FROM_SYSTEM | FORMAT_MESSAGE_IGNORE_INSERTS,
        None,
        err_code,
        0, // default language
        windows::core::PWSTR(buf.as_mut_ptr()),
        buf.len() as u32,
        None,
    );
    if len == 0 { return None; }
    let s = String::from_utf16_lossy(&buf[..len as usize]);
    // FormatMessageW often appends \r\n — trim.
    Some(s.trim_end_matches(|c| c == '\r' || c == '\n').to_string())
}

/// Categorize an open error into the user-facing dialog message text.
/// `path` is the file the user attempted to open; used for the message body.
pub fn categorize_open_error(err: &crate::error::Error, path: &std::path::Path) -> String {
    use crate::error::Error;
    let path_display = path.display();
    match err {
        Error::FileNotFound(_) => {
            format!("File not found:\n\n{path_display}")
        }
        Error::PermissionDenied(_) => {
            // Try to render the OS-localized message via FormatMessageW.
            // Fall back to a hand-written hint if FormatMessageW fails.
            let hint = unsafe { format_last_error() }
                .unwrap_or_else(|| {
                    "Check that the file is not open in another program \
                     and that you have read permission.".to_owned()
                });
            format!("Access denied: {path_display}\n\n{hint}")
        }
        Error::OpenTimeout(_) => {
            format!(
                "Open timed out (5s): {path_display}\n\n\
                 The file may be on an unresponsive network share. \
                 Try opening from a local copy."
            )
        }
        Error::EncodingDetectFailed(_) => {
            format!(
                "Could not detect encoding for:\n\n{path_display}\n\n\
                 The file may be binary or corrupted."
            )
        }
        Error::Io(io_err) => {
            // Generic Io fallback. Render via FormatMessageW if possible.
            let os_msg = unsafe { format_last_error() }
                .unwrap_or_else(|| io_err.to_string());
            format!("Could not open '{path_display}':\n\n{os_msg}")
        }
        Error::Windows(_) | Error::Utf8(_) | Error::Misc(_) => {
            format!("Could not open '{path_display}':\n\n{err}")
        }
    }
}
```

```rust
// In src/dispatch_pure.rs (extending Plan 02-03's file — append this function):

pub fn categorize_open_error_with_os_msg(
    err: &crate::error::Error,
    path: &std::path::Path,
    os_msg_provider: impl Fn() -> Option<String>,
) -> String {
    use crate::error::Error;
    let path_display = path.display();
    match err {
        Error::FileNotFound(_) => {
            format!("File not found:\n\n{path_display}")
        }
        Error::PermissionDenied(_) => {
            let hint = os_msg_provider()
                .unwrap_or_else(|| {
                    "Check that the file is not open in another program \
                     and that you have read permission.".to_owned()
                });
            format!("Access denied: {path_display}\n\n{hint}")
        }
        Error::OpenTimeout(_) => {
            format!(
                "Open timed out (5s): {path_display}\n\n\
                 The file may be on an unresponsive network share. \
                 Try opening from a local copy."
            )
        }
        Error::EncodingDetectFailed(_) => {
            format!(
                "Could not detect encoding for:\n\n{path_display}\n\n\
                 The file may be binary or corrupted."
            )
        }
        Error::Io(io_err) => {
            let os_msg = os_msg_provider().unwrap_or_else(|| io_err.to_string());
            format!("Could not open '{path_display}':\n\n{os_msg}")
        }
        Error::Windows(_) | Error::Utf8(_) | Error::Misc(_) => {
            format!("Could not open '{path_display}':\n\n{err}")
        }
    }
}
```

Then `dispatch.rs::categorize_open_error` (added in Task 2) becomes a thin wrapper:

```rust
pub fn categorize_open_error(err: &Error, path: &Path) -> String {
    crate::dispatch_pure::categorize_open_error_with_os_msg(
        err, path,
        || unsafe { format_last_error() },
    )
}
```

This makes the pure logic testable with a stub `os_msg_provider` (e.g., `|| Some("test message".to_string())` or `|| None`). **Note:** Task 2's body of `categorize_open_error` should be refactored when this task lands — Task 2 may have inlined the match logic; this task moves the match into dispatch_pure.rs and turns dispatch.rs's version into a wrapper.
