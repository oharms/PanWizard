### Locked (executor MUST follow)
- **D-1:** EditorState gets exactly four new fields: `encoding: DetectedEncoding`, `eol: Eol`, `original_bytes: Vec<u8>`, `had_trailing_newline: bool`. Defaults at construction time (before any file is opened): `Utf8`, `Crlf` (Windows default for new buffers), empty Vec, `false`. Why: research.md Pattern A — these four pieces of metadata are exactly what's needed for byte-exact round-trip and trailing-newline preservation. Source: research.md Pattern A + Pitfall G.
- **D-2:** The open path is a single function `file::open_any_encoding(path: &Path) -> Result<OpenedFile>` returning a struct `OpenedFile { bytes: Vec<u8>, text: String, encoding: DetectedEncoding, eol: Eol, had_trailing_newline: bool }`. The caller (`dispatch::do_file_open` / `dispatch::open_path_external`) feeds `text.as_bytes()` into `EditorState::open_text_with_metadata` and stashes the metadata. Why: keeps file-IO concerns out of editor.rs and makes the open path testable headlessly (the test feeds bytes directly to open_any_encoding).
- **D-3:** `EditorState::open_text_with_metadata(text: &str, opened: OpenedFile) -> Result<()>`: streams the **decoded UTF-8 text** into RichEdit via EM_STREAMIN with `SF_TEXT | (CP_UTF8 << 16)` (Phase 1's wiring; unchanged), then sets `self.encoding = opened.encoding`, `self.eol = opened.eol`, `self.original_bytes = opened.bytes`, `self.had_trailing_newline = opened.had_trailing_newline`, `self.current_path = Some(path)`, runs `EM_SETMODIFY(0)` and `EM_EMPTYUNDOBUFFER` (existing Phase 1 discipline). Why: minimal surface change to RichEdit interaction; the encoding/eol logic is in pure-Rust modules.
- **D-4:** The save path's branch on `EM_GETMODIFY` is the heart of Pattern A:
  - If `EM_GETMODIFY == 0` (buffer unedited since last load/save): `save_atomic(path, &self.original_bytes)`. RichEdit is NOT queried. Cache is unchanged.
  - If `EM_GETMODIFY == 1` (buffer edited): EM_STREAMOUT → CR-only UTF-8 text → `normalize_to_lf` → conditionally append/strip trailing newline (Pitfall G logic) → `denormalize_to_eol(text, self.eol)` → `encode(self.encoding, &denormalized)` → `save_atomic(path, &final_bytes)` → `self.original_bytes = final_bytes` (cache refresh) → `EM_SETMODIFY(0)` (clear dirty for the next save). Why: research.md Pattern A; only design that makes byte-exact round-trip provably correct.
- **D-5:** Trailing-newline preservation rule (Pitfall G):
  - On open, compute `had_trailing_newline = bytes.last().map(|b| *b == b'\n' || *b == b'\r').unwrap_or(false)`. (Empty file → false.)
  - After EM_STREAMOUT + normalize_to_lf, the resulting text may or may not have a trailing `\n` (RichEdit may add one). Strip the trailing `\n` if present, then add it back only if `self.had_trailing_newline` was true. This handles both directions of RichEdit's paragraph-terminator behavior.
  - On edited save, do NOT update `had_trailing_newline` — preserving the original setting matches user intent (user edited the content; they didn't ask to change trailing-newline policy). If the user explicitly wants to add or remove a trailing newline, that's a separate edit operation (out of v1 scope).
- **D-6:** `file::save_atomic(target: &Path, bytes: &[u8]) -> Result<()>` implements Pattern D EXACTLY as in research.md Example 3:
  - Compute `tmp = target.with_extension("notepadrs.tmp")` (same directory → same volume, guaranteed by `with_extension` semantics).
  - `std::fs::write(&tmp, bytes)?`.
  - If `!target.exists()`: `std::fs::rename(&tmp, target)?` and return.
  - Else: convert paths to wide strings (existing pattern from Phase 1's dispatch.rs line 148), call `ReplaceFileW(target, tmp, NULL, REPLACEFILE_IGNORE_MERGE_ERRORS, NULL, NULL)`. On failure: best-effort `std::fs::remove_file(&tmp)`, return `Error::Io(std::io::Error::last_os_error())`.
  - Do NOT pass `REPLACEFILE_WRITE_THROUGH` (Microsoft documents it as "not supported"). Source: research.md Pattern D.
- **D-7:** Phase 1's `file::save_utf8` is REMOVED (its callers — `dispatch::do_file_save` — switch to `EditorState::save_text_for_disk` which delegates to `file::save_atomic` internally). Phase 1's `file::open_utf8_ascii` is KEPT to avoid breaking `tests/roundtrip_utf8_ascii.rs`, but is no longer called from production code. Why: avoids a non-atomic write path remaining as a footgun; one save path = one place to debug. Source: research.md State of the Art.
- **D-8:** `Win32_Storage_FileSystem` is added to the `windows` crate `features` list in Cargo.toml. This is the only required new feature for `ReplaceFileW` and the `REPLACEFILE_*` constants. `Win32_System_Diagnostics_Debug` (for `FormatMessageW`) is NOT needed in this plan (Plan 02-04 adds it for FILE-11's localized error messages). Source: research.md "Phase 2 windows crate features to add".
- **D-9:** The integration test `tests/roundtrip_matrix.rs` runs at the file/encoding/eol layer (NOT through HWND/RichEdit). The pattern: read fixture bytes → call `detect_encoding` + `decode` + `detect_eol` → cache bytes → call `save_atomic(tmp_path, &cached_bytes)` → read back → assert byte equality. The end-to-end through RichEdit is left to manual smoke. Why: research.md "Test Pattern (parametric)" — same justification as Phase 1 plan 01-03's `tests/roundtrip_utf8_ascii.rs`. Source: research.md.
- **D-10:** Edited-save test (`edited_save_preserves_eol_for_each_style`) exercises the encode/decode/denormalize chain WITHOUT RichEdit: simulates "user prepended 'X'" by `format!("X{text}")`, then runs the chain `denormalize_to_eol(edited, eol) → encode(enc, …)`, asserts the resulting bytes' length increased by exactly the encoded width of 'X' (1 for UTF-8/CP1252, 2 for UTF-16). Why: research.md "Edited-Save Test" — proves the dirty path's encoding/EOL preservation logic is correct independent of RichEdit's stream-out behavior.

### Considered and rejected
- **R-1:** Using `MoveFileExW(MOVEFILE_REPLACE_EXISTING | MOVEFILE_WRITE_THROUGH)` instead of `ReplaceFileW`. Rejected because: research.md Pattern D — does NOT preserve ACLs / NTFS streams / file ID. ReplaceFileW is the correct semantics for "user editing an existing file." Reserve MoveFileExW as the documented fallback if ReplaceFileW fails on certain network shares (the implementation handles this).
- **R-2:** Disabling Pattern A and always re-emitting through RichEdit on save. Rejected because: research.md Anti-Pattern 1 — RichEdit's CR-only normalization makes round-trip lossy for CRLF and LF files. Pattern A is non-negotiable.
- **R-3:** Caching the decoded UTF-8 text instead of the original bytes. Rejected because: text → bytes re-encoding is lossy in the BOM-presence dimension (encoding_rs::encode does not emit BOM) and in the EOL dimension (RichEdit normalizes CRLF→CR before EM_STREAMOUT). Caching the raw bytes is the only design that mechanically preserves both.
- **R-4:** Removing the trailing-newline-preservation logic from this plan and deferring to a v1.x patch. Rejected because: research.md Pitfall G + .planning/research/pitfalls.md Pitfall 17 — POSIX tools complain when the trailing newline is silently stripped. The fix is ~10 LOC; deferring would require a follow-up debugging session when the dogfood marker (Cargo.toml round-trip) fails.
- **R-5:** Atomic save also for first-time saves (Save As to a new path). Rejected because: ReplaceFileW requires the target file to exist (it replaces it). For Save As to a new path, `std::fs::rename(tmp, target)` is atomic on the same volume (POSIX rename + Win32 MoveFile semantics). Two paths in save_atomic, both atomic.
- **R-6:** Validating that ReplaceFileW preserved ACLs in tests. Rejected because: requires a non-trivial test fixture (file with explicit ACL set). Out of v1 scope; manual smoke test on the dogfood phase is sufficient. The Microsoft Learn doc (HIGH confidence in research.md) is the spec.

```rust
//! Phase 2 file I/O: full encoding cascade + atomic save via ReplaceFileW.

use std::path::Path;
use std::os::windows::ffi::OsStrExt;

use windows::core::PCWSTR;
use windows::Win32::Storage::FileSystem::{ReplaceFileW, REPLACEFILE_IGNORE_MERGE_ERRORS};

use crate::encoding::{detect_encoding, decode, DetectedEncoding};
use crate::eol::{detect_eol, Eol};
use crate::error::{Error, Result};

/// Result of opening a file: the raw bytes (cached for byte-exact round-trip),
/// the decoded UTF-8 text (for streaming into RichEdit), and the detected
/// encoding/EOL/trailing-newline metadata.
pub struct OpenedFile {
    pub bytes: Vec<u8>,
    pub text: String,
    pub encoding: DetectedEncoding,
    pub eol: Eol,
    pub had_trailing_newline: bool,
}

/// Read a file, detect encoding + EOL, decode to UTF-8.
/// Phase 2 successor to Phase 1's `open_utf8_ascii`.
pub fn open_any_encoding(path: &Path) -> Result<OpenedFile> {
    let bytes = std::fs::read(path)?;
    let (encoding, bom_len) = detect_encoding(&bytes);
    let text = decode(encoding, &bytes[bom_len..]);
    let eol = detect_eol(&text);
    let had_trailing_newline = bytes.last()
        .map(|b| *b == b'\n' || *b == b'\r')
        .unwrap_or(false);
    Ok(OpenedFile { bytes, text, encoding, eol, had_trailing_newline })
}

/// Atomic save via ReplaceFileW (preserves ACLs, NTFS streams, file ID).
/// First-time save (target doesn't exist): falls back to std::fs::rename.
pub fn save_atomic(target: &Path, bytes: &[u8]) -> Result<()> {
    let tmp = target.with_extension("notepadrs.tmp");
    std::fs::write(&tmp, bytes)?;

    if !target.exists() {
        // First save — just rename. Same directory => same volume => atomic on Windows.
        return std::fs::rename(&tmp, target).map_err(Into::into);
    }

    let target_w: Vec<u16> = target.as_os_str().encode_wide().chain(Some(0)).collect();
    let tmp_w: Vec<u16> = tmp.as_os_str().encode_wide().chain(Some(0)).collect();

    let result = unsafe {
        ReplaceFileW(
            PCWSTR(target_w.as_ptr()),
            PCWSTR(tmp_w.as_ptr()),
            PCWSTR::null(),
            REPLACEFILE_IGNORE_MERGE_ERRORS,
            None, None,
        )
    };

    if result.is_err() {
        // Best-effort cleanup of the orphan tmp file; report the OS error.
        let _ = std::fs::remove_file(&tmp);
        return Err(Error::Io(std::io::Error::last_os_error()));
    }
    Ok(())
}

/// Phase 1 legacy. Tests still use it via `tests/roundtrip_utf8_ascii.rs`.
#[deprecated(note = "Use open_any_encoding for full encoding cascade support")]
pub fn open_utf8_ascii(path: &Path) -> Result<Vec<u8>> {
    let bytes = std::fs::read(path)?;
    if let Err(e) = std::str::from_utf8(&bytes) {
        return Err(Error::Utf8(e));
    }
    if !bytes.iter().all(|b| *b <= 0x7F) {
        return Err(Error::Misc(
            "File is not valid UTF-8 ASCII (encoding cascade lands in Phase 2)".to_owned(),
        ));
    }
    Ok(bytes)
}
```

```rust
pub unsafe fn open_text_with_metadata(
    &mut self,
    opened: &crate::file::OpenedFile,
    path: std::path::PathBuf,
) -> Result<()> {
    use windows::Win32::UI::Controls::{EM_EMPTYUNDOBUFFER, EM_SETMODIFY};

    // Stream the decoded UTF-8 text into RichEdit. RichEdit will normalize
    // CRLF/CR to CR-only paragraph terminators internally; that's fine because
    // the original_bytes cache below preserves the on-disk representation.
    let utf8_bytes = opened.text.as_bytes();
    let mut state = StreamInState { bytes: utf8_bytes, pos: 0 };
    let es = EDITSTREAM {
        dwCookie: &mut state as *mut _ as usize,
        dwError: 0,
        pfnCallback: Some(stream_in_cb),
    };
    let result = SendMessageW(
        self.hwnd_re,
        EM_STREAMIN,
        Some(WPARAM(STREAM_FORMAT_UTF8 as usize)),
        Some(LPARAM(&es as *const _ as isize)),
    );
    if es.dwError != 0 {
        return Err(crate::error::Error::Misc(format!(
            "EM_STREAMIN failed (dwError={}, result={:?})",
            es.dwError, result
        )));
    }

    // Cache metadata for round-trip preservation.
    self.encoding = opened.encoding;
    self.eol = opened.eol;
    self.original_bytes = opened.bytes.clone();
    self.had_trailing_newline = opened.had_trailing_newline;
    self.current_path = Some(path);

    // Phase 1 discipline: clear modified flag + empty undo (load is not a user edit).
    let _ = SendMessageW(self.hwnd_re, EM_SETMODIFY, Some(WPARAM(0)), Some(LPARAM(0)));
    let _ = SendMessageW(self.hwnd_re, EM_EMPTYUNDOBUFFER, Some(WPARAM(0)), Some(LPARAM(0)));

    Ok(())
}
```

```rust
pub unsafe fn save_text_for_disk(&mut self, target_path: &std::path::Path) -> Result<()> {
    use windows::Win32::UI::Controls::{EM_GETMODIFY, EM_SETMODIFY};

    // Pattern A — Original-Bytes Cache.
    let modified: LRESULT = SendMessageW(
        self.hwnd_re,
        EM_GETMODIFY,
        Some(WPARAM(0)),
        Some(LPARAM(0)),
    );
    if modified.0 == 0 {
        // Buffer unedited since last load/save → write cached bytes verbatim.
        crate::file::save_atomic(target_path, &self.original_bytes)?;
        return Ok(());
    }

    // Buffer edited → re-emit through encoding/EOL pipeline.
    let cr_text_bytes = self.save_text()?; // existing Phase 1 EM_STREAMOUT path
    let cr_text = std::str::from_utf8(&cr_text_bytes)
        .map_err(crate::error::Error::Utf8)?;
    let mut lf_text = crate::eol::normalize_to_lf(cr_text);

    // Trailing-newline preservation (Pitfall G).
    // RichEdit may add or strip a trailing newline; reconcile against the cached flag.
    let has_trailing = lf_text.ends_with('\n');
    if has_trailing && !self.had_trailing_newline {
        lf_text.pop(); // strip RichEdit-added trailing \n
    } else if !has_trailing && self.had_trailing_newline {
        lf_text.push('\n'); // restore stripped trailing \n
    }

    let denormalized = crate::eol::denormalize_to_eol(&lf_text, self.eol);
    let final_bytes = crate::encoding::encode(self.encoding, &denormalized);

    crate::file::save_atomic(target_path, &final_bytes)?;

    // Cache refresh: the new on-disk content is the new round-trip target.
    self.original_bytes = final_bytes;

    // Clear modify flag so the next Ctrl+S takes the cache-emit fast path.
    let _ = SendMessageW(self.hwnd_re, EM_SETMODIFY, Some(WPARAM(0)), Some(LPARAM(0)));
    Ok(())
}
```

```rust
#[test]
fn edited_save_preserves_eol_for_each_style() {
    use notepadrs::encoding::encode;

    let cases = [
        ("tests/fixtures/utf8_no_bom_lf.txt",  DetectedEncoding::Utf8,    Eol::Lf,   1),
        ("tests/fixtures/utf8_bom_crlf.txt",   DetectedEncoding::Utf8Bom, Eol::Crlf, 1),
        ("tests/fixtures/utf16le_bom_crlf.txt", DetectedEncoding::Utf16Le, Eol::Crlf, 2),
        ("tests/fixtures/utf16be_bom_lf.txt",  DetectedEncoding::Utf16Be, Eol::Lf,   2),
        ("tests/fixtures/cp1252_crlf.txt",     DetectedEncoding::Cp1252,  Eol::Crlf, 1),
    ];
    for (path, enc, eol, x_width) in cases {
        let opened = open_any_encoding(Path::new(path)).unwrap();
        assert_eq!(opened.encoding, enc, "encoding mismatch in case {path}");

        // Simulate "user prepended 'X'" onto the LF-canonical text.
        let lf_text = notepadrs::eol::normalize_to_lf(&opened.text);
        let edited = format!("X{}", lf_text);
        let denormalized = denormalize_to_eol(&edited, eol);
        let bytes = encode(enc, &denormalized);

        // Property: byte length increased by exactly x_width plus any EOL overhead.
        // For LF-only fixtures: no EOL overhead change. For CRLF, the denormalize
        // step expands `\n` → `\r\n` for every newline; that's already reflected
        // in `original.len()` because the original was CRLF-encoded too.
        // Pure-content delta = x_width bytes (1 for ASCII/CP1252, 2 for UTF-16).
        let expected_min = opened.bytes.len() + x_width;
        assert!(bytes.len() >= expected_min,
            "edited-save length too small for {path}: \
             original.len={}, expected ≥ {}, got {}",
            opened.bytes.len(), expected_min, bytes.len());
        // Also: the saved bytes' first content byte (after BOM) must be the
        // encoded representation of 'X'.
        let bom_skip = match enc {
            DetectedEncoding::Utf8Bom => 3,
            DetectedEncoding::Utf16Le | DetectedEncoding::Utf16Be => 2,
            _ => 0,
        };
        match enc {
            DetectedEncoding::Utf16Le => {
                assert_eq!(bytes[bom_skip..bom_skip+2], [0x58, 0x00],
                    "first content code unit must be 'X' in UTF-16 LE for {path}");
            }
            DetectedEncoding::Utf16Be => {
                assert_eq!(bytes[bom_skip..bom_skip+2], [0x00, 0x58],
                    "first content code unit must be 'X' in UTF-16 BE for {path}");
            }
            _ => {
                assert_eq!(bytes[bom_skip], b'X',
                    "first content byte must be 'X' for {path}");
            }
        }
    }
}
```
