## Plan Decisions

### Locked (executor MUST follow)
- **D-1:** File open uses `EM_STREAMIN` (NOT `WM_SETTEXT` or `EM_SETTEXTEX`). Source bytes are passed UTF-8 with `SF_TEXT | (CP_UTF8 << 16)` flags. Why: pitfalls.md Pitfall 4 — `EM_SETTEXTEX` silently normalizes line endings; `EM_STREAMIN` preserves them. Locked in context.md and required for Phase 2's round-trip test to work without retrofit. Source: research.md Pattern 5 + Pitfall E.
- **D-2:** File save uses `EM_STREAMOUT` (matching `EM_STREAMIN`'s round-trip semantics). Same `SF_TEXT | (CP_UTF8 << 16)` flags. Output is `Vec<u8>` written via `std::fs::write` (no atomic-save in Phase 1; that lands in Phase 2 with the full encoding cascade — pitfalls.md Pitfall 12 marks atomic save as Phase-2 work). Why: matches D-1; preserves whatever line endings RichEdit holds.
- **D-3:** Phase 1 accepts ONLY UTF-8 that is also valid ASCII (every byte ≤ 0x7F). The full UTF-8 cascade (with non-ASCII codepoints, BOM detection, etc.) is Phase 2. Reason: `std::str::from_utf8` over Vec<u8> validates UTF-8; an additional ASCII-byte check (`bytes.iter().all(|b| *b <= 0x7F)`) gates whether Phase 1 can accept the file. Why: research.md FILE-01/FILE-02 row + the success criterion explicitly says "UTF-8 ASCII content" for Phase 1.
- **D-4:** Save target is the path the file was opened from (stored in `EditorState::current_path` from Plan 01-02). If `current_path` is `None` (user typed into a fresh buffer), Ctrl+S shows a MessageBox "Save As is not yet implemented (Phase 2)". This is a Phase-1 limitation explicitly accepted in research.md and the roadmap (FILE-04 "Save As" is Phase 2's responsibility).
- **D-5:** Open errors (file-not-found, permission-denied, not-UTF-8-ASCII) show a MessageBoxW with a clear message (the OS error string for I/O errors; "File is not valid UTF-8 ASCII (encoding cascade lands in Phase 2)" for the encoding case) and do NOT modify the editor's current buffer. The error path is the only explicit Phase-1 response to FILE-11 (which is fully addressed in Phase 2). Why: pitfalls.md Pitfall 12 — "save errors are user-facing, never silent."
- **D-6:** CLI path argument: `std::env::args().nth(1)` is parsed AFTER window creation (so any error dialog has the main window as parent). If the path is invalid, the same MessageBox path applies; the empty editor remains visible.
- **D-7:** The integration test `tests/roundtrip_utf8_ascii.rs` does NOT instantiate a Win32 window. Instead, it tests the `file::open_utf8_ascii` and `file::save_utf8` functions directly — these are the byte-level pure-Rust I/O — and asserts byte equality. The end-to-end integration through RichEdit is left to manual smoke (because spinning up a real HWND in a unit test is brittle and adds CI complexity disproportionate to value at this stage). Why: research.md Test Map "T3 (manual smoke + integration)" — the unit-level round-trip is the actionable automated check.
- **D-8:** For Phase 1, the streaming callbacks for `EM_STREAMIN`/`EM_STREAMOUT` are minimal: input callback feeds bytes from a `&[u8]`, output callback appends bytes to a `Vec<u8>`. Both are `extern "system" fn` per the `EDITSTREAM` ABI. Implementation in `src/editor.rs`.

### Open (executor's discretion within constraints)
- **O-1:** Whether `file::open_utf8_ascii` and `file::save_utf8` are public free functions or methods on a `FileIo` struct. Constraints: must be callable from `dispatch.rs` and from `tests/roundtrip_utf8_ascii.rs`. Reason left open: free functions are simpler for Phase 1; struct may be useful in Phase 2 if FileIo grows state. Recommend free functions.
- **O-2:** Whether `EditorState::open_text` accepts `&[u8]` or `&str`. Constraints: must work with `EM_STREAMIN`'s UTF-8 callback. Reason left open: `&[u8]` is more direct for the FFI; `&str` adds an extra allocation but is more Rust-idiomatic. Recommend `&[u8]` since the bytes go straight into the streamer.
- **O-3:** Whether the File menu also includes "Exit" or relies on Alt+F4 / window close button. Constraints: IDM_FILE_EXIT was reserved in Plan 01-02. Reason left open: cosmetic; Notepad++ has both. Recommend including File→Exit since the command ID is already reserved.
- **O-4:** Whether `GetOpenFileNameW` filter defaults to `"All Files\0*.*\0\0"` or `"Text Files\0*.txt\0All Files\0*.*\0\0"`. Constraints: must allow opening any file (UTF-8 ASCII validation happens after read, regardless of extension). Reason left open: Notepad++ defaults to "All Files"; that matches Phase 1's spirit. Recommend "All Files".

### Considered and rejected
- **R-1:** Using `WM_SETTEXT` to load file contents. Rejected because: D-1 — silently normalizes line endings; would force Phase 2 to switch to `EM_STREAMIN` later anyway. Use `EM_STREAMIN` from day one.
- **R-2:** Adding atomic-save (`MoveFileExW(MOVEFILE_REPLACE_EXISTING | MOVEFILE_WRITE_THROUGH)`) in Phase 1. Rejected because: pitfalls.md Pitfall 12 maps atomic save to Phase 2 (the encoding-cascade phase); Phase 1's round-trip is checked by tests, and `std::fs::write` is sufficient for the dogfood pre-test. Adds complexity without Phase-1 benefit.
- **R-3:** Implementing Save As (FILE-04) in Phase 1. Rejected because: explicitly out of scope per context.md "Deferred Ideas" and roadmap.md (FILE-04 is in Phase 2). Phase 1's `current_path` is set when the file was opened; saving without an open path shows a Phase-2 placeholder MessageBox.
- **R-4:** Using `encoding_rs` for the UTF-8 path. Rejected because: `std::str::from_utf8` validates UTF-8 in std with no extra dep; `encoding_rs` adds compile time and binary size for zero Phase-1 benefit (its real value is the UTF-16/CP1252 detectors in Phase 2).
- **R-5:** Adding a "modified" flag and "* notepadrs - filename" title bar. Rejected because: Phase 1 success criteria do not require the dirty marker; it lands in Phase 3 with multi-tab. Adding it now would be partial implementation flagged "no half-finished" per project.md.

//! Phase 1 file I/O: UTF-8 ASCII open and save.
//! Phase 2 replaces this with the full encoding cascade (BOM, UTF-16 LE/BE, CP1252).

use std::path::Path;

use crate::error::{Error, Result};

/// Read a file and verify it contains only UTF-8 ASCII (every byte <= 0x7F).
/// Returns the bytes as-is on success — line endings preserved.
pub fn open_utf8_ascii(path: &Path) -> Result<Vec<u8>> {
    let bytes = std::fs::read(path)?;

    // Validate UTF-8 (subset: ASCII).
    if let Err(e) = std::str::from_utf8(&bytes) {
        return Err(Error::Utf8(e));
    }
    if !bytes.iter().all(|b| *b <= 0x7F) {
        return Err(Error::Misc(
            "File is not valid UTF-8 ASCII (encoding cascade lands in Phase 2)".to_owned(),
        ));
    }
    Ok(bytes)
}

/// Write bytes to disk. Phase 1: simple std::fs::write.
/// Phase 2 will switch this to atomic-save via MoveFileExW.
pub fn save_utf8(path: &Path, bytes: &[u8]) -> Result<()> {
    std::fs::write(path, bytes)?;
    Ok(())
}
```

// Add these imports at the top of src/editor.rs:
// use windows::Win32::UI::Controls::RichEdit::{EM_STREAMIN, EM_STREAMOUT, EDITSTREAM, SF_TEXT, SF_USECODEPAGE};

// Constants for the streaming flags. SF_TEXT = 1, SF_USECODEPAGE = 0x20.
// CP_UTF8 = 65001 packed into the high WORD of the format LPARAM.
const SF_TEXT_CONST: u32 = 0x0001;
const SF_USECODEPAGE_CONST: u32 = 0x0020;
const CP_UTF8: u32 = 65001;

#[repr(C)]
struct StreamInState<'a> {
    bytes: &'a [u8],
    pos: usize,
}

#[repr(C)]
struct StreamOutState {
    out: Vec<u8>,
}

unsafe extern "system" fn stream_in_cb(
    cookie: usize,
    buf: *mut u8,
    cb: i32,
    pcb: *mut i32,
) -> i32 {
    if cookie == 0 || buf.is_null() || pcb.is_null() || cb <= 0 {
        return 1; // failure
    }
    let state = &mut *(cookie as *mut StreamInState);
    let remaining = state.bytes.len().saturating_sub(state.pos);
    let to_copy = remaining.min(cb as usize);
    if to_copy > 0 {
        std::ptr::copy_nonoverlapping(
            state.bytes.as_ptr().add(state.pos),
            buf,
            to_copy,
        );
        state.pos += to_copy;
    }
    *pcb = to_copy as i32;
    0 // success
}

unsafe extern "system" fn stream_out_cb(
    cookie: usize,
    buf: *const u8,
    cb: i32,
    pcb: *mut i32,
) -> i32 {
    if cookie == 0 || buf.is_null() || pcb.is_null() || cb < 0 {
        return 1;
    }
    let state = &mut *(cookie as *mut StreamOutState);
    if cb > 0 {
        let slice = std::slice::from_raw_parts(buf, cb as usize);
        state.out.extend_from_slice(slice);
    }
    *pcb = cb;
    0
}

impl EditorState {
    /// Stream UTF-8 bytes into the RichEdit control via EM_STREAMIN. Preserves
    /// whatever line endings the bytes have. Bytes must be valid UTF-8 (caller
    /// has already validated via file::open_utf8_ascii in Phase 1).
    pub unsafe fn open_text(&mut self, utf8_bytes: &[u8], path: std::path::PathBuf)
        -> crate::error::Result<()>
    {
        use windows::Win32::Foundation::*;
        use windows::Win32::UI::Controls::RichEdit::*;
        use windows::Win32::UI::WindowsAndMessaging::*;

        let mut state = StreamInState { bytes: utf8_bytes, pos: 0 };
        let es = EDITSTREAM {
            dwCookie: &mut state as *mut _ as usize,
            dwError: 0,
            pfnCallback: Some(std::mem::transmute::<unsafe extern "system" fn(usize, *mut u8, i32, *mut i32) -> i32, EDITSTREAMCALLBACK>(stream_in_cb)),
        };
        let format = (SF_TEXT_CONST | SF_USECODEPAGE_CONST) | (CP_UTF8 << 16);
        let result = SendMessageW(
            self.hwnd_re,
            EM_STREAMIN,
            WPARAM(format as usize),
            LPARAM(&es as *const _ as isize),
        );
        if es.dwError != 0 {
            return Err(crate::error::Error::Misc(
                format!("EM_STREAMIN failed (dwError={}, result={:?})", es.dwError, result)
            ));
        }
        // Clear modified flag — fresh load.
        let _ = SendMessageW(self.hwnd_re, EM_SETMODIFY, WPARAM(0), LPARAM(0));
        // Empty undo buffer (load is not a user edit).
        let _ = SendMessageW(self.hwnd_re, EM_EMPTYUNDOBUFFER, WPARAM(0), LPARAM(0));

        self.current_path = Some(path);
        Ok(())
    }

    /// Stream UTF-8 bytes out of the RichEdit control via EM_STREAMOUT. Returns
    /// the bytes with whatever line endings RichEdit holds.
    pub unsafe fn save_text(&self) -> crate::error::Result<Vec<u8>> {
        use windows::Win32::Foundation::*;
        use windows::Win32::UI::Controls::RichEdit::*;
        use windows::Win32::UI::WindowsAndMessaging::*;

        let mut state = StreamOutState { out: Vec::new() };
        let es = EDITSTREAM {
            dwCookie: &mut state as *mut _ as usize,
            dwError: 0,
            pfnCallback: Some(std::mem::transmute::<unsafe extern "system" fn(usize, *const u8, i32, *mut i32) -> i32, EDITSTREAMCALLBACK>(stream_out_cb)),
        };
        let format = (SF_TEXT_CONST | SF_USECODEPAGE_CONST) | (CP_UTF8 << 16);
        let _ = SendMessageW(
            self.hwnd_re,
            EM_STREAMOUT,
            WPARAM(format as usize),
            LPARAM(&es as *const _ as isize),
        );
        if es.dwError != 0 {
            return Err(crate::error::Error::Misc(
                format!("EM_STREAMOUT failed (dwError={})", es.dwError)
            ));
        }
        Ok(state.out)
    }
}
```

**On the EDITSTREAMCALLBACK ABI:** the `windows` crate defines `EDITSTREAMCALLBACK` as `Option<unsafe extern "system" fn(DWORD_PTR, *mut u8, LONG, *mut LONG) -> LONG>`. Note that `EM_STREAMIN`'s callback receives `*mut u8` (write target) while `EM_STREAMOUT`'s receives `*const u8` (read source from the perspective of the user — but the type is shared, so we use `*mut u8` everywhere and treat it as `*const u8` for stream-out's read). The transmute above forces the Rust function pointer to match the typedef. **Verify** the callback signature on docs.rs/windows/0.62.2; if it differs, adjust the function signature to match.

**Note on `EDITSTREAMCALLBACK` — `windows` crate sometimes types this as `Option<...>`:** if so, wrap with `Some(stream_in_cb)` directly (no transmute needed). The transmute above is a guard against the type mismatch; if the direct assignment compiles, prefer that path. The executor should try the simpler form first.

unsafe fn do_file_open(app: &mut App, hwnd: HWND) {
    use windows::Win32::UI::Controls::Dialogs::*;
    use windows::Win32::Foundation::*;

    let mut buf = [0u16; 260]; // MAX_PATH
    let filter: Vec<u16> = "All Files\0*.*\0\0".encode_utf16().collect();
    let mut ofn = OPENFILENAMEW {
        lStructSize: std::mem::size_of::<OPENFILENAMEW>() as u32,
        hwndOwner: hwnd,
        lpstrFilter: windows::core::PCWSTR(filter.as_ptr()),
        lpstrFile: windows::core::PWSTR(buf.as_mut_ptr()),
        nMaxFile: buf.len() as u32,
        Flags: OFN_FILEMUSTEXIST | OFN_PATHMUSTEXIST,
        ..Default::default()
    };
    if !GetOpenFileNameW(&mut ofn).as_bool() {
        return; // user cancelled
    }
    // Convert UTF-16 path back to PathBuf.
    let len = buf.iter().position(|&c| c == 0).unwrap_or(buf.len());
    let path = std::path::PathBuf::from(String::from_utf16_lossy(&buf[..len]));
    open_path_external(hwnd, &path);
}

pub unsafe fn open_path_external(hwnd: HWND, path: &std::path::Path) {
    use windows::Win32::UI::WindowsAndMessaging::*;

    let app_ptr = GetWindowLongPtrW(hwnd, GWLP_USERDATA) as *mut App;
    if app_ptr.is_null() { return; }
    let app = &mut *app_ptr;

    let bytes = match crate::file::open_utf8_ascii(path) {
        Ok(b) => b,
        Err(e) => { show_error(hwnd, &format!("Could not open '{}':\n\n{}", path.display(), e)); return; }
    };
    if let Some(ed) = app.editor.as_mut() {
        if let Err(e) = ed.open_text(&bytes, path.to_path_buf()) {
            show_error(hwnd, &format!("Could not load '{}' into editor:\n\n{}", path.display(), e));
        }
    }
}

unsafe fn do_file_save(app: &mut App, hwnd: HWND) {
    let path = match app.editor.as_ref().and_then(|e| e.current_path.clone()) {
        Some(p) => p,
        None => {
            show_error(hwnd, "Save As is not yet implemented (Phase 2).\nOpen a file first via File→Open or the CLI.");
            return;
        }
    };
    let bytes = match app.editor.as_ref() {
        Some(ed) => match ed.save_text() {
            Ok(b) => b,
            Err(e) => { show_error(hwnd, &format!("Could not extract editor content:\n\n{}", e)); return; }
        },
        None => return,
    };
    if let Err(e) = crate::file::save_utf8(&path, &bytes) {
        show_error(hwnd, &format!("Could not save '{}':\n\n{}", path.display(), e));
    }
}

unsafe fn show_error(hwnd: HWND, message: &str) {
    use windows::Win32::UI::WindowsAndMessaging::*;
    let mut msg: Vec<u16> = format!("{}\0", message).encode_utf16().collect();
    let mut title: Vec<u16> = "notepadrs\0".encode_utf16().collect();
    MessageBoxW(Some(hwnd), windows::core::PCWSTR(msg.as_mut_ptr()), windows::core::PCWSTR(title.as_mut_ptr()), MB_OK | MB_ICONERROR);
}

    // Open via the same path the editor uses.
    let opened = file::open_utf8_ascii(fixture).expect("ASCII fixture must open");

    let _ = std::fs::remove_file(&out_path);

    assert_eq!(saved, original,
