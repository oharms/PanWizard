## Plan Decisions

### Locked (executor MUST follow)
- **D-1:** The undo model is bounded to **1000 ops** as a `VecDeque<Edit>` with FIFO eviction. Why: locked in context.md ("Per-tab undo bounded ~1000 ops"). The number 1000 is the SC-7 number from idea.md.
- **D-2:** The contract under test is `redo(undo(state)) == state` for every prefix — i.e., for any sequence of operations applied to an initial state, undoing the last operation and then redoing it must yield the same state. The test uses a property-style approach: 100+ random sequences of operations, seeded for determinism. Why: locked in context.md ("Property test: redo(undo(state)) == state").
- **D-3:** A **single buffer-mutation chokepoint** clears the redo stack on every new edit. The model exposes exactly one mutation entry point (`apply(&mut self, edit: Edit)`) which (a) inverts the edit to compute the inverse, (b) pushes the inverse onto the undo stack, (c) clears the redo stack unconditionally. There is no other public mutation method. Why: pitfalls.md Pitfall 6 — drift bug; locked in context.md.
- **D-4:** Coalescing rules (mirrors pitfalls.md Pitfall 6 + research.md Pattern 3):
  - Group consecutive **insert** operations IF they are temporally adjacent (no explicit boundary between) AND the inserts are at adjacent byte offsets (cursor moved forward between them, no jump).
  - Break the group on: explicit boundary (`commit_group()`), insertion at a non-adjacent offset, transition between insert and delete, deletion direction change.
  - Phase 1's coalescing model uses **explicit boundaries** (the test calls `commit_group()` to mark phrases) rather than time-based — wall-clock idle-timeout testing is non-deterministic and not appropriate for `cargo test`. The wall-clock variant lands in Phase 4 or later when there's an event loop driving it.
- **D-5:** Property test uses a **deterministic seeded PRNG** (`StdRng::seed_from_u64(0xC0FFEE)`). Why: reproducibility — flaky property tests are worse than no tests. Source: pitfalls.md anti-pattern guidance + standard property-testing wisdom. The `rand` crate is NOT on the allowlist; instead use a simple linear-congruential generator (LCG) hand-rolled in the test (~10 LOC). This avoids adding a dependency.
- **D-6:** The `Edit` enum has exactly two variants: `Insert { at: usize, bytes: Vec<u8> }` and `Delete { at: usize, bytes: Vec<u8> }`. The state being modeled is `Vec<u8>` (not `String`) — bytes match what RichEdit holds and matches Phase 2's encoding-aware byte buffer. Why: simplest model that matches the eventual real implementation. Source: research.md Pattern 3.
- **D-7:** The `UndoModel` is generic over the buffer type for testability — `UndoModel<B: BufferOps>` where `BufferOps` defines `insert(at, bytes)` and `delete(at, len) -> bytes` — but the property test concretely uses `Vec<u8>`. The generic shape is for v2's flexibility; Phase 1 only needs the `Vec<u8>` instantiation. **Simplification:** if generic introduces compile-time complexity, drop the trait and hard-code `Vec<u8>` operations. The test's invariants don't depend on the abstraction. Recommend: hard-code `Vec<u8>` in Phase 1.

  <files>src/undo_model.rs, src/lib.rs, tests/undo_property.rs</files>
  <behavior>
The model represents a `Vec<u8>` buffer with insert/delete operations. The contract:

1. **State equality after redo(undo)**: For any state S after a sequence of `apply` calls, calling `undo()` then `redo()` returns to S byte-exact. Formally:
   - Given S, e₁..eₙ, apply(e₁), ..., apply(eₙ), undo() → S', redo() → S''. Assert S == S''.

2. **State equality after multiple undo/redo**: After K undos followed by K redos, state == before-the-undos. Formally:
   - apply(e₁..eₙ), state₀ = state(); undo() × K, redo() × K. Assert state() == state₀.

3. **Redo stack clears on new edit**: After `apply(e₁), undo(), apply(e₂)`, calling `redo()` returns false (or yields no state change) — the original e₁ cannot be redone after e₂ replaces it.

4. **Bound enforcement**: Pushing > 1000 edits results in `undo_stack.len() == 1000`; the oldest edits are dropped from the front.

5. **Explicit-boundary coalescing**: Consecutive `apply(Insert { at: i, bytes: b₁ }), apply(Insert { at: i + b₁.len(), bytes: b₂ }), apply(Insert { at: i + b₁.len() + b₂.len(), bytes: b₃ })` (typing forward without commit_group between) groups into a single undo step. After this sequence, `undo()` reverses ALL THREE inserts in one call; `undo_stack.len() == 1` (not 3).
   After `commit_group()` is called between two inserts, they remain separate undo steps.

Cases (random sequences):
- Empty state, 100 random insert/delete operations, then 100 random undo calls, then 100 random redo calls. Assert state matches snapshot taken after the apply phase.
- Empty state, 1500 random inserts. Assert undo_stack.len() == 1000 (FIFO truncation).
- Specific case: insert "Hello" (5 chars), no boundary, insert " world" (6 chars). undo() → buffer == "" (entire group reversed); redo() → buffer == "Hello world".
- Specific case: insert "Hello", commit_group(), insert " world". undo() → buffer == "Hello"; undo() → buffer == ""; redo() → "Hello"; redo() → "Hello world".
  </behavior>

```rust
//! Pure-logic undo model for Phase 1's TEST-05 contract.
//!
//! NOT WIRED INTO src/editor.rs — RichEdit owns the live undo at runtime in v1.
//! This model is the formal contract that any future undo implementation
//! (Phase 2's per-tab encoding state, a v2 custom-render switch) must satisfy.
//!
//! Single buffer-mutation chokepoint: only `apply()` mutates state and undo/redo
//! stacks. `apply()` clears the redo stack unconditionally before pushing.

use std::collections::VecDeque;

/// Maximum undo depth (project decision, idea.md SC-7).
pub const UNDO_LIMIT: usize = 1000;

#[derive(Debug, Clone, PartialEq, Eq)]
pub enum Edit {
    Insert { at: usize, bytes: Vec<u8> },
    Delete { at: usize, bytes: Vec<u8> },
}

/// One undo step. May represent a single Edit or a coalesced group of Edits
/// (which apply/undo as one logical operation).
#[derive(Debug, Clone)]
struct UndoStep {
    /// In-order edits to redo this step. Reversed and inverted for undo.
    edits: Vec<Edit>,
}

impl UndoStep {
    fn single(edit: Edit) -> Self { Self { edits: vec![edit] } }
}

pub struct UndoModel {
    buffer: Vec<u8>,
    undo_stack: VecDeque<UndoStep>,
    redo_stack: VecDeque<UndoStep>,
    /// True between `apply` calls within a coalescing group. Reset by commit_group()
    /// or by any non-insert apply.
    in_group: bool,
    /// Last insert end-offset, for adjacent-insert coalescing detection.
    last_insert_end: Option<usize>,
}

impl UndoModel {
    pub fn new() -> Self {
        Self {
            buffer: Vec::new(),
            undo_stack: VecDeque::new(),
            redo_stack: VecDeque::new(),
            in_group: false,
            last_insert_end: None,
        }
    }

    pub fn from_bytes(bytes: Vec<u8>) -> Self {
        Self { buffer: bytes, ..Self::new() }
    }

    pub fn state(&self) -> &[u8] { &self.buffer }

    /// Single chokepoint. Always clears the redo stack.
    pub fn apply(&mut self, edit: Edit) -> Result<(), String> {
        // Compute inverse and apply forward.
        let inverse = match &edit {
            Edit::Insert { at, bytes } => {
                if *at > self.buffer.len() { return Err(format!("insert at out of range: {at} > {}", self.buffer.len())); }
                self.buffer.splice(*at..*at, bytes.iter().copied());
                Edit::Delete { at: *at, bytes: bytes.clone() }
            }
            Edit::Delete { at, bytes } => {
                let end = at.checked_add(bytes.len()).ok_or("overflow")?;
                if end > self.buffer.len() { return Err(format!("delete out of range: {at}..{end} > {}", self.buffer.len())); }
                if &self.buffer[*at..end] != &bytes[..] {
                    return Err("delete: bytes do not match buffer at offset".to_owned());
                }
                self.buffer.drain(*at..end);
                Edit::Insert { at: *at, bytes: bytes.clone() }
            }
        };

        // Coalescing logic. Group consecutive adjacent-forward inserts when in_group is true.
        let coalesced = match (&edit, self.in_group, self.last_insert_end, self.undo_stack.back_mut()) {
            (Edit::Insert { at, bytes }, true, Some(last_end), Some(top))
                if *at == last_end =>
            {
                top.edits.push(inverse.clone());
                self.last_insert_end = Some(at + bytes.len());
                true
            }
            _ => false,
        };

        if !coalesced {
            self.undo_stack.push_back(UndoStep::single(inverse));
            if self.undo_stack.len() > UNDO_LIMIT {
                self.undo_stack.pop_front();
            }
            // Track for next adjacency check.
            self.in_group = matches!(&edit, Edit::Insert { .. });
            self.last_insert_end = match &edit {
                Edit::Insert { at, bytes } => Some(at + bytes.len()),
                _ => None,
            };
        }

        // Single-chokepoint requirement: any new edit clears redo.
        self.redo_stack.clear();
        Ok(())
    }

    /// Mark the end of a coalescing group. Subsequent inserts will not coalesce
    /// with the most recent step.
    pub fn commit_group(&mut self) {
        self.in_group = false;
        self.last_insert_end = None;
    }

    /// Returns true if an undo was performed.
    pub fn undo(&mut self) -> bool {
        let Some(step) = self.undo_stack.pop_back() else { return false; };

        // Inverse-apply each edit IN REVERSE ORDER. The inverse of the inverse
        // is the original — so to undo, we re-apply the inverses we stored.
        // But the stored "inverse" IS the undo direction; applying it directly
        // undoes the operation.
        let mut redo_step = UndoStep { edits: Vec::with_capacity(step.edits.len()) };
        for edit in step.edits.iter().rev() {
            let result = match edit {
                Edit::Insert { at, bytes } => {
                    self.buffer.splice(*at..*at, bytes.iter().copied());
                    Edit::Delete { at: *at, bytes: bytes.clone() }
                }
                Edit::Delete { at, bytes } => {
                    let end = at + bytes.len();
                    self.buffer.drain(*at..end);
                    Edit::Insert { at: *at, bytes: bytes.clone() }
                }
            };
            redo_step.edits.push(result);
        }
        // redo_step.edits is in reverse-of-reverse = forward order? We pushed
        // inverses-of-inverses (forward edits) in reverse iteration order, so
        // the redo_step.edits is in REVERSE of the original step. Reverse to restore.
        redo_step.edits.reverse();
        self.redo_stack.push_back(redo_step);
        // Reset coalescing state — undo invalidates adjacency.
        self.in_group = false;
        self.last_insert_end = None;
        true
    }

    pub fn redo(&mut self) -> bool {
        let Some(step) = self.redo_stack.pop_back() else { return false; };
        // Apply each forward edit, building a new undo step.
        let mut undo_step = UndoStep { edits: Vec::with_capacity(step.edits.len()) };
        for edit in &step.edits {
            let inverse = match edit {
                Edit::Insert { at, bytes } => {
                    self.buffer.splice(*at..*at, bytes.iter().copied());
                    Edit::Delete { at: *at, bytes: bytes.clone() }
                }
                Edit::Delete { at, bytes } => {
                    let end = at + bytes.len();
                    self.buffer.drain(*at..end);
                    Edit::Insert { at: *at, bytes: bytes.clone() }
                }
            };
            undo_step.edits.push(inverse);
        }
        // We pushed inverses in forward iteration order, so undo_step.edits is forward.
        // For undo to replay them in reverse, that's already what undo() does.
        self.undo_stack.push_back(undo_step);
        if self.undo_stack.len() > UNDO_LIMIT {
            self.undo_stack.pop_front();
        }
        self.in_group = false;
        self.last_insert_end = None;
        true
    }

    pub fn undo_depth(&self) -> usize { self.undo_stack.len() }
    pub fn redo_depth(&self) -> usize { self.redo_stack.len() }
}

impl Default for UndoModel {
    fn default() -> Self { Self::new() }
}
```

**Note on the inverse-edit logic**: the model stores the *inverse* of each forward edit on the undo stack, so undoing means applying the inverse to the buffer (which actually applies a forward edit relative to the undo stack — i.e., re-creates the deleted bytes or removes the inserted bytes). This requires careful reverse-iteration when undoing a coalesced multi-edit group: the most-recently-applied edit must be undone first. The implementation above iterates `step.edits.iter().rev()` to do this.

If the implementation as written has bugs (the in-place inverse-of-inverse logic is subtle), the executor must adjust. The property test will catch correctness bugs — that's the entire point of the TDD ordering.

//! TEST-05 property test for the undo contract. Uses a deterministic LCG —
//! no `rand` dependency (not on allowlist).

use notepadrs::undo_model::{Edit, UndoModel, UNDO_LIMIT};

/// Linear congruential generator. Numerical Recipes constants. Deterministic seed.
struct Lcg(u64);
impl Lcg {
    fn new(seed: u64) -> Self { Self(seed.wrapping_add(0xC0FFEE_DEAD_BEEF)) }
    fn next_u64(&mut self) -> u64 {
        self.0 = self.0.wrapping_mul(6364136223846793005).wrapping_add(1442695040888963407);
        self.0
    }
    fn range(&mut self, n: usize) -> usize {
        if n == 0 { 0 } else { (self.next_u64() as usize) % n }
    }
}

/// Generate a random valid edit for the current buffer length.
fn random_edit(rng: &mut Lcg, buf_len: usize) -> Edit {
    if buf_len == 0 || rng.next_u64() % 2 == 0 {
        // Insert.
        let at = rng.range(buf_len + 1);
        let len = 1 + rng.range(8);
        let bytes: Vec<u8> = (0..len).map(|_| (b'a' + (rng.range(26) as u8))).collect();
        Edit::Insert { at, bytes }
    } else {
        // Delete (length 1..=4 if possible).
        let at = rng.range(buf_len);
        let max_len = (buf_len - at).min(4);
        let len = 1 + rng.range(max_len.max(1));
        // Need to know the actual bytes at the offset for the model to validate them.
        // For the test we just generate and let the model error if mismatched —
        // but here we have to ask the model for the bytes. Easier: do a "marker"
        // delete that the test driver fills in from the model state. So in the
        // driver, after picking 'at' and 'len', read state slice and use it.
        Edit::Delete { at, bytes: vec![0u8; len] }  // placeholder; driver fills bytes
    }
}

#[test]
fn redo_undo_is_identity_random_sequences() {
    let mut rng = Lcg::new(0xC0FFEE);
    for trial in 0..32 {
        let mut model = UndoModel::new();
        let n_ops = 30 + rng.range(50);
        for _ in 0..n_ops {
            let mut e = random_edit(&mut rng, model.state().len());
            // Patch delete bytes from the actual buffer.
            if let Edit::Delete { at, bytes } = &mut e {
                let buf = model.state();
                let len = bytes.len().min(buf.len() - *at);
                *bytes = buf[*at..*at + len].to_vec();
                if len == 0 { continue; }
            }
            let _ = model.apply(e);
        }
        let snapshot = model.state().to_vec();

        // Undo K, then redo K, must equal snapshot.
        let depth = model.undo_depth().min(20 + rng.range(20));
        for _ in 0..depth { model.undo(); }
        for _ in 0..depth { model.redo(); }
        assert_eq!(model.state(), snapshot.as_slice(),
            "trial {trial}: redo(undo(s)) must equal s");
    }
}

#[test]
fn redo_stack_clears_on_new_edit() {
    let mut model = UndoModel::new();
    model.apply(Edit::Insert { at: 0, bytes: b"AAA".to_vec() }).unwrap();
    model.apply(Edit::Insert { at: 3, bytes: b"BBB".to_vec() }).unwrap();
    model.commit_group();
    assert!(model.undo());  // undoes BBB
    assert_eq!(model.state(), b"AAA");
    assert_eq!(model.redo_depth(), 1);

    // New edit clears redo.
    model.apply(Edit::Insert { at: 3, bytes: b"CCC".to_vec() }).unwrap();
    assert_eq!(model.redo_depth(), 0);

    // Now redo() must return false (nothing to redo).
    assert!(!model.redo());
    assert_eq!(model.state(), b"AAACCC");
}

#[test]
fn bound_enforced_at_1000_with_fifo_eviction() {
    let mut model = UndoModel::new();
    for i in 0..1500u32 {
        let bytes = format!("{:04}", i).into_bytes();
        model.apply(Edit::Insert { at: model.state().len(), bytes }).unwrap();
        model.commit_group();  // ensure each is a distinct undo step
    }
    assert_eq!(model.undo_depth(), UNDO_LIMIT,
        "after 1500 distinct edits, undo depth must be exactly UNDO_LIMIT (1000)");
}

#[test]
fn coalescing_groups_adjacent_forward_inserts() {
    let mut model = UndoModel::new();
    // Type "Hello" character by character at advancing offsets.
    for (i, ch) in b"Hello".iter().enumerate() {
        model.apply(Edit::Insert { at: i, bytes: vec![*ch] }).unwrap();
    }
    // No commit_group between them — they should coalesce into ONE undo step.
    assert_eq!(model.undo_depth(), 1,
        "five adjacent inserts without commit_group must coalesce to 1 step");

    // Single undo reverses the entire word.
    assert!(model.undo());
    assert_eq!(model.state(), b"");

    // Single redo reapplies the entire word.
    assert!(model.redo());
    assert_eq!(model.state(), b"Hello");
}

#[test]
fn explicit_boundary_breaks_coalescing() {
    let mut model = UndoModel::new();
    model.apply(Edit::Insert { at: 0, bytes: b"Hello".to_vec() }).unwrap();
    model.commit_group();
    model.apply(Edit::Insert { at: 5, bytes: b" world".to_vec() }).unwrap();

}

#[test]
fn delete_breaks_coalescing() {
    let mut model = UndoModel::new();
    model.apply(Edit::Insert { at: 0, bytes: b"Hello".to_vec() }).unwrap();
    // No commit_group, but a delete must break the group.
