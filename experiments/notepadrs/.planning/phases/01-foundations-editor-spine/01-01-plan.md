## Plan Decisions

(See @./.claude/pan-wizard-core/references/handoff-decisions.md for the schema.)

### Locked (executor MUST follow)
- **D-1:** `[profile.release]` flags are exactly `opt-level = "z"`, `lto = "fat"`, `codegen-units = 1`, `strip = "symbols"`, `panic = "abort"`. No additions, no substitutions. Why: locked in context.md and verified in research.md (min-sized-rust HIGH-confidence). Source: `.planning/phases/01-foundations-editor-spine/01-context.md` "From research/summary.md" decisions.
- **D-2:** `+crt-static` is set via `.cargo/config.toml` `[target.x86_64-pc-windows-msvc] rustflags`. Why: BUILD-03 demands no third-party DLL dependencies; static CRT linkage is the textbook fix and avoids the vcruntime redistributable (~200-500KB cost, well under the 10MB ceiling). Source: research.md "Alternatives Considered" + min-sized-rust.
- **D-3:** `rust-toolchain.toml` pins `channel = "1.82.0"` (exact version, not `>=1.82`) to ensure CI determinism. Why: Phase-1 CI runs on `windows-latest` runner with whatever Rust version is convenient; pinning means a Rust 1.84 toolchain shift cannot regress us. Source: research.md "Toolchain installation" + BUILD-05.
- **D-4:** Application manifest (`notepadrs.manifest`) declares supported-OS GUIDs only for Windows 10 (`{8e0f7a12-bfb3-4fe8-b9a5-48fd50a15a9a}`) and Windows 11 — and `dpiAwareness` is `PerMonitorV2`. Why: BUILD-04 (Win10 1809+ / Win11 only) and pitfalls.md Pitfall 16 (HiDPI). Source: Microsoft Learn supportedOS docs.
- **D-5:** Cargo dependency on `windows = "0.62"` includes the **minimum** feature set listed in research.md ("Phase-1 `windows` crate features"). The executor may add **only** what is required for compilation success, and must justify each addition in the plan summary file. Why: every feature flag has a binary-size cost; speculative breadth blows the 10MB ceiling. Source: research.md Phase-1 features list.
- **D-6:** CI binary-size threshold is exactly 10485760 bytes (10 MiB, i.e. 10 × 1024 × 1024). Why: matches BUILD-02's "<10MB" specification (interpreted as MiB per industry convention; the 10MB ceiling is a binary-distribution metric, not a marketing one). Source: research.md Example 4.
- **D-7:** CI runs on `windows-latest` GitHub-hosted runner with `dtolnay/rust-toolchain@1.82.0` action. The matrix has exactly one entry (`x86_64-pc-windows-msvc`); no Linux/macOS cross-compile in CI. Why: project is Windows-only (out-of-scope per requirements.md). Source: research.md "Phase Requirements → Test Map".

### Open (executor's discretion within constraints)
- **O-1:** Specific manifest XML formatting and which embed-manifest mechanism to use (`embed-manifest` crate, `winres`, or hand-rolled `cc::Build` with rc.exe). Constraints: must end with the manifest embedded in the .exe such that `dpiAwareness` is `PerMonitorV2` and supported-OS includes Win10/11. Reason left open: multiple working approaches; `embed-manifest` crate is simplest but adds a build-dependency (no runtime cost). The executor picks based on what compiles cleanly with windows 0.62 + Rust 1.82.
- **O-2:** Whether to add a `[lib]` section to Cargo.toml or keep `notepadrs` as binary-only. Constraints: integration tests in `tests/` must work either way. Reason left open: integration tests for a binary crate are usually fine without an explicit lib target, but if the panic-safety test (Plan 01-02) needs to invoke `app::run()` directly, a lib target may be cleaner. For Plan 01-01 the simpler binary-only is the default; revisit in 01-02.
- **O-3:** Whether the CI workflow runs `cargo test` in `--debug` (default) or `--release`. Constraints: at least one `cargo test` invocation must run in default profile so the panic-safety integration test (which depends on `panic = "unwind"` being active) actually exercises catch_unwind. Reason left open: a release-profile test pass would catch some bugs but not the panic ones; a default-profile test pass is sufficient. Recommend default profile for `cargo test`.

### Considered and rejected
- **R-1:** Adding `[profile.release.package."*"]` overrides for sub-deps. Rejected because: Pitfall C in research.md notes these can backfire (some deps need `opt-level=3` for proc-macro correctness). Verify with `cargo bloat` if size creeps; don't preempt.
- **R-2:** Setting `panic = "abort"` for both `[profile.release]` and `[profile.dev]`. Rejected because: dev/test profiles need `unwind` for the panic-safety integration test (Plan 01-02) to actually catch a panic. Locking abort on dev would defeat QUAL-03's verifiability.
- **R-3:** Using `winapi` instead of `windows`. Rejected because: locked decision; `winapi` is frozen since 2021. Source: stack.md.
- **R-4:** Skipping the application manifest because Phase 1 doesn't strictly need DPI awareness. Rejected because: Pitfall 16 says retrofitting HiDPI later is harder than adding the manifest now; the manifest is one file that costs nothing.

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:

jobs:
  build-and-test:
    runs-on: windows-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Install Rust 1.82.0
        uses: dtolnay/rust-toolchain@master
        with:
          toolchain: 1.82.0
          targets: x86_64-pc-windows-msvc
          components: clippy,rustfmt

      - name: Cache cargo
        uses: actions/cache@v4
        with:
          path: |
            ~/.cargo/registry
            ~/.cargo/git
            target
          key: ${{ runner.os }}-cargo-${{ hashFiles('**/Cargo.lock') }}

      - name: Build (release)
        run: cargo build --release --target x86_64-pc-windows-msvc

      - name: Assert binary size <10MiB
        shell: pwsh
        run: |
          $exe = "target\x86_64-pc-windows-msvc\release\notepadrs.exe"
          if (-not (Test-Path $exe)) { Write-Error "Binary not found: $exe"; exit 1 }
          $size = (Get-Item $exe).Length
          $max = 10485760
          $sizeMB = [math]::Round($size / 1MB, 2)
          Write-Host "Binary size: $size bytes ($sizeMB MiB) — ceiling: 10 MiB"
          if ($size -gt $max) {
            Write-Error "FAIL: Binary $size bytes exceeds 10 MiB ceiling ($max bytes)"
            exit 1
          }

      - name: Assert no third-party DLL dependencies
        shell: pwsh
        run: |
          $exe = "target\x86_64-pc-windows-msvc\release\notepadrs.exe"
          # Locate dumpbin from any installed VS BuildTools instance on the runner
          $vswhere = "${env:ProgramFiles(x86)}\Microsoft Visual Studio\Installer\vswhere.exe"
          if (-not (Test-Path $vswhere)) { Write-Warning "vswhere not found; skipping DLL check"; exit 0 }
          $vs = & $vswhere -latest -property installationPath
          $dumpbin = Get-ChildItem -Path "$vs\VC\Tools\MSVC" -Filter dumpbin.exe -Recurse -ErrorAction SilentlyContinue | Select-Object -First 1
          if (-not $dumpbin) { Write-Warning "dumpbin not found; skipping DLL check"; exit 0 }
          $deps = & $dumpbin.FullName /dependents $exe | Select-String -Pattern "^\s+\S+\.dll\s*$" | ForEach-Object { $_.Matches.Value.Trim() }
          $allowed = @(
            'KERNEL32.dll', 'USER32.dll', 'GDI32.dll', 'ADVAPI32.dll', 'SHELL32.dll',
            'COMDLG32.dll', 'COMCTL32.dll', 'OLE32.dll', 'OLEAUT32.dll', 'msftedit.dll',
            'ntdll.dll', 'msvcrt.dll', 'WS2_32.dll', 'IMM32.dll', 'SHCORE.dll',
            'bcrypt.dll', 'api-ms-win-*.dll'
          )
          $bad = @()
          foreach ($d in $deps) {
            $allowMatch = $false
            foreach ($a in $allowed) { if ($d -like $a) { $allowMatch = $true; break } }
            if (-not $allowMatch) { $bad += $d }
          }
          if ($bad.Count -gt 0) {
            Write-Error "FAIL: Binary depends on non-Windows-shipped DLLs: $($bad -join ', ')"
            exit 1
          }
          Write-Host "DLL dependencies OK: $($deps -join ', ')"

      - name: Clippy (deny WndProc-relevant lints)
        run: cargo clippy --release --target x86_64-pc-windows-msvc -- -D warnings

      - name: Test
        run: cargo test --target x86_64-pc-windows-msvc
```

```rust
//! QUAL-01 verification: assert Cargo.toml [profile.release] contains the
//! five required flags.

use std::fs;

#[test]
fn release_profile_has_required_flags() {
    let cargo_toml = fs::read_to_string("Cargo.toml")
        .unwrap_or_else(|e| panic!("Cargo.toml unreadable: {e}"));

    // Find the [profile.release] section.
    let start = cargo_toml.find("[profile.release]")
        .expect("Cargo.toml missing [profile.release] section");
    let after = &cargo_toml[start..];
    // Section ends at the next [section] header or EOF.
    let end = after[1..].find('[').map(|i| i + 1).unwrap_or(after.len());
    let section = &after[..end];

    let required = [
        ("opt-level", "\"z\""),
        ("lto", "\"fat\""),
        ("codegen-units", "1"),
        ("strip", "\"symbols\""),
        ("panic", "\"abort\""),
    ];

    for (key, value) in required {
        let needle_a = format!("{key} = {value}");
        let needle_b = format!("{key}={value}");
        assert!(
            section.contains(&needle_a) || section.contains(&needle_b),
            "[profile.release] missing required flag: {key} = {value}\n\
             section content was:\n{section}"
        );
    }
}
```
