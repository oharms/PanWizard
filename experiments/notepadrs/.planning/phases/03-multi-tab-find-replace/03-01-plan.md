## Plan Decisions

(See @./.claude/pan-wizard-core/references/handoff-decisions.md for the schema.)

### Locked (executor MUST follow)

- D-1: **One RichEdit child window per tab.** `Tab` owns a fresh `EditorState` (and therefore a unique `hwnd_re`). On tab switch (Plan 03-02), visibility toggles via `ShowWindow(SW_HIDE/SW_SHOW)`. Why: Pitfall 11 (IME/focus state on tab switch) is decisively cleaner than single-RichEdit-with-buffer-swap; per-tab undo (SC-7, EM_SETUNDOLIMIT(1000)) comes for free. Source: research.md Pattern 1, pitfalls.md Pitfall 11.
- D-2: **Phase 2 EditorState fields stay where they are.** The struct shape (`hwnd_re`, `current_path`, `encoding`, `eol`, `original_bytes`, `had_trailing_newline`) is preserved verbatim — Plan 03-01 only WRAPS it inside `Tab`. No renames, no field removals, no field migrations to `Tab`. Why: avoids breaking Phase 2's open/save methods (`open_text_with_metadata`, `save_text_for_disk`, `save_text_for_disk_as`, `save_text`); avoids re-running the Phase 2 round-trip matrix. Source: research.md Pattern 1; src/editor.rs lines 18-91 (current EditorState shape).
- D-3: **Stable u64 id per tab.** `Tab.id: u64`, assigned monotonically from `App.next_tab_id`. Why: vector indices shift on close (`Vec::remove(idx)` shifts everything after `idx` down by one); the tab strip's per-item `lParam` carries the stable id so 03-02's `TCN_SELCHANGE` lookup is correct under reorder/close. Source: research.md Pattern 8 (Tab-ID stability).
- D-4: **Drop impl destroys hwnd_re.** `impl Drop for Tab { fn drop(&mut self) { unsafe { DestroyWindow(self.editor.hwnd_re); } } }`. Why: `Vec::remove(idx)` calls `Drop` on the popped element; without an explicit `DestroyWindow`, the RichEdit child window leaks GDI handles until the parent dies. Source: research.md Pitfall 2 (tab close orphaning RichEdit).
- D-5: **Pre-allocate ALL Phase 3 IDM_* and WM_APP_* constants in this plan.** `IDM_TAB_NEW=120, IDM_TAB_CLOSE=121, IDM_TAB_NEXT=122, IDM_TAB_PREV=123, IDM_FIND_OPEN=130, IDM_FIND_REPLACE_OPEN=131, IDM_FIND_NEXT=132, IDM_FIND_PREV=133, IDM_FIND_REPLACE_ONE=134, IDM_FIND_REPLACE_ALL=135, WM_APP_FIND_RESULT = WM_USER+10, WM_APP_FIND_DIALOG_CLOSED = WM_USER+11`. All use `#[allow(dead_code)]` until their plan wires them. Why: prevents Plans 03-02..03-05 from each having to add constants to `app.rs` and merge-conflicting on it; 03-04 (modeless dialog wiring) and 03-05 (worker wakeup) reference these by name from day one. Source: research.md "Plan Breakdown" §; research-emitted ID range 120-135 chosen to avoid collision with Phase 1/2 ranges 100-118.
- D-6: **Pre-allocate ALL Phase 3 accelerators in `build_accelerator_table()`.** Add Ctrl+T → IDM_TAB_NEW, Ctrl+W → IDM_TAB_CLOSE, Ctrl+Tab → IDM_TAB_NEXT, Ctrl+Shift+Tab → IDM_TAB_PREV, Ctrl+F → IDM_FIND_OPEN, Ctrl+H → IDM_FIND_REPLACE_OPEN, VK_F3 → IDM_FIND_NEXT, Shift+F3 → IDM_FIND_PREV. The dispatch handlers are stubs (`oslog("IDM_TAB_NEW: not yet wired (Plan 03-02)"); LRESULT(0)`) that 03-02 and 03-04 will replace. Why: same reason as D-5 — keep accelerator table edits to one plan. **Per orchestrator quality_gate: `SysTabControl32` + accelerator-table extension in `src/app.rs`, NOT subclassing the tab control.** Source: research.md Pattern 3.
- D-7: **Pre-allocate App fields for find/replace and tab strip:** `htabs: HWND` (default `HWND(std::ptr::null_mut())` until 03-02 creates it), `find_state: FindState` (a tiny struct stub: `pub struct FindState { pub pattern: String, pub replacement: String, pub flags: FindFlags }` with `Default` derived; `FindFlags` is also stubbed here as `pub struct FindFlags { pub case_sensitive: bool, pub whole_word: bool, pub regex: bool, pub wrap: bool, pub backward: bool }` with `Default`), `find_dlg_hwnd: Option<HWND>` (None until 03-04), `find_epoch: std::sync::atomic::AtomicU64` (initialized 0; 03-05 will use it), `find_tx: std::sync::mpsc::Sender<FindMsg>` and `find_rx: std::sync::mpsc::Receiver<FindMsg>` where `FindMsg` is also stubbed: `pub enum FindMsg { Hit { epoch: u64, start: usize, end: usize, wrapped: bool }, Done { epoch: u64, total: usize }, Error { epoch: u64, msg: String } }`, AND `find_pending: Option<(u64, String)>` (default `None`; 03-05 stashes the LF-text snapshot here while a worker is in flight so the UI can apply hits without re-streaming the buffer). The `mpsc::channel()` is created in `App::new()`. Why: 03-03 (find engine) is in its own file (no app.rs touch); 03-04 (find UI) and 03-05 (worker) can each replace stub implementations without each adding their own App fields; pre-allocating `find_pending` here means 03-05 D-13 only has to USE the field, not introduce it. Source: research.md Pattern 5 (worker), Pattern 4 (find engine I/O), §"App.rs additions in Phase 3".
- D-8: **`Tab::new_empty(parent: HWND, id: u64) -> Result<Self>` creates the per-tab RichEdit child via `EditorState::create(parent)` AND immediately `ShowWindow(hwnd_re, SW_HIDE)`** so Plan 03-02 controls visibility. **Exception:** the very first tab created in WM_CREATE stays visible (caller passes `visible: true`). Decide via a second arg: `Tab::new_empty(parent, id, initially_visible: bool)`. Why: matches Plan 03-02's hide/show contract; on Phase 3 launch the user sees one editable tab even before 03-02 wires the tab strip. Source: research.md Code Example 1 + Pattern 1 visibility semantics.
- D-9: **WM_SIZE iterates ALL tabs.** Replace `if let Some(ed) = app.editor.as_ref() { MoveWindow(ed.hwnd_re, ...) }` with `for tab in &app.tabs { let _ = MoveWindow(tab.editor.hwnd_re, 0, 0, cx, cy, true); }`. The vertical offset for the tab strip is added in 03-02 (here all tabs get the full client area; visually identical for the single-tab case). Why: Pitfall 8 in research.md — sizing must apply to all tabs so on tab switch the incoming RichEdit isn't sized stale. Source: research.md Pitfall 8.
- D-10: **`tab_close_decision(dirty: bool) -> CloseDecision` is the pure-logic test surface.** `CloseDecision { CanClose, PromptUser }`. The MessageBoxW call lives in 03-02's dispatch.rs handler; this function is testable in `cargo test --lib` without a window. Why: matches Phase 2's `dispatch_pure.rs` precedent — pure decision logic split from Win32 effects. Source: research.md §"Tab UI" + Phase 2 dispatch_pure pattern.
- D-11: **Update lib.rs to export `tab` module** (matching the binary's `mod tab;` in main.rs). Phase 2 had to do this for `encoding`/`eol`; Phase 3 needs it for `tab` because `tests/tab_model.rs` will `use notepadrs::tab::tab_close_decision`. Source: src/lib.rs (read it; current pub mod list does NOT include `tab`).

### Open (executor's discretion within constraints)

- O-1: **Where the Phase 3 IDM stubs live in dispatch.rs's `WM_COMMAND match`.** Adjacent to existing IDM_FILE_* and IDM_ENCODING_* arms is fine; group them as a new `// === Phase 3 stubs (wired in 03-02/03-04) ===` block. Constraint: stubs MUST log via `oslog` (not panic, not no-op) so 03-02/03-04 can verify their own wiring is replacing the stub.
- O-2: **`FindState`/`FindFlags`/`FindMsg` placement.** Either (a) inside `app.rs` with `#[allow(dead_code)]` until 03-03/03-04/03-05 use them, or (b) inside an empty stub `src/find.rs` (3-4 lines) that 03-03 then expands. Recommendation: (a) — keeps `src/find.rs` an exclusive 03-03 file (Wave-2 file ownership). Constraint: types must be `Default + Clone + Copy`-able (FindFlags) and `Default`-able (FindState).
- O-3: **Whether to expose `Tab::new_empty` or `App::push_empty_tab(&mut self) -> Result<&mut Tab>` for the WM_CREATE call site.** Either is fine; the latter centralizes id-assignment via `App.next_tab_id` and is friendlier for 03-02's IDM_TAB_NEW handler. Recommendation: provide BOTH — `Tab::new_empty` is the low-level constructor, `App::push_empty_tab` calls it and assigns the id.

### Considered and rejected

- R-1: **Single RichEdit with buffer swap per tab switch.** Rejected: Pitfall 11 (committing IME compositions and saving/restoring cursor+scroll+undo on every switch) plus EM_STREAMIN cost on every switch (multi-ms for 1MB) plus losing per-tab undo guarantees. The "saves working set" argument doesn't outweigh the correctness risk.
- R-2: **Storing `EditorState` directly in `Vec<EditorState>` without wrapping in `Tab`.** Rejected: `title`, `dirty`, `id`, `word_wrap` are tab-strip concerns, not editor-state concerns; conflating them muddies the Phase 2 boundaries (e.g., `EditorState::open_text_with_metadata` shouldn't have to know about tab titles).
- R-3: **Splitting Phase 3 IDM_* and accelerators across the plans that use them.** Rejected: every plan would have to edit `app.rs` to add a constant or accelerator, which serializes Wave 2/3 unnecessarily. Pre-allocating in 03-01 is a single-file edit and keeps later plans focused.
- R-4: **Allocating `find_state` etc. lazily on first Find dialog open.** Rejected: `Default` derives make pre-allocation free; lazy allocation requires `Option<FindState>` everywhere and adds a None-check at every call site.

```rust
//! Tab — wraps Phase 2's EditorState with multi-tab metadata (title, stable id,
//! dirty flag, word-wrap placeholder). Each Tab owns its own RichEdit child
//! window (Pattern 1 from research.md): tab switch toggles visibility, never
//! restreams the buffer.

#![deny(clippy::unwrap_used)]
#![deny(clippy::expect_used)]

use windows::Win32::Foundation::HWND;
use windows::Win32::UI::WindowsAndMessaging::{DestroyWindow, ShowWindow, SW_HIDE};

use crate::editor::EditorState;
use crate::error::Result;

/// One open buffer/document. Phase 2's EditorState carries the buffer/encoding/EOL
/// data; Tab adds the multi-tab presentation layer (title, dirty marker, stable id,
/// word-wrap setting placeholder for Phase 5).
pub struct Tab {
    pub editor: EditorState,
    pub title: String,
    /// Stable across reorder/close. Tab strip carries this in TCITEM.lParam (Plan 03-02).
    pub id: u64,
    /// True when the buffer has unsaved edits. Updated by EN_CHANGE in 03-02; cleared on save.
    pub dirty: bool,
    /// Per-tab word-wrap setting (SC-8 / VIEW-06). Phase 5 wires this; Phase 3 reserves the field
    /// so the per-tab shape is in place — avoids retrofitting later (research.md Open Question 4).
    pub word_wrap: bool,
}

impl Tab {
    /// Create a fresh empty tab. Spawns a per-tab RichEdit child via `EditorState::create`.
    /// If `initially_visible` is false, the new RichEdit is hidden immediately so 03-02 can
    /// control visibility via TCN_SELCHANGE; on first launch (WM_CREATE) the caller passes true.
    pub unsafe fn new_empty(parent: HWND, id: u64, initially_visible: bool) -> Result<Self> {
        let editor = EditorState::create(parent)?;
        if !initially_visible {
            let _ = ShowWindow(editor.hwnd_re, SW_HIDE);
        }
        Ok(Self {
            editor,
            title: "Untitled".to_owned(),
            id,
            dirty: false,
            word_wrap: false,
        })
    }
}

impl Drop for Tab {
    fn drop(&mut self) {
        // Pitfall 2 (research.md): explicit DestroyWindow on tab close.
        // Without this, Vec::remove(idx) drops Tab without releasing the OS HWND,
        // leaking the RichEdit child until the parent window dies.
        unsafe {
            let _ = DestroyWindow(self.editor.hwnd_re);
        }
    }
}

/// Pure-logic close decision (testable headlessly).
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum CloseDecision {
    /// Buffer has no unsaved edits → close immediately.
    CanClose,
    /// Buffer has unsaved edits → caller must show MessageBoxW with Save/Discard/Cancel.
    PromptUser,
}

/// Decide whether closing a tab requires a user prompt.
/// (The MessageBoxW call lives in 03-02's dispatch handler; this function is the
/// test-friendly pure shape — same split pattern as Phase 2's dispatch_pure.rs.)
pub fn tab_close_decision(dirty: bool) -> CloseDecision {
    if dirty {
        CloseDecision::PromptUser
    } else {
        CloseDecision::CanClose
    }
}
```
