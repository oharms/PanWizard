## Plan Decisions

(See @./.claude/pan-wizard-core/references/handoff-decisions.md for the schema.)

### Locked (executor MUST follow)

- D-1: **`SysTabControl32` with `TCS_OWNERDRAWFIXED`.** Style: `WS_CHILD | WS_VISIBLE | TCS_OWNERDRAWFIXED`. Class: `WC_TABCONTROL` (which is `"SysTabControl32"`). Why: per the orchestrator quality_gate ("`SysTabControl32` + accelerator-table extension in `src/app.rs` (NOT subclassing the tab control)"). Owner-draw is required so we can paint the dirty asterisk; a fully custom-painted tab bar would mean reimplementing hit-testing, theme integration, scroll-on-overflow, and keyboard nav — explicitly out of scope. Source: research.md Pattern 2; stack.md "Critical Decision: Tab UI"; orchestrator quality_gate.
- D-2: **One MessageBoxW with `MB_YESNOCANCEL | MB_ICONQUESTION`** for the dirty-close prompt. Body text spells out the mapping ("Yes = Save and close / No = Discard changes and close / Cancel = Keep tab open"). Yes → invoke `do_file_save(app, hwnd)` then close; No → close; Cancel/anything else → abort the close. Why: TaskDialog with re-labelable buttons would be cleaner but adds COMCTL32 v6 manifest plumbing — accepted v1.x polish (research.md Pattern 6, Open Question 2). Source: research.md Pattern 6, pitfalls.md Pitfall 13.
- D-3: **Last-tab-close keeps one Untitled tab.** When `IDM_TAB_CLOSE` removes the only remaining tab, immediately push a fresh empty tab. Quitting requires Alt+F4 / File→Exit. Why: Notepad++ behavior; matches the user's mental model of "the editor always has at least one tab". Avoids the awkward state of an editor with zero tabs. Source: research.md TAB-02 row.
- D-4: **Tab switch increments `app.find_epoch`** via `fetch_add(1, Release)`. Even though the worker thread isn't wired until 03-05, this discipline is established here so 03-05 doesn't have to retrofit it. Why: Pitfall 9 (research.md) — async find result arrives after tab switch must be discarded. Source: research.md Pitfall 9.
- D-5: **EN_CHANGE → `tab.dirty = true` + `InvalidateRect(htabs, &tab_rect, FALSE)`** for just that tab. Save success → `tab.dirty = false` + same partial invalidate. **Avoid full-tabstrip InvalidateRect**. Why: Pattern 7 dirty-rect repaint discipline (research.md Pattern 7) — repainting all 30 tabs on every keystroke when only one changed is wasteful and may flicker. Source: research.md Pattern 7, Pattern 2 (Tab Strip Owner-Draw EN_CHANGE handling).
- D-6: **`EN_CHANGE` is delivered via the parent's `WM_COMMAND` with `HIWORD(wparam) == EN_CHANGE` (which is `0x0300`)** — RichEdit by default does NOT send EN_CHANGE; we must enable it via `EM_SETEVENTMASK` with `ENM_CHANGE` bit set in `EditorState::create`. **Action:** Plan 03-02 must MODIFY `EditorState::create` to add this. Hand-write the exact line: `let _ = SendMessageW(hwnd_re, EM_SETEVENTMASK, Some(WPARAM(0)), Some(LPARAM(ENM_CHANGE.0 as isize)));`. Why: without this, the dirty asterisk never lights up because the notification never fires. Source: Microsoft Learn EM_SETEVENTMASK + ENM_CHANGE (HIGH); cross-checked with research.md TAB-06 row.
  - **Note on the boundary:** this is a one-line edit to `src/editor.rs`, declared in this plan's `files_modified`. Plan 03-03 (also Wave 2) does NOT touch editor.rs (its file ownership is `src/find.rs` + `tests/find_engine.rs` + `tests/replace_engine.rs`). Later sequential plans 03-04 (find dialog UI) and 03-05 (worker thread) also do NOT touch editor.rs — their `files_modified` lists are 03-04: `src/find_ui.rs` + `src/dispatch.rs` + `src/main.rs` + `src/lib.rs` + `src/menu.rs` + `src/app.rs`; 03-05: `src/find_worker.rs` + `src/dispatch.rs` + `src/main.rs` + `src/lib.rs` + `src/app.rs` + tests. So this one-line ENM_CHANGE edit has no conflict surface in the entire phase.
- D-7: **Tab strip height is exactly 28 pixels** at 96 DPI; the per-tab RichEdit's y-origin becomes 28 (was 0 in Plan 03-01). All `MoveWindow` calls in WM_SIZE shift accordingly. **High-DPI note:** for v1, do not scale the 28px — the tab control itself respects DPI via owner-draw. If 4K usability becomes a complaint, revisit in v1.x. Why: Notepad++ tab strip is roughly 24-30px; matches user expectation. Source: research.md Pattern 2.
- D-8: **Owner-draw paint hands the title via DrawTextW into `dis.hDC`** within `dis.rcItem`. The string is `"* {title}"` if `tab.dirty`, else `tab.title`. Use `SetTextColor`/`SetBkMode(TRANSPARENT)` for selected vs unselected (selected = `COLOR_WINDOWTEXT`, unselected = `COLOR_GRAYTEXT`). Honor `dis.itemState & ODS_SELECTED` for the selected highlight. Why: minimum sufficient visual; matches Notepad++ default theme adequately for v1. Source: research.md Code Example owner-draw block.
- D-9: **Stable id resolution at TCN_SELCHANGE.** Read the new selected index via `TCM_GETCURSEL`; look up `lParam` (the tab id) via `TCM_GETITEMW(TCIF_PARAM)`; find the `Tab` whose `id == lParam` to get the index. Why: Plan 03-01 D-3 — vector indices shift on close so the `lParam` id is the truth, not the index. Source: research.md Pattern 8.
- D-10: **`WM_NOTIFY` dispatch for TCN_SELCHANGE filters by `nm.hwndFrom == app.htabs`.** A future control adding NMHDR-style notifications must not stomp the tab handler. Why: defensive; matches Phase 2's WM_INITMENUPOPUP filtering. Source: standard Win32 idiom.
- D-11: **Switch tab order:** outgoing first hides, then incoming shows, then `SetFocus(incoming.editor.hwnd_re)`, then `app.active_tab = new_idx`, then `app.find_epoch.fetch_add(1, Release)`, then `update_window_title(hwnd, &app.tabs[new_idx])`. Why: Pitfall 11 (focus must follow visibility) + D-4 (epoch must bump on switch). Source: research.md Pitfall 11, Code Example 2.
- D-12: **Window title format: `"notepadrs - {tab.title}"` for clean tabs, `"notepadrs - * {tab.title}"` for dirty.** Reuse the same dirty-prefix string the tab strip uses to avoid drift. Why: matches Notepad++ window title convention. Source: pitfalls.md Pitfall 13 row 1 ("Dirty-tab asterisk renders identically — make it bold or distinct"). For v1 the tabstrip leading `* ` is sufficient.
- D-13: **`do_file_open` updates the active tab's title** to `path.file_name().to_string_lossy().to_string()` (or `"Untitled"` if no path) and invalidates the active tab's strip rect. Same for `do_file_save_as`. **`do_file_save` does NOT change the title** (path didn't change). Why: tab title must reflect the file the buffer represents. Source: research.md TAB-06.
- D-14: **`do_file_open` opens the file into the ACTIVE tab if it is unmodified and untitled** (i.e., a fresh `IDM_TAB_NEW` tab); otherwise opens into a new tab. Why: matches Notepad++/VS Code behavior — clicking File→Open on a blank tab loads into it; on an edited tab it opens a new tab. Source: research.md (implicit in Notepad++ behavior); Phase 2 currently always loads into the single tab (no other choice).
  - **Heuristic:** "fresh empty tab" = `tab.editor.current_path.is_none() && !tab.dirty && tab.editor.original_bytes.is_empty()`. Otherwise push a new tab via `app.push_empty_tab(hwnd, false)` then `TCM_INSERTITEM` and `TCM_SETCURSEL` so it becomes active.
  - **Note:** the Phase 2 menu_action_tests.rs may exercise the in-place open behavior. If those tests fail post-refactor, the heuristic still loads into the same tab when starting from an empty Untitled — they should pass. If a test specifically asserts "always loads into active tab even when it's dirty", it must be updated to match the new contract; document the change in the summary.
- D-15: **WM_SIZE refines beyond 03-01:** the tab strip is sized to `(0, 0, cx, 28)` and each tab's RichEdit is sized to `(0, 28, cx, cy - 28)`. Why: implements D-7 layout. Source: research.md.
- D-16: **`prompt_close_dirty` is a thin Win32 wrapper** that returns `CloseAction { Save, Discard, Cancel }`. The pure-logic decision (`tab_close_decision` from 03-01) is consumed BEFORE the prompt; the prompt itself is a side-effecting helper. Why: keeps the test surface clean — `tab_close_decision` is fully tested; the MessageBoxW shell is tested manually. Source: research.md Pattern 6 split.

### Open (executor's discretion within constraints)

- O-1: **Where to place `build_tab_strip`** — either inside `src/tab.rs` or inside `src/dispatch.rs` near WM_CREATE. Recommendation: `src/tab.rs` — keeps tab presentation in one place and matches how Phase 2's `build_main_menu` lives in `src/menu.rs`. Constraint: must be `pub unsafe fn build_tab_strip(parent: HWND) -> Result<HWND>` returning the strip HWND for App to store.
- O-2: **Whether to subclass the tab control to handle Ctrl+Tab inside the strip itself.** Per orchestrator quality_gate: **NO** — accelerator-table is the canonical mechanism. Subclassing is rejected. So this isn't really an open decision, just a check.
- O-3: **`update_window_title` location.** Could live in `src/window.rs`, `src/dispatch.rs`, or `src/tab.rs`. Recommendation: `src/dispatch.rs` (private helper) since it's only called from dispatch handlers. Constraint: must use `SetWindowTextW(hwnd, ...)` (NOT `SetWindowTextA` — Phase 1 discipline).
- O-4: **Tab strip TCITEM text length.** Path basenames can exceed 256 chars; `SysTabControl32` truncates with ellipsis but a 1024-char path doesn't display nicely. Recommendation: truncate `tab.title` to `path.file_name()` (basename only) at title-set time; full path stays in `tab.editor.current_path`. The owner-draw paint truncates further if `dis.rcItem` is too narrow (DrawTextW with `DT_END_ELLIPSIS`). Constraint: NEVER pass the full path as the tab title.

### Considered and rejected

- R-1: **Custom-painted tab bar (no `SysTabControl32`).** Rejected by orchestrator quality_gate. Also: 500+ LOC for parity with built-in features (theme integration, hit-testing, scroll-on-overflow, keyboard nav).
- R-2: **TaskDialog instead of MessageBoxW** for the dirty-close prompt. Rejected for v1: requires COMCTL32 v6 manifest assertion + new Cargo features. Polish-phase swap (Plan 5 territory). The MessageBoxW prompt body explicitly maps Yes/No/Cancel → Save/Discard/Cancel.
- R-3: **Closing the last tab quits the application.** Rejected: Notepad++ keeps one Untitled tab open; matches user mental model. Quit-on-last-close is a setting some editors offer, but adds a config decision out of scope for v1.
- R-4: **Per-tab file-modified-on-disk detection** (Pitfall 9 from project pitfalls.md, not the Phase 3 Pitfall 9). Rejected for Phase 3: needs `ReadDirectoryChangesW` + worker thread. Tracked as v1.x feature; not in Phase 3 reqs.

```rust
// Append to src/tab.rs (after existing Tab/Drop/CloseDecision items):

use windows::core::*;
use windows::Win32::Foundation::{HWND, LPARAM, WPARAM, RECT};
use windows::Win32::UI::Controls::*;
use windows::Win32::UI::WindowsAndMessaging::*;

/// Tab strip height in pixels (96 DPI baseline). RichEdit children sit below this.
pub const TAB_STRIP_HEIGHT: i32 = 28;

/// Create the SysTabControl32 strip, owner-drawn for the dirty-asterisk paint.
/// Caller stores the returned HWND on App.htabs and inserts items via insert_tab_item.
pub unsafe fn build_tab_strip(parent: HWND) -> crate::error::Result<HWND> {
    use windows::Win32::System::LibraryLoader::GetModuleHandleW;
    let hinstance = GetModuleHandleW(None)?.into();
    let style = WS_CHILD.0 | WS_VISIBLE.0
        | TCS_OWNERDRAWFIXED as u32
        | TCS_FOCUSNEVER as u32;            // tab control never grabs Tab-key focus
    let htabs = CreateWindowExW(
        WINDOW_EX_STYLE(0),
        WC_TABCONTROL,
        w!(""),
        WINDOW_STYLE(style),
        0, 0, 100, TAB_STRIP_HEIGHT,        // resized in WM_SIZE
        Some(parent),
        None,
        Some(hinstance),
        None,
    )?;
    Ok(htabs)
}

/// Insert a TCITEM for the given Tab. The tab.id is stored in lParam (Pattern 8 — stable id).
pub unsafe fn insert_tab_item(htabs: HWND, idx: i32, tab: &Tab) -> crate::error::Result<()> {
    let display = format_tab_label(tab);
    let mut text_utf16: Vec<u16> = display.encode_utf16().chain(std::iter::once(0)).collect();
    let mut item: TCITEMW = std::mem::zeroed();
    item.mask = TCIF_TEXT | TCIF_PARAM;
    item.pszText = PWSTR(text_utf16.as_mut_ptr());
    item.cchTextMax = text_utf16.len() as i32;
    item.lParam = LPARAM(tab.id as isize);
    let r = SendMessageW(
        htabs,
        TCM_INSERTITEMW,
        Some(WPARAM(idx as usize)),
        Some(LPARAM(&mut item as *mut _ as isize)),
    );
    if r.0 < 0 {
        return Err(crate::error::Error::Misc(format!("TCM_INSERTITEMW failed (idx={idx})")));
    }
    Ok(())
}

/// Compute the tab strip rect for a given tab index (used for partial-invalidate on dirty change).
pub unsafe fn tab_rect(htabs: HWND, idx: i32) -> Option<RECT> {
    let mut rc = RECT::default();
    let r = SendMessageW(
        htabs,
        TCM_GETITEMRECT,
        Some(WPARAM(idx as usize)),
        Some(LPARAM(&mut rc as *mut _ as isize)),
    );
    if r.0 != 0 { Some(rc) } else { None }
}

/// Format the tab strip label: "* {title}" if dirty, else "{title}". Same string is used in
/// the window title (D-12), so dirty-prefix discipline lives in one place.
pub fn format_tab_label(tab: &Tab) -> String {
    if tab.dirty {
        format!("* {}", tab.title)
    } else {
        tab.title.clone()
    }
}

/// Side-effecting helper: prompt the user for save/discard/cancel on a dirty tab close.
/// Pure decision logic is in `tab_close_decision`; this is the Win32 shell.
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum CloseAction {
    Save,
    Discard,
    Cancel,
}

pub unsafe fn prompt_close_dirty(parent: HWND, tab: &Tab) -> CloseAction {
    let title_w: Vec<u16> = format!("notepadrs - Save changes?\0").encode_utf16().collect();
    let body = format!(
        "The file '{}' has unsaved changes.\n\n\
         Yes  = Save and close\n\
         No   = Discard changes and close\n\
         Cancel = Keep tab open\0",
        tab.title
    );
    let body_w: Vec<u16> = body.encode_utf16().collect();
    let result = MessageBoxW(
        Some(parent),
        PCWSTR(body_w.as_ptr()),
        PCWSTR(title_w.as_ptr()),
        MB_YESNOCANCEL | MB_ICONQUESTION,
    );
    match result.0 {
        x if x == IDYES.0 => CloseAction::Save,
        x if x == IDNO.0  => CloseAction::Discard,
        _                 => CloseAction::Cancel,
    }
}
```

unsafe fn switch_active_tab(app: &mut App, hwnd: HWND, new_idx: usize) {
    use std::sync::atomic::Ordering;
    if new_idx >= app.tabs.len() || new_idx == app.active_tab {
        return;
    }
    let outgoing = app.tabs[app.active_tab].editor.hwnd_re;
    let incoming = app.tabs[new_idx].editor.hwnd_re;
    let _ = ShowWindow(outgoing, SW_HIDE);
    let _ = ShowWindow(incoming, SW_SHOW);
    let _ = SetFocus(Some(incoming));
    app.active_tab = new_idx;
    app.find_epoch.fetch_add(1, Ordering::Release);
    update_window_title(hwnd, &app.tabs[new_idx]);
}

unsafe fn close_tab_at(app: &mut App, hwnd: HWND, idx: usize) {
    use std::sync::atomic::Ordering;
    if idx >= app.tabs.len() { return; }
    // Remove from tab strip first.
    let _ = SendMessageW(app.htabs, TCM_DELETEITEM, Some(WPARAM(idx)), Some(LPARAM(0)));
    // Drop the Tab — its Drop impl calls DestroyWindow on the RichEdit child.
    app.tabs.remove(idx);
    app.find_epoch.fetch_add(1, Ordering::Release);
    // D-3: if no tabs remain, push a fresh empty Untitled tab.
    if app.tabs.is_empty() {
        if let Err(err) = app.push_empty_tab(hwnd, true) {
            oslog(&format!("close_tab_at: push_empty_tab failed: {err}"));
            return;
        }
        if let Some(t) = app.tabs.first() {
            let _ = crate::tab::insert_tab_item(app.htabs, 0, t);
        }
        let _ = SendMessageW(app.htabs, TCM_SETCURSEL, Some(WPARAM(0)), Some(LPARAM(0)));
        app.active_tab = 0;
        update_window_title(hwnd, &app.tabs[0]);
        return;
    }
    // Otherwise: pick a sane new active_tab.
    let new_active = idx.min(app.tabs.len() - 1);
    let _ = SendMessageW(app.htabs, TCM_SETCURSEL, Some(WPARAM(new_active)), Some(LPARAM(0)));
    let _ = ShowWindow(app.tabs[new_active].editor.hwnd_re, SW_SHOW);
    let _ = SetFocus(Some(app.tabs[new_active].editor.hwnd_re));
    app.active_tab = new_active;
    update_window_title(hwnd, &app.tabs[new_active]);
}

unsafe fn update_window_title(hwnd: HWND, tab: &crate::tab::Tab) {
    let label = crate::tab::format_tab_label(tab);
    let title: Vec<u16> = format!("notepadrs - {}\0", label).encode_utf16().collect();
    let _ = SetWindowTextW(hwnd, PCWSTR(title.as_ptr()));
}

unsafe fn invalidate_layout(app: &mut App, hwnd: HWND) -> windows::core::Result<()> {
    let mut rc = RECT::default();
    let _ = GetClientRect(hwnd, &mut rc);
    let cx = rc.right - rc.left;
    let cy = rc.bottom - rc.top;
    let strip_h = crate::tab::TAB_STRIP_HEIGHT;
    let editor_h = (cy - strip_h).max(0);
    for tab in &app.tabs {
        let _ = MoveWindow(tab.editor.hwnd_re, 0, strip_h, cx, editor_h, true);
    }
    Ok(())
}
