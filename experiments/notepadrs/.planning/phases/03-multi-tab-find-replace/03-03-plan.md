### Locked (executor MUST follow)

- D-1: **`regex = "1.12"` is the ONLY new dependency.** Already on the v1 allowlist. NO `fancy-regex`, NO `regex-lite`, NO `regex::bytes`. Why: `fancy-regex` is a backtracking engine (Pitfall 5 — UI freeze hazard); `regex-lite` lacks `(?i)` and `\b` Unicode features needed for FIND-04/05; `regex::bytes` is for `&[u8]` matching but RichEdit gives us UTF-8 via `EM_STREAMOUT` so `&str` is the natural shape. Source: research.md "Standard Stack" + "Don't Add" tables.
- D-2: **`FindFlags` is REUSED from `src/app.rs` (Plan 03-01)** — DO NOT redefine it in `src/find.rs`. Re-export from `crate::app::FindFlags` so the engine and the dialog share one canonical type. Why: 03-04 reads/writes the same struct from `App.find_state.flags`; redefining would force a conversion at every call. Source: research.md "Pattern 4" — flags is a single shared shape.
- D-3: **`find_next` returns UTF-8 BYTE OFFSETS** in `FindHit { start, end, wrapped }`. The conversion to UTF-16 code units (RichEdit selection currency) lives in two helpers in `find.rs`: `utf8_to_utf16_units(text, byte_offset) -> usize` and `utf16_to_utf8_offset(text, cu_offset) -> usize`. Why: keeps the engine pure (no Win32 dependency) and makes the conversion testable. Plan 03-04 calls these helpers when crossing the RichEdit boundary. Source: research.md Pattern 4 + Pattern 7.
- D-4: **Empty pattern → `Err(CompileError::Empty)`.** Empty-match patterns (`.*`, `^$`, `\b`, etc.) → `Err(CompileError::EmptyMatch)`. Detection: after `RegexBuilder::build`, call `regex.is_match("")` — if true, fail the compile. Why: Pitfall 3 in 03-research (empty-match regex causes Replace All infinite loop). Source: research.md Pitfall 3.
- D-5: **Whole-word in literal mode** = `\b{escaped}\b` after `regex::escape`. Whole-word in regex mode → flag IGNORED (no automatic wrapping; user composes their own boundaries). Why: regex-mode patterns may already contain alternation/repetition that boundary wrapping would break (`\b(foo|bar)\b` is the user's call, not ours); Notepad++ greys out the toggle in regex mode. The engine ignores; the UI plan greys. Source: research.md FIND-05.
- D-6: **Case-insensitive** = `RegexBuilder::case_insensitive(!flags.case_sensitive)`. NEVER prepend `(?i)` to the pattern string — RegexBuilder does this correctly across regex and literal modes. Why: `(?i)` inside an escaped literal would be escaped further (`\(\?i\)` would match the literal `(?i)`). Source: regex crate docs; verified at docs.rs/regex/1.12.3/regex/struct.RegexBuilder.html.
- D-7: **`size_limit(10 * 1024 * 1024)` cap on every compile.** Why: Pitfall 5 — pathological pattern compile-time DoS. 10 MB is the regex crate default but make it explicit so future maintainers see the budget. Source: research.md Pitfall 5; regex crate RegexBuilder::size_limit docs.
- D-8: **Forward find from `cursor`:** `regex.find(&text[cursor..])` returns `Match` with offsets relative to the slice; add `cursor` to get absolute offset. If None and `flags.wrap`: `regex.find(&text[..cursor.min(text.len())])` returns the wrapped match (offsets already absolute). Why: research.md "find_next forward branch" code example.
- D-9: **Backward find from `cursor`:** `regex.find_iter(&text[..cursor.min(text.len())]).last()` returns the last match strictly before cursor. If None and `flags.wrap`: `regex.find_iter(text).last()` returns the last overall match (mark wrapped=true). **Special case:** if `cursor == 0`, the slice is empty so first call returns None — the wrap path runs immediately and returns the buffer's last match. Why: Pitfall 4 (research.md) — backward at offset 0 wraps; without this special handling we'd return None even with wrap enabled. Source: research.md Pattern 4 backward branch + Pitfall 4.
- D-10: **`replace_all(text, regex, replacement) -> (String, usize)`** uses `regex.replace_all(text, replacement).into_owned()` for the new text, and a separate `regex.find_iter(text).count()` for the count. Why: `Captures::expand` is invoked automatically by `replace_all` when `replacement` contains `$0`/`$1`/`${name}` — research.md confirms this is the standard idiom. Counting separately is O(2n) but trivial; the alternative (writing a manual replace loop with a counter) duplicates the regex crate's logic. Source: regex crate Regex::replace_all docs (HIGH).
- D-11: **`utf8_to_utf16_units(text: &str, byte_offset: usize) -> usize` = `text[..byte_offset.min(text.len())].encode_utf16().count()`.** **`utf16_to_utf8_offset(text: &str, cu_offset: usize) -> usize` = walks `text.char_indices()` while accumulating utf16 code units; returns the byte offset where the accumulated count first reaches `cu_offset`.** Both clamp on overflow (`.min(text.len())` / saturating fallback to `text.len()`). Why: RichEdit's `EM_EXSETSEL`/`EM_GETSEL` use UTF-16 code units; engine offsets are UTF-8 bytes; conversion at the boundary is the single foot-gun called out in research.md. Source: research.md Pattern 4 + Pattern 7.
- D-12: **`FindHit.wrapped: bool` indicates whether wrap-around fired** for THIS find call (not a permanent state). Plan 03-04 uses this to show "Wrapped to top" / "Wrapped to bottom" in the status line. Source: research.md FIND-09.
- D-13: **No I/O in `src/find.rs`.** No `std::fs::*`, no `windows::*`, no `unsafe`, no `oslog` calls. The module is `#![deny(unsafe_code)]` and `#![deny(clippy::unwrap_used)]` and `#![deny(clippy::expect_used)]`. Why: pure-logic boundary; `cargo test` runs every test in microseconds without OS access. Source: research.md "Pure-logic engine boundary".
- D-14: **Add `pub mod find;` to `src/main.rs` and `pub mod find;` to `src/lib.rs`** so both the binary and integration tests can use `notepadrs::find::*`. Mirror Phase 2's encoding/eol module exposure. Source: src/lib.rs current state.

### Open (executor's discretion within constraints)

- O-1: **Whether `compile` returns `Result<Regex, CompileError>` or `Result<Regex, regex::Error>`.** Recommendation: custom `CompileError` enum with variants `Empty`, `EmptyMatch`, `Regex(regex::Error)`. Why: 03-04 needs to differentiate "user typed empty pattern" (silent no-op) from "user typed a malformed regex" (red border + status message); a flat `regex::Error` can't express that distinction. Constraint: `CompileError` must implement `Debug` (for tests) and have a `Display` impl (for 03-04's status line).
- O-2: **Whether `FindHit` carries the matched text.** Recommendation: NO — just `start`, `end`, `wrapped`. The caller (03-04) can slice `text[start..end]` if it wants the match. Why: avoids cloning bytes; keeps `FindHit` `Copy`-able. Constraint: `FindHit` is `#[derive(Debug, Clone, Copy, PartialEq, Eq)]`.
- O-3: **Helper for capture-group expansion preview** (e.g., `regex.captures(text).map(|c| { let mut out = String::new(); c.expand(replacement, &mut out); out })` for showing "this match → that replacement" in a future preview pane). Recommendation: NOT in this plan — it's not in the 18 reqs. Add only if 03-04 needs it.
- O-4: **Whether to expose `find_iter_all(text, regex) -> impl Iterator<Item=FindHit>`** as a public API for "highlight all matches" UI. Recommendation: NO — not in 18 reqs (FIND-08 is "Replace All" which is its own function; "highlight all" is v1.x). Add later. Constraint: don't pre-build features.

### Considered and rejected

- R-1: **`fancy-regex` for lookbehind/atomic-groups.** Rejected: backtracking engine, Pitfall 5 hazard, not on allowlist.
- R-2: **`regex::bytes::Regex`.** Rejected: RichEdit hands us UTF-8 strings; matching on `&str` is natural and avoids byte-vs-char-offset confusion.
- R-3: **Custom NFA engine.** Rejected: `regex` crate is on the allowlist and is a 600KB Pike-NFA. Reinventing it is multi-week work for zero user-visible benefit.
- R-4: **Storing `Regex` instances in `App.find_state` for caching.** Rejected: `Regex` is not `Default + Clone + Copy` (it's heap-allocated), and recompiling on every Find-Next is fine for v1 (compile is fast for sane patterns; size_limit caps pathological cases). Plan 03-04 caches the last compiled regex via `Option<Regex>` if profiling demands it.

//! Pure-logic find/replace engine. No Win32, no I/O, no unsafe.
//! All match offsets are UTF-8 byte offsets; conversion to UTF-16 code units (RichEdit's
//! selection currency) lives in `utf8_to_utf16_units` / `utf16_to_utf8_offset` helpers.

#![deny(unsafe_code)]
#![deny(clippy::unwrap_used)]
#![deny(clippy::expect_used)]

use regex::{Regex, RegexBuilder};

pub use crate::app::FindFlags;

/// Errors produced by `compile`. Distinguishes empty-pattern, empty-match, and regex-syntax
/// errors so the UI plan (03-04) can show appropriate feedback (silent no-op vs red border).
#[derive(Debug)]
pub enum CompileError {
    Empty,
    EmptyMatch,
    Regex(regex::Error),
}

impl std::fmt::Display for CompileError {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
        match self {
            CompileError::Empty => write!(f, "Find pattern is empty"),
            CompileError::EmptyMatch => write!(f, "Pattern matches the empty string (would loop)"),
            CompileError::Regex(e) => write!(f, "Regex error: {e}"),
        }
    }
}

impl std::error::Error for CompileError {}

/// A single found match. Byte offsets refer to the input `&str`.
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub struct FindHit {
    pub start: usize,
    pub end: usize,
    pub wrapped: bool,
}

/// Compile a pattern under the given flags. Wraps `regex::RegexBuilder` with:
///   - literal-mode escape (D-5)
///   - whole-word boundary wrapping in literal mode (greyed in regex mode — UI ignores)
///   - case_insensitive(!case_sensitive)
///   - size_limit(10 MB) — Pitfall 5 cap
///   - empty-match rejection — Pitfall 3
pub fn compile(pattern: &str, flags: FindFlags) -> Result<Regex, CompileError> {
    if pattern.is_empty() {
        return Err(CompileError::Empty);
    }
    let final_pat = if flags.regex {
        pattern.to_owned()
    } else if flags.whole_word {
        format!(r"\b{}\b", regex::escape(pattern))
    } else {
        regex::escape(pattern)
    };
    let regex = RegexBuilder::new(&final_pat)
        .case_insensitive(!flags.case_sensitive)
        .size_limit(10 * 1024 * 1024)
        .build()
        .map_err(CompileError::Regex)?;
    if regex.is_match("") {
        return Err(CompileError::EmptyMatch);
    }
    Ok(regex)
}

/// Find the next match relative to `cursor` (UTF-8 byte offset).
/// Forward search returns the first match at-or-after cursor.
/// Backward search returns the last match strictly before cursor.
/// If wrap is enabled and the directed search yields nothing, the search restarts from the
/// other end and the returned FindHit is marked `wrapped: true`.
///
/// Backward at cursor=0 with wrap returns the buffer's last match (Pitfall 4 — D-9).
pub fn find_next(text: &str, regex: &Regex, cursor: usize, flags: FindFlags) -> Option<FindHit> {
    let cursor = cursor.min(text.len());
    if flags.backward {
        // Search [0..cursor) for the last match strictly before cursor.
        if cursor > 0 {
            if let Some(m) = regex.find_iter(&text[..cursor]).last() {
                return Some(FindHit { start: m.start(), end: m.end(), wrapped: false });
            }
        }
        if flags.wrap {
            // Wrap: return the last match in the entire buffer (D-9 special case for cursor=0).
            if let Some(m) = regex.find_iter(text).last() {
                return Some(FindHit { start: m.start(), end: m.end(), wrapped: true });
            }
        }
        None
    } else {
        // Forward: search [cursor..end) for the first match.
        if cursor < text.len() {
            if let Some(m) = regex.find(&text[cursor..]) {
                return Some(FindHit {
                    start: cursor + m.start(),
                    end: cursor + m.end(),
                    wrapped: false,
                });
            }
        }
        if flags.wrap {
            // Wrap: search [0..cursor) for the first match.
            if let Some(m) = regex.find(&text[..cursor]) {
                return Some(FindHit { start: m.start(), end: m.end(), wrapped: true });
            }
        }
        None
    }
}

/// Replace all matches in `text` with `replacement`. The replacement string honors
/// `$0` (full match), `$1`/`$2`/... (numbered groups), and `${name}` (named groups)
/// per the regex crate's `Captures::expand` semantics — invoked automatically by
/// `Regex::replace_all`. Returns (new_text, count).
pub fn replace_all(text: &str, regex: &Regex, replacement: &str) -> (String, usize) {
    let count = regex.find_iter(text).count();
    let new_text = regex.replace_all(text, replacement).into_owned();
    (new_text, count)
}

/// Convert a UTF-8 byte offset (used by the engine) into a UTF-16 code-unit offset
/// (used by RichEdit's `EM_EXSETSEL` / `EM_GETSEL`).
///
/// Behavior at boundaries:
///   - `byte_offset` past `text.len()` clamps to `text.len()`.
///   - For valid UTF-8 (which `&str` guarantees), this returns the exact UTF-16 cu count
///     of the prefix `text[..byte_offset]`.
///
/// Performance: O(n) in `byte_offset`. For 1MB buffers this is ~1ms — not a hot-path concern.
pub fn utf8_to_utf16_units(text: &str, byte_offset: usize) -> usize {
    let clamped = byte_offset.min(text.len());
    text[..clamped].encode_utf16().count()
}

/// Inverse of `utf8_to_utf16_units`. Walks `text.char_indices()` accumulating UTF-16 code
/// units until `cu_offset` is reached, returning the corresponding UTF-8 byte offset.
///
/// Behavior at boundaries:
///   - `cu_offset` past the total UTF-16 length clamps to `text.len()`.
///   - On UTF-16 surrogate-pair codepoints (which take 2 cu), the returned byte offset is
///     where the codepoint STARTS — i.e., we never split a Rust `char`.
pub fn utf16_to_utf8_offset(text: &str, cu_offset: usize) -> usize {
    let mut cu_count = 0usize;
    for (byte_idx, ch) in text.char_indices() {
        if cu_count >= cu_offset {
            return byte_idx;
        }
        cu_count = cu_count.saturating_add(ch.len_utf16());
    }
    text.len()
}

#[test]

        Err(CompileError::Empty) => {}
        other => panic!("expected Empty, got {other:?}"),

#[test]
fn compile_rejects_empty_match_in_regex_mode() {

#[test]
