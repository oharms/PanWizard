## Plan Decisions

(See @./.claude/pan-wizard-core/references/handoff-decisions.md for the schema.)

### Locked (executor MUST follow)

- D-1: **Modeless dialog built via `CreateWindowExW` on a registered class** — NOT `DialogBoxIndirectParamW`, NOT a `.rc` resource. Why: the project has a no-resource-files constraint (idea.md "bake icons / language definitions into the binary"), and `DialogBoxIndirectParamW` requires a DLGTEMPLATE at runtime; building a child window with manually-positioned button/edit children is mechanically simpler. Source: research.md §"Find/Replace UI" + Plan Breakdown 03-04 row.
- D-2: **One dialog template handles BOTH Find and Find & Replace.** A `FindMode` enum (`Find`/`Replace`) toggles which row is shown. In `Replace` mode, the Replace input + Replace + Replace All buttons are visible; in `Find` mode they're hidden (or the dialog is shorter). Why: same set of toggles, same engine; doubling code would just create maintenance burden. Source: research.md §"Find/Replace UI: dockable bar vs modeless dialog" Open Question.
- D-3: **Find/Replace state is GLOBAL on `App.find_state`** (not per-tab). When user presses Ctrl+F, dialog opens with `app.find_state.pattern` pre-filled. When user types, `app.find_state.pattern/replacement/flags` update. Why: Notepad++ default; matches user expectation; cheaper to implement than per-tab state. Source: research.md §"Find/Replace state scope: per-tab vs global".
- D-4: **Pitfall 10: ALL find/replace operations run on `eol::normalize_to_lf`-normalized text.** The pipeline: `EditorState::save_text` (EM_STREAMOUT, gives RichEdit's CR-only) → `std::str::from_utf8` → `eol::normalize_to_lf` → run regex. Selection offsets are converted from LF-text byte offsets back to RichEdit UTF-16 cu offsets via `find::utf8_to_utf16_units` — but **the byte-offset-to-UTF-16-cu conversion must use the LF-text** since that's the basis of regex matching. **Caveat:** RichEdit uses CR-only paragraph terminators, but for selection offsets the conversion is still byte-count-equivalent because LF and CR are both 1 byte/1 cu. The conversion DOES NOT need a CR↔LF re-mapping for selection — only for replace. Verify in QA: a CRLF buffer's "Find Next" puts the selection where the user expects (it should — both \r and \n are 1 cu, and find offsets work in characters not bytes-vs-cu mismatch). Source: research.md Pitfall 10.
- D-5: **Selection via `EM_EXSETSEL` (NOT `EM_SETSEL`)** with a `CHARRANGE { cpMin, cpMax }` struct. EM_SETSEL packs offsets into 16-bit ints (capped at 65535 chars — too small for 1MB+ buffers). Why: orchestrator/research.md Pattern 7. Source: research.md Pattern 7.
- D-6: **After selection: `EM_SCROLLCARET`** to scroll the match into view. Do NOT call `SetFocus(richedit)` after selection — leave focus in the Find dialog so user can press Find Next again. The active RichEdit has `ES_NOHIDESEL` (Phase 1 — verified at src/editor.rs line 47) so the selection is visible even when not focused. Source: research.md Pitfall 6.
- D-7: **Pitfall 7 — `IsDialogMessageW` pre-dispatch in the message loop.** Add to `src/main.rs::run_app` message loop: when `app.find_dlg_hwnd` is `Some(hdlg)` AND the dialog is visible, call `IsDialogMessageW(hdlg, &mut msg)` BEFORE `TranslateMessage` — but AFTER `TranslateAcceleratorW`. **Order:** `TranslateAccelerator → IsDialogMessage(if find_dlg_hwnd) → TranslateMessage → DispatchMessage`. Why: F3/Ctrl+F/Esc accelerators must work even when the dialog has focus; Tab/Enter dialog navigation must still work for the dialog. Source: research.md Pitfall 7.
- D-8: **Replace All wrapping (Pitfall 6 + orchestrator quality_gate):**
  ```
  SendMessageW(hwnd_re, EM_STOPGROUPTYPING, 0, 0)   // start a single undo group
  // Build new_text via find::replace_all(text, regex, replacement)
  // Convert new_text → UTF-16 vec
  let st = SETTEXTEX { flags: ST_KEEPUNDO, codepage: 1200 /* unicode */ };
  SendMessageW(hwnd_re, EM_SETTEXTEX, &st, new_text_utf16.as_ptr())
  SendMessageW(hwnd_re, EM_STOPGROUPTYPING, 0, 0)   // close the group
  SendMessageW(hwnd_re, EM_SETMODIFY, 1, 0)         // mark dirty
  ```
  **One Ctrl+Z reverts the entire Replace All.** Verify in QA: after Replace All on a multi-match buffer, one Ctrl+Z restores the original text. Source: research.md FIND-08 + Pitfall 6 + orchestrator quality_gate. **NOTE:** if the QA reveals RichEdit creates multiple undo entries despite EM_STOPGROUPTYPING, the fallback (per research.md Tertiary Sources LOW confidence) is to wrap the loop differently — document the deviation in the summary and consider QA-flag work.
- D-9: **Whole-word toggle: greyed when regex mode is on** (Notepad++ behavior). Backward toggle: greyed when regex mode is on. Implementation: `EnableWindow(hwnd_check_wholeword, !flags.regex)`; same for backward. Why: research.md FIND-05 (whole-word in regex is user's responsibility); regex crate has no native reverse search. Source: research.md FIND-05, FIND-03 row.
- D-10: **Compile errors render in the status line + red-tint pattern input.** When `find::compile` returns `Err`, set status to format!("Bad pattern: {}", err) and disable Find Next / Find Previous / Replace / Replace All buttons. `CompileError::Empty` → silent no-op (no error visible — empty pattern is the "user just opened the dialog" state). Why: research.md FIND-06 + Pitfall 5. Source: research.md FIND-06.
- D-11: **Status line text** for FIND-09 wrap-around: "Wrapped to top" (forward wrap), "Wrapped to bottom" (backward wrap), "Pattern not found" (no match anywhere). Mirror Notepad++. Source: research.md FIND-09 + features.md SC-3 surface.
- D-12: **The dialog's WndProc is a private function** in `find_ui.rs`. WM_COMMAND from buttons / edits / checkboxes posts directly to the parent's WndProc via `SendMessageW(parent, WM_COMMAND, ...)` to keep all dispatch logic in one place. Specifically: clicking "Find Next" sends `WM_COMMAND(IDM_FIND_NEXT)` to the parent; clicking "Replace" sends `WM_COMMAND(IDM_FIND_REPLACE_ONE)` (NEW IDM constant added in this plan, since 03-01 reserved 130-133 but didn't add a "replace one" — see Open question O-3); clicking "Replace All" sends `WM_COMMAND(IDM_FIND_REPLACE_ALL)`. Source: research.md §"Wave 2 Plan 03-04" Code Example.
  - **Constants — pre-allocated by Plan 03-01 (D-5 update via revision iteration 1):** `IDM_FIND_REPLACE_ONE: u16 = 134;` and `IDM_FIND_REPLACE_ALL: u16 = 135;` already exist as `pub const` in `src/app.rs` from 03-01. This plan only references them by name; **do NOT re-declare them**. Same for `WM_APP_FIND_DIALOG_CLOSED = WM_USER + 11` (03-01 D-5).
- D-13: **`IsDialogMessageW` only fires when dialog has focus.** Verify with `IsWindowVisible(hdlg)` AND `GetFocus()`-is-descendant-of-`hdlg`. The simplest implementation: `if app.find_dlg_hwnd.is_some() && unsafe { IsDialogMessageW(app.find_dlg_hwnd.unwrap(), &mut msg).as_bool() } { continue; }`. Source: research.md Pattern 3 message loop snippet.
- D-14: **Closing the dialog** clears `app.find_dlg_hwnd = None` and increments `app.find_epoch` (Pitfall 9 — discard any in-flight worker results). Why: 03-05's worker uses `find_epoch` for cancellation; closing the dialog should cancel any pending search. Source: research.md Pitfall 9.
- D-15: **NO Search menu in this plan.** The dialog is reachable via Ctrl+F / Ctrl+H accelerators (Plan 03-01 already wired) and via the Tabs menu (no — Tabs menu is for tabs). **Add a Search menu with Find / Replace / Find Next / Find Previous entries to `src/menu.rs`.** This is a small addition, mirrors the Tabs menu Plan 03-02 added; needed so users without keyboard discoverability can find the feature. Source: pitfalls.md Pitfall 13 row 8 — keyboard shortcuts must have menu equivalents.
- D-16: **Module declaration:** add `mod find_ui;` to `src/main.rs` and `pub mod find_ui;` to `src/lib.rs`. Source: project convention.

### Open (executor's discretion within constraints)

- O-1: **Layout/sizing of the dialog.** Recommendation: 400x180 px in Find mode, 400x240 px in Replace mode. Pattern label+input + Replace label+input each ~24px tall; toggles row ~30px; buttons row ~30px; status line ~20px; padding. Constraint: NO `.rc` files; all dimensions hardcoded in `find_ui.rs`. The exact layout is a judgement call — pick what looks reasonable; high-DPI scaling can be a v1.x polish task.
- O-2: **Whether the dialog has a close button (X).** The dialog is a `WS_POPUP | WS_CAPTION | WS_SYSMENU` style window so it gets a close X for free. Pressing Escape should also close (handle in WM_KEYDOWN with VK_ESCAPE). Recommendation: include both; cheap.
- O-3: **IDM_FIND_REPLACE_ONE / IDM_FIND_REPLACE_ALL constants.** Either (a) add to `src/app.rs` (mirror Plan 03-01's pre-allocation pattern) or (b) define inside `src/find_ui.rs` as private constants. Recommendation: (a) — they're command IDs that the dispatch.rs WM_COMMAND match needs, and Plan 03-05 might reference them too. Constraint: 134/135 (next available after Plan 03-01's 130-133).
- O-4: **Caching the compiled regex on `App.find_state`.** Currently `FindState` has `pattern`/`replacement`/`flags` but no compiled `Regex`. Compiling on every find call is fine for v1 (research.md says compile is fast for sane patterns). If the executor finds Find Next perceptibly slow on a large pattern, add `compiled: Option<Regex>` to FindState and invalidate when pattern/flags change. Recommendation: don't add unless profiling shows need.
- O-5: **Whether to share the dialog instance across Ctrl+F / Ctrl+H presses.** Two options: (a) ALWAYS reuse `app.find_dlg_hwnd` if it's `Some` — show/hide and toggle mode; (b) destroy + recreate on each Ctrl+F/Ctrl+H. Recommendation: (a) — Notepad++ does this; the dialog state persists (pattern, flags) across opens, which matches user expectation. If toggling between Find and Replace modes is awkward (extra row appears/disappears), the user can close and reopen.

### Considered and rejected

- R-1: **`DialogBoxIndirectParamW` / `.rc`-file template.** Rejected: no-resource-files constraint; building a child window with manual `CreateWindowExW` on each control is the same code shape and avoids the COMCTL32 dialog procedure layer.
- R-2: **`FindText` / `ReplaceText` Win32 common dialogs.** Rejected: deprecated UX (research.md Stack Patterns; 1995-vintage UI); no extensibility for our 4 toggles + status line.
- R-3: **Per-tab find state.** Rejected: research.md (Notepad++ default is global). Adds complexity for no user-visible benefit.
- R-4: **Synchronous Replace All on >1MB buffers.** Accepted for now (Plan 03-05 may move it to worker if needed). Research.md Open Question 3 explicitly says "Phase 3 ships Replace All synchronously regardless of size; worker-thread Replace All is a v1.x optimization."

//! Modeless Find / Find & Replace dialog. Built via CreateWindowExW on a registered class
//! (no .rc resource — honors the no-resource-files constraint). User input is forwarded to
//! the parent's WM_COMMAND so all dispatch logic stays in src/dispatch.rs.

#![deny(clippy::unwrap_used)]
#![deny(clippy::expect_used)]

use windows::core::*;
use windows::Win32::Foundation::*;
use windows::Win32::Graphics::Gdi::*;
use windows::Win32::System::LibraryLoader::*;
use windows::Win32::UI::Input::KeyboardAndMouse::*;
use windows::Win32::UI::WindowsAndMessaging::*;

use crate::app::{
    IDM_FIND_NEXT, IDM_FIND_PREV, IDM_FIND_REPLACE_ALL, IDM_FIND_REPLACE_ONE,
};

/// Find vs Replace mode. Toggling the mode shows/hides the Replace input + Replace buttons.
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum FindMode {
    Find,
    Replace,
}

/// Window-class name for the find dialog.
const FIND_DLG_CLASS: PCWSTR = w!("notepadrs_find_dialog");

/// Child-control IDs (within the dialog only — distinct from app.rs's IDM_*).
const ID_PATTERN_EDIT: u16     = 1001;
const ID_REPLACE_EDIT: u16     = 1002;
const ID_CHECK_CASE: u16       = 1010;
const ID_CHECK_WHOLEWORD: u16  = 1011;
const ID_CHECK_REGEX: u16      = 1012;
const ID_CHECK_WRAP: u16       = 1013;
const ID_CHECK_BACKWARD: u16   = 1014;
const ID_BTN_FIND_NEXT: u16    = 1020;
const ID_BTN_FIND_PREV: u16    = 1021;
const ID_BTN_REPLACE: u16      = 1022;
const ID_BTN_REPLACE_ALL: u16  = 1023;
const ID_BTN_CLOSE: u16        = 1024;
const ID_STATUS_LABEL: u16     = 1030;

/// Per-dialog state stored in GWLP_USERDATA. Owned by Box; freed in WM_NCDESTROY.
struct DlgState {
    parent: HWND,
    mode: FindMode,
    /// HWNDs of the child controls — cached so we can EnableWindow/SetWindowText etc.
    h_pattern: HWND,
    h_replace: HWND,
    h_check_case: HWND,
    h_check_wholeword: HWND,
    h_check_regex: HWND,
    h_check_wrap: HWND,
    h_check_backward: HWND,
    h_btn_find_next: HWND,
    h_btn_find_prev: HWND,
    h_btn_replace: HWND,
    h_btn_replace_all: HWND,
    h_btn_close: HWND,
    h_status: HWND,
}

/// Register the dialog window class. Idempotent — safe to call multiple times via OnceLock.
unsafe fn register_class() -> Result<()> {
    use std::sync::OnceLock;
    static REGISTERED: OnceLock<()> = OnceLock::new();
    let mut err: Option<windows::core::Error> = None;
    REGISTERED.get_or_init(|| {
        let hinstance: HINSTANCE = match GetModuleHandleW(None) {
            Ok(h) => h.into(),
            Err(e) => { err = Some(e); return; }
        };
        let wc = WNDCLASSW {
            lpfnWndProc: Some(find_dlg_proc),
            hInstance: hinstance,
            lpszClassName: FIND_DLG_CLASS,
            hCursor: LoadCursorW(None, IDC_ARROW).unwrap_or_default(),
            hbrBackground: HBRUSH((COLOR_BTNFACE.0 + 1) as *mut _),
            ..Default::default()
        };
        if RegisterClassW(&wc) == 0 {
            err = Some(windows::core::Error::from_thread());
        }
    });
    if let Some(e) = err { return Err(e); }
    Ok(())
}

/// Open (or focus, if already open) the find dialog in the given mode. Returns the dialog HWND.
pub unsafe fn open_find_dialog(
    parent: HWND,
    existing: Option<HWND>,
    mode: FindMode,
    initial_pattern: &str,
    initial_replacement: &str,
    flags: crate::app::FindFlags,
) -> Result<HWND> {
    if let Some(existing_hwnd) = existing {
        if IsWindow(Some(existing_hwnd)).as_bool() {
            // Already open — switch mode and re-show.
            set_dialog_mode(existing_hwnd, mode);
            let _ = ShowWindow(existing_hwnd, SW_SHOW);
            let _ = SetForegroundWindow(existing_hwnd);
            return Ok(existing_hwnd);
        }
    }
    register_class()?;

    let hinstance: HINSTANCE = GetModuleHandleW(None)?.into();
    let title = if mode == FindMode::Replace { w!("Find && Replace") } else { w!("Find") };

    // Position: roughly center over parent. For v1 use a fixed 100,100 offset.
    let width = 420;
    let height = if mode == FindMode::Replace { 250 } else { 180 };

    let hwnd_dlg = CreateWindowExW(
        WS_EX_DLGMODALFRAME | WS_EX_TOOLWINDOW,
        FIND_DLG_CLASS,
        title,
        WS_POPUP | WS_CAPTION | WS_SYSMENU,
        100, 100, width, height,
        Some(parent),
        None,
        Some(hinstance),
        None,
    )?;

/// Dialog WndProc. Routes WM_COMMAND from buttons to the parent's WM_COMMAND.
unsafe extern "system" fn find_dlg_proc(
    hwnd: HWND, msg: u32, wparam: WPARAM, lparam: LPARAM,
) -> LRESULT {
    match msg {
        WM_COMMAND => {
            let id = (wparam.0 as u16) as u32;
            // Forward button clicks to the parent's WM_COMMAND.
            let parent = GetParent(hwnd).unwrap_or(HWND(std::ptr::null_mut()));
            match id as u16 {
                ID_BTN_FIND_NEXT => {
                    let _ = SendMessageW(parent, WM_COMMAND, Some(WPARAM(IDM_FIND_NEXT as usize)), Some(LPARAM(0)));
                }
                ID_BTN_FIND_PREV => {
                    let _ = SendMessageW(parent, WM_COMMAND, Some(WPARAM(IDM_FIND_PREV as usize)), Some(LPARAM(0)));
                }
                ID_BTN_REPLACE => {
                    let _ = SendMessageW(parent, WM_COMMAND, Some(WPARAM(IDM_FIND_REPLACE_ONE as usize)), Some(LPARAM(0)));
                }
                ID_BTN_REPLACE_ALL => {
                    let _ = SendMessageW(parent, WM_COMMAND, Some(WPARAM(IDM_FIND_REPLACE_ALL as usize)), Some(LPARAM(0)));
                }
                ID_BTN_CLOSE => {
                    let _ = DestroyWindow(hwnd);
                }
                ID_CHECK_REGEX => {
                    // Update greyout state when regex toggle changes.
                    let ptr = GetWindowLongPtrW(hwnd, GWLP_USERDATA) as *const DlgState;
                    if !ptr.is_null() {
                        let s = &*ptr;
                        let regex_on = is_checked(s.h_check_regex);
                        let _ = EnableWindow(s.h_check_wholeword, !regex_on);
                        let _ = EnableWindow(s.h_check_backward, !regex_on);
                    }
                }
                _ => {}
            }
            LRESULT(0)
        }
        WM_KEYDOWN if wparam.0 as u16 == VK_ESCAPE.0 => {
            let _ = DestroyWindow(hwnd);
            LRESULT(0)
        }
        WM_NCDESTROY => {
            let ptr = GetWindowLongPtrW(hwnd, GWLP_USERDATA) as *mut DlgState;
            if !ptr.is_null() {
                SetWindowLongPtrW(hwnd, GWLP_USERDATA, 0);
                let _boxed = Box::from_raw(ptr);
            }
            DefWindowProcW(hwnd, msg, wparam, lparam)
        }
        _ => DefWindowProcW(hwnd, msg, wparam, lparam),
    }
}

unsafe fn do_find_next(app: &mut App, hwnd: HWND, backward: bool) {
    use std::sync::atomic::Ordering;
    use windows::Win32::UI::Controls::RichEdit::*;
    use windows::Win32::UI::Controls::{EM_EXGETSEL, EM_EXSETSEL, EM_SCROLLCARET};

    // Read latest dialog state into FindState.
    if let Some(hdlg) = app.find_dlg_hwnd {
        crate::find_ui::read_find_state(hdlg, &mut app.find_state);
    }
    let mut flags = app.find_state.flags;
    flags.backward = backward;

    // D-10: Empty pattern → silent no-op.
    if app.find_state.pattern.is_empty() {
        return;
    }

    let regex = match crate::find::compile(&app.find_state.pattern, flags) {
        Ok(r) => r,
        Err(e) => {
            if let Some(hdlg) = app.find_dlg_hwnd {
                let msg = match &e {
                    crate::find::CompileError::Empty => return,  // silent
                    crate::find::CompileError::EmptyMatch => "Pattern matches empty string".to_owned(),
                    crate::find::CompileError::Regex(re) => format!("Bad pattern: {re}"),
                };
                crate::find_ui::set_status(hdlg, &msg);
            }
            return;
        }
    };

    let tab = match app.tabs.get(app.active_tab) {
        Some(t) => t,
        None => return,
    };

    // D-4: stream out, normalize to LF.
    let cr_bytes = match tab.editor.save_text() { Ok(b) => b, Err(_) => return };
    let cr_text = match std::str::from_utf8(&cr_bytes) { Ok(s) => s, Err(_) => return };
    let lf_text = crate::eol::normalize_to_lf(cr_text);

    // Resolve cursor (UTF-16 cu) from RichEdit.
    let mut cr = windows::Win32::UI::Controls::CHARRANGE { cpMin: 0, cpMax: 0 };
    let _ = SendMessageW(
        tab.editor.hwnd_re,
        EM_EXGETSEL,
        Some(WPARAM(0)),
        Some(LPARAM(&mut cr as *mut _ as isize)),
    );
    // For "Find Next" (forward): cursor = current selection END (so we don't re-find current match).
    // For "Find Prev" (backward): cursor = current selection START.
    let cursor_cu = if backward { cr.cpMin as usize } else { cr.cpMax as usize };
    let cursor_byte = crate::find::utf16_to_utf8_offset(&lf_text, cursor_cu);

    let hit = crate::find::find_next(&lf_text, &regex, cursor_byte, flags);
    match hit {
        Some(h) => {
            // Convert byte offsets back to UTF-16 cu and apply selection.
            let cu_start = crate::find::utf8_to_utf16_units(&lf_text, h.start) as i32;
            let cu_end = crate::find::utf8_to_utf16_units(&lf_text, h.end) as i32;
            let cr_set = windows::Win32::UI::Controls::CHARRANGE { cpMin: cu_start, cpMax: cu_end };
            let _ = SendMessageW(
                tab.editor.hwnd_re,
                EM_EXSETSEL,
                Some(WPARAM(0)),
                Some(LPARAM(&cr_set as *const _ as isize)),
            );
            let _ = SendMessageW(tab.editor.hwnd_re, EM_SCROLLCARET, Some(WPARAM(0)), Some(LPARAM(0)));
            // D-6: do NOT SetFocus — leave focus in dialog.
            if let Some(hdlg) = app.find_dlg_hwnd {
                let msg = if h.wrapped {
                    if backward { "Wrapped to bottom" } else { "Wrapped to top" }
                } else { "" };
                crate::find_ui::set_status(hdlg, msg);
            }
        }
        None => {
            if let Some(hdlg) = app.find_dlg_hwnd {
                crate::find_ui::set_status(hdlg, "Pattern not found");
            }
        }
    }
    // D-14: bump epoch (not strictly needed for sync path, but discipline for 03-05).
    app.find_epoch.fetch_add(1, Ordering::Release);
}

unsafe fn do_replace_one(app: &mut App, hwnd: HWND) {
    use windows::Win32::UI::Controls::RichEdit::*;
    use windows::Win32::UI::Controls::{EM_EXGETSEL, EM_REPLACESEL};

    if let Some(hdlg) = app.find_dlg_hwnd {
        crate::find_ui::read_find_state(hdlg, &mut app.find_state);
    }
    let flags = app.find_state.flags;
    if app.find_state.pattern.is_empty() { return; }

    let regex = match crate::find::compile(&app.find_state.pattern, flags) {
        Ok(r) => r,
        Err(_) => { do_find_next(app, hwnd, false); return; }
    };

    let tab = match app.tabs.get(app.active_tab) { Some(t) => t, None => return };
    let cr_bytes = match tab.editor.save_text() { Ok(b) => b, Err(_) => return };
    let cr_text = match std::str::from_utf8(&cr_bytes) { Ok(s) => s, Err(_) => return };
    let lf_text = crate::eol::normalize_to_lf(cr_text);

    // Get current selection.
    let mut cr = windows::Win32::UI::Controls::CHARRANGE { cpMin: 0, cpMax: 0 };
    let _ = SendMessageW(tab.editor.hwnd_re, EM_EXGETSEL, Some(WPARAM(0)), Some(LPARAM(&mut cr as *mut _ as isize)));

    if cr.cpMin == cr.cpMax {
        // No selection — do a Find Next instead.
        do_find_next(app, hwnd, false);
        return;
    }

    // Check whether the current selection actually matches the pattern.
    let sel_byte_start = crate::find::utf16_to_utf8_offset(&lf_text, cr.cpMin as usize);
    let sel_byte_end = crate::find::utf16_to_utf8_offset(&lf_text, cr.cpMax as usize);
    let sel_text = &lf_text[sel_byte_start..sel_byte_end];
    let m = regex.find(sel_text);
    let exact_match = m.map(|m| m.start() == 0 && m.end() == sel_text.len()).unwrap_or(false);

    if !exact_match {
        // Selection isn't a match — Find Next.
        do_find_next(app, hwnd, false);
        return;
    }

    // Replace the selection with the replacement (with $0/$1 expansion).
    let mut replacement_expanded = String::new();
    let caps = match regex.captures(sel_text) {
        Some(c) => c,
        None => { do_find_next(app, hwnd, false); return; }
    };
    caps.expand(&app.find_state.replacement, &mut replacement_expanded);
    let repl_w: Vec<u16> = replacement_expanded.encode_utf16().chain(std::iter::once(0)).collect();
    let _ = SendMessageW(
        tab.editor.hwnd_re,
        EM_REPLACESEL,
        Some(WPARAM(1 /* TRUE — undoable */)),
        Some(LPARAM(repl_w.as_ptr() as isize)),
    );
    // Advance to next match.
    do_find_next(app, hwnd, false);
}

unsafe fn do_replace_all(app: &mut App, hwnd: HWND) {
    use windows::Win32::UI::Controls::RichEdit::*;

    if let Some(hdlg) = app.find_dlg_hwnd {
        crate::find_ui::read_find_state(hdlg, &mut app.find_state);
    }
    let flags = app.find_state.flags;
    if app.find_state.pattern.is_empty() { return; }

    let regex = match crate::find::compile(&app.find_state.pattern, flags) {
        Ok(r) => r,
        Err(e) => {
            if let Some(hdlg) = app.find_dlg_hwnd {
                let msg = match &e {
                    crate::find::CompileError::Empty => return,
                    crate::find::CompileError::EmptyMatch => "Pattern matches empty string".to_owned(),
                    crate::find::CompileError::Regex(re) => format!("Bad pattern: {re}"),
                };
                crate::find_ui::set_status(hdlg, &msg);
            }
            return;
        }
    };

    let tab = match app.tabs.get_mut(app.active_tab) { Some(t) => t, None => return };
    let cr_bytes = match tab.editor.save_text() { Ok(b) => b, Err(_) => return };
    let cr_text = match std::str::from_utf8(&cr_bytes) { Ok(s) => s, Err(_) => return };
    let lf_text = crate::eol::normalize_to_lf(cr_text);

    let (new_lf_text, count) = crate::find::replace_all(&lf_text, &regex, &app.find_state.replacement);

    // Re-emit with the tab's EOL style so RichEdit sees correct paragraph breaks.
    let new_text_for_richedit = crate::eol::denormalize_to_eol(&new_lf_text, tab.editor.eol);
    let new_text_str = std::str::from_utf8(&new_text_for_richedit).unwrap_or(&new_lf_text);
    let new_text_w: Vec<u16> = new_text_str.encode_utf16().chain(std::iter::once(0)).collect();

    // D-8: single undo step via EM_STOPGROUPTYPING + EM_SETTEXTEX(ST_KEEPUNDO).
    let _ = SendMessageW(tab.editor.hwnd_re, EM_STOPGROUPTYPING, Some(WPARAM(0)), Some(LPARAM(0)));
    let st = SETTEXTEX { flags: ST_KEEPUNDO, codepage: 1200 /* unicode */ };
    let _ = SendMessageW(
        tab.editor.hwnd_re,
        EM_SETTEXTEX,
        Some(WPARAM(&st as *const _ as usize)),
        Some(LPARAM(new_text_w.as_ptr() as isize)),
    );
    let _ = SendMessageW(tab.editor.hwnd_re, EM_STOPGROUPTYPING, Some(WPARAM(0)), Some(LPARAM(0)));
    // Mark dirty.
    use windows::Win32::UI::Controls::EM_SETMODIFY;
    let _ = SendMessageW(tab.editor.hwnd_re, EM_SETMODIFY, Some(WPARAM(1)), Some(LPARAM(0)));
    tab.dirty = true;
    invalidate_tab_strip_item(app, app.active_tab);

    if let Some(hdlg) = app.find_dlg_hwnd {
        crate::find_ui::set_status(hdlg, &format!("Replaced {count} occurrence(s)"));
    }
}

/// Pure-logic core of do_find_next: given the LF-normalized buffer and a regex, returns the
/// next match's UTF-8 byte offsets (start, end, wrapped). No Win32, no EditorState, no HWND.
/// Win32 wrapper (do_find_next) handles EM_STREAMOUT + cursor resolution + EM_EXSETSEL.
pub fn find_next_pure(

/// Pure-logic core of do_replace_one: given the LF-normalized buffer + selection + regex +
/// replacement, returns (new_text, new_selection_cu_range, did_replace). Tests verify
/// (1) selection-matches-pattern semantics, (2) capture-group $0/$1 expansion, (3) cursor advance.
pub fn replace_one_pure(
    lf_text: &str,
    sel_cu_start: usize,
    sel_cu_end: usize,
    regex: &regex::Regex,
    replacement: &str,
) -> Option<(String, usize, usize)> {
    let sel_byte_start = crate::find::utf16_to_utf8_offset(lf_text, sel_cu_start);
    let sel_byte_end = crate::find::utf16_to_utf8_offset(lf_text, sel_cu_end);
    if sel_byte_start >= sel_byte_end || sel_byte_end > lf_text.len() {
        return None;
    }
    let sel_text = &lf_text[sel_byte_start..sel_byte_end];
    let caps = regex.captures(sel_text)?;
    if caps.get(0).map(|m| m.start() != 0 || m.end() != sel_text.len()).unwrap_or(true) {
        return None;
    }
    let mut expanded = String::new();
    caps.expand(replacement, &mut expanded);
    let mut new_text = String::with_capacity(lf_text.len() + expanded.len());
    new_text.push_str(&lf_text[..sel_byte_start]);
    new_text.push_str(&expanded);
    new_text.push_str(&lf_text[sel_byte_end..]);
    let new_sel_byte = sel_byte_start + expanded.len();
    let new_sel_cu = crate::find::utf8_to_utf16_units(&new_text, new_sel_byte);
    Some((new_text, new_sel_cu, new_sel_cu))
}
