## Plan Decisions

(See @./.claude/pan-wizard-core/references/handoff-decisions.md for the schema.)

### Locked (executor MUST follow)

- **D-1**: Use `SHGetKnownFolderPath(FOLDERID_RoamingAppData, KF_FLAG_CREATE, None)` for AppData path
  resolution. NOT `%APPDATA%` env var. Why: env vars miss elevation/impersonation cases; KF_FLAG_CREATE
  also creates the folder if missing (RECENT-04 robustness). Source: research.md §"Pattern 2"
  + §"Don't Hand-Roll" line "AppData path".
- **D-2**: Persist as JSON via `serde_json::to_string_pretty` / `from_str`. NOT bincode or
  hand-rolled. Why: human-debuggable; recent.json is tiny (<1KB) so size is irrelevant; serde_json
  pulls ~150-300KB into binary which is well under the 8MB headroom. Source: research.md §"Standard
  Stack" + §"Don't Hand-Roll".
- **D-3**: Store paths as `String` (UTF-8) inside `RecentFiles`, NOT `PathBuf`. Why: PathBuf's
  serde impl is platform-quirky on non-UTF-8 paths; reject non-UTF-8 paths at the boundary instead.
  Source: research.md §"Anti-Patterns" line "Storing PathBuf directly".
- **D-4**: Atomic write via tmp+rename (mirror existing `file::save_atomic` at src/file.rs:92).
  Sequence: serialize → write to `recent.json.tmp` → `fs::rename` to `recent.json`. Why: prevents
  half-written files on power-loss/crash. Source: research.md §"Pattern 2".
- **D-5**: Capacity = exactly 10 (RECENT-01 spec). Constant `RECENT_MAX: usize = 10`.
  Path-length sanity cap = 32_768 (`RECENT_PATH_MAX_LEN`). Why: hardens against tampered
  recent.json (Pitfall 10). Source: research.md §"Pattern 2" + pitfalls.md Pitfall 10.
- **D-6**: LRU semantics via `Vec::retain` + `insert(0, x)` + `truncate(10)`. Dedupe key:
  case-insensitive ASCII compare (`eq_ignore_ascii_case`) — Windows paths are case-insensitive
  but we preserve the user's casing in display. Why: research.md §"Don't Hand-Roll".
- **D-7**: IDM range 200-209 (10 slots). `IDM_RECENT_BASE = 200`, `IDM_RECENT_MAX = 209`.
  Click handler: `cmd in IDM_RECENT_BASE..=IDM_RECENT_MAX` → `idx = cmd - IDM_RECENT_BASE` →
  lookup path by index. Why: Phase 5 reserved 200-209 in research.md; static range avoids the
  "menu surgery on every push" anti-pattern.
- **D-8**: WM_INITMENUPOPUP-rebuild pattern (NOT incremental update). Submenu cleared via
  `DeleteMenu` loop on every popup, then repopulated from `app.recent.paths`. Why: simpler to
  reason about; matches MFC/Notepad++ canonical pattern. Source: research.md §"Pattern 4".
- **D-9**: Empty list shows a single `MF_GRAYED` "(No recent files)" entry, NOT an empty popup.
  Why: better UX than a zero-height popup; matches research.md §"Pattern 4" example skeleton.
- **D-10**: Persistence trigger: only on `push_lru` success in `open_path_external`. NOT on
  startup load, NOT on every menu mouseover. Why: research.md §"Anti-Patterns" — eager save
  wastes disk I/O.
- **D-11**: Test file name MUST be `tests/recent_files.rs` (NOT `tests/recent_dispatch.rs`).
  Why: research.md §"Test Pitfalls" — Windows AppCompat shim error 740 blocks `*dispatch*`
  test binaries.
- **D-12**: Cargo.toml additions are EXACTLY: `serde = { version = "1.0", features = ["derive"] }`
  and `serde_json = "1.0"`. No other crates. Why: stack.md allowlist (research.md §"Standard
  Stack" — these two are explicitly justified; no others are).
- **D-13**: Sanitize-on-load is mandatory. Sequence: deserialize → call `RecentFiles::sanitize()`
  → return. `sanitize()` drops oversized entries and truncates to RECENT_MAX. Why: hardens
  against tampered recent.json (Pitfall 10).

### Open (executor's discretion within constraints)

- **O-1**: Display format for menu labels. Suggested: `"&{idx} {truncated_path}"` where idx is
  1-9 and 0 for the 10th entry (matching Notepad++/VS Code). Truncation strategy (head, tail,
  or middle ellipsis) is at executor's discretion; constraint is that label width fits within
  ~80 chars and accelerator key (`&N`) is preserved.
- **O-2**: Where to call `recent::load()` at startup. Two viable points: (a) inside `App::new`
  (cleanest — App owns the field) or (b) after `App::new` returns in main.rs run_app (avoids
  Win32 SHGetKnownFolderPath in App::new which currently doesn't depend on COM init order —
  but main.rs has already called OleInitialize by then so either works). Constraint: load
  must happen BEFORE first WM_INITMENUPOPUP, which means BEFORE the first user menu interaction.
- **O-3**: Whether to filter folder paths in the click handler (RECENT-02 says "file"). Recommend
  yes via `path.is_file()` check before calling `open_path_external`; if false, log and remove
  the entry. Constraint: must not crash on stale entries pointing to deleted files.
- **O-4**: Whether to make `do_file_open` (the GUI path dialog) push to recent. Currently
  `do_file_open` calls `open_path_external` at the end; if `open_path_external` is the push
  point, drag-drop (Plan 05-03) and CLI-arg auto-feed Recent Files transitively. Recommended:
  hook in `open_path_external` (single source of truth). Constraint: must not double-push if
  multiple code paths chain.

### Considered and rejected

- **R-1**: `serde_yaml` instead of serde_json. Rejected: not on stack.md allowlist; YAML
  adds ~500KB to binary; no benefit for this trivial schema.
- **R-2**: Read recent.json on every WM_INITMENUPOPUP rebuild. Rejected: redundant disk I/O;
  in-memory `app.recent` is the source of truth between launches; only load once.
- **R-3**: Store full `Tab` snapshots (cursor, selection, scroll position) in recent.json
  to restore session state. Rejected: scope creep; idea.md / requirements.md spec is "last 10
  paths" only — session restore is a v2 feature.
- **R-4**: Use `dirs` crate for AppData path. Rejected: adds dependency outside allowlist;
  windows-rs has the canonical API already and we already depend on `Win32_UI_Shell`.
- **R-5**: PathBuf instead of String. Rejected by D-3.
- **R-6**: Unbounded list. Rejected by D-5 (RECENT-01 says "last 10").

   ```rust
   //! RECENT-01..04 / TEST-06: persisted Most-Recently-Used file list.
   //!
   //! - Capped at 10 entries (RECENT-01)
   //! - Persisted to %APPDATA%/notepadrs/recent.json (RECENT-03)
   //! - Atomic write via tmp+rename (mirror file::save_atomic)
   //! - Graceful fallback to empty on missing/malformed file (RECENT-04, Pitfall 10)
   //! - Sanity cap on path length to harden against tampered recent.json
   //!
   //! Pure-logic helpers (push_lru, sanitize, save_to_path, load_from_path) are split
   //! from the Win32 path-lookup wrapper so tests/recent_files.rs can run without
   //! SHGetKnownFolderPath. Pattern mirrors Phase 2's dispatch_pure.rs.

   #![deny(clippy::unwrap_used)]
   #![deny(clippy::expect_used)]

   use serde::{Serialize, Deserialize};
   use std::path::{Path, PathBuf};
   use crate::error::{Error, Result};

   pub const RECENT_MAX: usize = 10;
   pub const RECENT_PATH_MAX_LEN: usize = 32_768;

   #[derive(Serialize, Deserialize, Default, Debug, Clone, PartialEq, Eq)]
   pub struct RecentFiles {
       /// Most-recently-opened first. Cap of RECENT_MAX. Use `push_lru()` to maintain.
       pub paths: Vec<String>,
   }

   impl RecentFiles {
       pub fn push_lru(&mut self, path: &str) {
           // Case-insensitive dedupe (Windows paths are case-insensitive but preserve user casing).
           self.paths.retain(|p| !p.eq_ignore_ascii_case(path));
           self.paths.insert(0, path.to_string());
           self.paths.truncate(RECENT_MAX);
       }

       /// Discard tampered/oversized entries; truncate to RECENT_MAX.
       /// Called after deserialize to harden against malicious recent.json.
       pub fn sanitize(&mut self) {
           self.paths.retain(|p| p.len() <= RECENT_PATH_MAX_LEN);
           self.paths.truncate(RECENT_MAX);
       }
   }

   /// Resolve %APPDATA%/notepadrs/recent.json via SHGetKnownFolderPath.
   /// KF_FLAG_CREATE creates the notepadrs subfolder if missing.
   pub fn recent_path() -> Result<PathBuf> {
       use windows::Win32::UI::Shell::{SHGetKnownFolderPath, FOLDERID_RoamingAppData, KF_FLAG_CREATE};
       unsafe {
           let pwstr = SHGetKnownFolderPath(&FOLDERID_RoamingAppData, KF_FLAG_CREATE, None)
               .map_err(|e| Error::Misc(format!("SHGetKnownFolderPath: {e}")))?;
           let s = pwstr.to_string()
               .map_err(|e| Error::Misc(format!("recent_path UTF-16 decode: {e}")))?;
           let dir = PathBuf::from(s).join("notepadrs");
           std::fs::create_dir_all(&dir)?;
           Ok(dir.join("recent.json"))
       }
   }

   /// Pure-logic save: serialize → write to tmp → rename. Test-friendly (takes explicit path).
   pub fn save_to_path(rf: &RecentFiles, target: &Path) -> Result<()> {
       let json = serde_json::to_string_pretty(rf)
           .map_err(|e| Error::Misc(format!("recent serialize: {e}")))?;
       let tmp = target.with_extension("json.tmp");
       std::fs::write(&tmp, json.as_bytes())?;
       std::fs::rename(&tmp, target)?;
       Ok(())
   }

   /// Pure-logic load. Returns Default on missing or malformed file (RECENT-04).
   /// `sanitize()` is called automatically on success.
   pub fn load_from_path(target: &Path) -> RecentFiles {
       let text = match std::fs::read_to_string(target) {
           Ok(s) => s,
           Err(_) => return RecentFiles::default(),    // missing file → empty
       };
       let mut rf: RecentFiles = serde_json::from_str(&text).unwrap_or_default(); // malformed → empty
       rf.sanitize();
       rf
   }

   /// Win32 wrapper: resolve AppData path then delegate to save_to_path.
   pub fn save(rf: &RecentFiles) -> Result<()> {
       let target = recent_path()?;
       save_to_path(rf, &target)
   }

   /// Win32 wrapper: resolve AppData path then delegate to load_from_path.
   /// Returns Default if path resolution fails (extreme sandbox case).
   pub fn load() -> RecentFiles {
       match recent_path() {
           Ok(p) => load_from_path(&p),
           Err(_) => RecentFiles::default(),
       }
   }
   ```

   Note `unwrap_used`/`expect_used` deny — use `.unwrap_or_default()` (allowed) or `.map_err(|e| ...)`.

   ```rust
   //! TEST-06: recent-files persistence (RECENT-01..04).
   //!
   //! Pure-logic tests using save_to_path/load_from_path so we don't depend on
   //! SHGetKnownFolderPath being present (works in CI without RoamingAppData).

   use notepadrs::recent::{RecentFiles, RECENT_MAX, RECENT_PATH_MAX_LEN, save_to_path, load_from_path};

   fn tmp_target(name: &str) -> std::path::PathBuf {
       let mut p = std::env::temp_dir();
       p.push(format!("notepadrs_test_recent_{}_{}.json",
           name, std::process::id()));
       p
   }

   #[test]
   fn push_lru_inserts_at_front() {
       let mut rf = RecentFiles::default();
       rf.push_lru("a");
       rf.push_lru("b");
       rf.push_lru("c");
       assert_eq!(rf.paths, vec!["c","b","a"]);
   }

   #[test]
   fn push_lru_dedupes_case_insensitive() {
       let mut rf = RecentFiles::default();
       rf.push_lru(r"C:\Foo\Bar.txt");
       rf.push_lru(r"c:\foo\bar.txt");  // same path, different casing
       assert_eq!(rf.paths.len(), 1);
       assert_eq!(rf.paths[0], r"c:\foo\bar.txt"); // most recent casing wins
   }

   #[test]
   fn push_lru_caps_at_recent_max() {
       let mut rf = RecentFiles::default();
       for i in 0..RECENT_MAX + 5 {
           rf.push_lru(&format!("file_{i}.txt"));
       }
       assert_eq!(rf.paths.len(), RECENT_MAX);
       assert_eq!(rf.paths[0], format!("file_{}.txt", RECENT_MAX + 4));
   }

   #[test]
   fn sanitize_drops_oversized_entries() {
       let mut rf = RecentFiles {
           paths: vec!["short".into(), "x".repeat(RECENT_PATH_MAX_LEN + 1)],
       };
       rf.sanitize();
       assert_eq!(rf.paths, vec!["short"]);
   }

   #[test]
   fn sanitize_truncates_oversized_list() {
       let mut rf = RecentFiles {
           paths: (0..RECENT_MAX + 3).map(|i| format!("p{i}")).collect(),
       };
       rf.sanitize();
       assert_eq!(rf.paths.len(), RECENT_MAX);
   }

   #[test]
   fn save_then_load_roundtrip() {
       let path = tmp_target("roundtrip");
       let mut rf = RecentFiles::default();
       rf.push_lru("a.txt");
       rf.push_lru("b.txt");
       save_to_path(&rf, &path).expect("save");
       let loaded = load_from_path(&path);
       assert_eq!(loaded, rf);
       let _ = std::fs::remove_file(&path);
   }

   #[test]
   fn load_from_missing_file_returns_default() {
       let path = tmp_target("missing");
       let _ = std::fs::remove_file(&path); // ensure missing
       let rf = load_from_path(&path);
       assert_eq!(rf, RecentFiles::default());
   }

   #[test]
   fn load_from_malformed_returns_default() {
       let path = tmp_target("malformed");
       std::fs::write(&path, b"this is not json {[}").expect("write garbage");
       let rf = load_from_path(&path);
       assert_eq!(rf, RecentFiles::default());
       let _ = std::fs::remove_file(&path);
   }

   #[test]
   fn save_does_not_leak_tmp_file() {
       let path = tmp_target("no_tmp_leak");
       let mut rf = RecentFiles::default();
       rf.push_lru("a.txt");
       save_to_path(&rf, &path).expect("save");
       let tmp = path.with_extension("json.tmp");
       assert!(!tmp.exists(), "tmp file should be renamed away");
       assert!(path.exists(), "target should exist");
       let _ = std::fs::remove_file(&path);
   }

   #[test]
   fn load_after_sanitize_caps_to_recent_max() {
       let path = tmp_target("oversized_disk");
       // Write a hand-crafted recent.json with 15 entries (bypass push_lru cap).
       let many = (0..15).map(|i| format!("\"f{i}\"")).collect::<Vec<_>>().join(",");
       let json = format!(r#"{{"paths":[{}]}}"#, many);
       std::fs::write(&path, json).expect("write");
       let rf = load_from_path(&path);
       assert_eq!(rf.paths.len(), RECENT_MAX, "load should sanitize-truncate");
       let _ = std::fs::remove_file(&path);
   }

   #[test]
   fn empty_recent_serializes_minimal_json() {
       let path = tmp_target("empty");
       let rf = RecentFiles::default();
       save_to_path(&rf, &path).expect("save");
   ```

   ```rust
   /// Plan 05-02: rebuild the dynamic Recent Files submenu from the current MRU list.
   /// Called from dispatch.rs WM_INITMENUPOPUP. Clears existing items, re-populates from
   /// `recent.paths`. Empty list → single MF_GRAYED placeholder.
   ///
   /// # Safety
   /// `recent_menu` must be a valid HMENU returned by build_main_menu.
   pub unsafe fn rebuild_recent_submenu(
       recent_menu: HMENU,
       recent: &crate::recent::RecentFiles,
   ) {
       // Clear all existing items.
       let count = GetMenuItemCount(recent_menu);
       for i in (0..count).rev() {
           let _ = DeleteMenu(recent_menu, i as u32, MF_BYPOSITION);
       }
       if recent.paths.is_empty() {
           let _ = AppendMenuW(recent_menu, MF_STRING | MF_GRAYED, 0, w!("(No recent files)"));
           return;
       }
       for (idx, path) in recent.paths.iter().enumerate().take(crate::recent::RECENT_MAX) {
           let label = format!("&{} {}\0",
               if idx < 9 { (idx + 1) as u32 } else { 0 },
               truncate_for_menu(path, 60));
           let label_w: Vec<u16> = label.encode_utf16().collect();
           let cmd = (crate::app::IDM_RECENT_BASE as usize) + idx;
           let _ = AppendMenuW(recent_menu, MF_STRING, cmd, PCWSTR(label_w.as_ptr()));
       }
   }

   fn truncate_for_menu(path: &str, max_chars: usize) -> String {
       if path.chars().count() <= max_chars { path.to_string() }
       else {
           // Middle ellipsis: keep prefix and suffix, drop middle.
           let half = max_chars / 2 - 2;
           let chars: Vec<char> = path.chars().collect();
           let prefix: String = chars[..half].iter().collect();
           let suffix: String = chars[chars.len() - half..].iter().collect();
           format!("{prefix}...{suffix}")
       }
   }
   ```
