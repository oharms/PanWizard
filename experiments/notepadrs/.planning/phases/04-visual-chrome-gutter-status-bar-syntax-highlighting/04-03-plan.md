<objective>
Wire the four status-bar segments to live editor state: row:col, encoding, EOL, line count. Each segment refreshes on the appropriate events — row:col on every cursor move (EN_SELCHANGE) and edit (EN_CHANGE), encoding/EOL on tab switch and menu conversion, line count on every EN_CHANGE. Tab switch refreshes all four together. Add `label_for_status` methods to `DetectedEncoding` and `Eol` so the formatting is one-line at the call site, and add a `format_row_col` pure helper for headless testing. **This plan does NOT mutate `src/editor.rs` — Plan 04-01 already set the full Phase-4 EM_SETEVENTMASK mask (ENM_CHANGE | ENM_SCROLL | ENM_SELCHANGE), so EN_SELCHANGE fires natively without any change here.**

Purpose: Close VIEW-02..05. Notepad++ users instinctively read the status bar for "what encoding is this file" and "what line am I on" — without it the editor feels broken even if the buffer round-trip is correct. Plan 04-01 created the status bar HWND; this plan makes it speak.
Output: Full segment-update helpers in `src/status_bar.rs` (with the P4-5 WONTFIX-v1 documented in the source per Warning #10); `label_for_status` methods on encoding + eol enums; event hooks in `src/dispatch.rs`; pure-logic tests in `tests/status_bar_labels.rs`; one human-verify checkpoint.

  Why: matches user expectation that row:col is "live" while encoding/EOL only changes when the user converts. Source: 04-research.md Pattern S1 §"Update cadence".

```rust
use crate::app::App;
use windows::Win32::UI::Controls::RichEdit::{
    EM_GETLINECOUNT, EM_GETSEL, EM_LINEFROMCHAR, EM_LINEINDEX,
};

/// Pure-logic helper: format row:col for the status bar. Inputs are 1-based.
pub fn format_row_col(line: u32, col: u32) -> String {
    format!("Ln {line}, Col {col}")
}

/// Update segment 0 (row:col) by querying EM_GETSEL on the active tab's RichEdit.
/// Costs ~3 SendMessageW calls; safe to call on every EN_CHANGE / EN_SELCHANGE.
///
/// WONTFIX-v1 (Pitfall P4-5): `col` counts UTF-16 code units from line start,
/// not graphemes and not visual columns with tab expansion. VS Code uses the
/// same convention; Notepad++ uses visual columns. The toggle is deferred to
/// v1.x. See `.planning/phases/04-visual-chrome-gutter-status-bar-syntax-highlighting/04-research.md`
/// Pitfall P4-5 for the full rationale (test impact, user-expectation matrix,
/// emoji edge case).
pub unsafe fn update_row_col_segment(app: &App) {
    if app.hstatus.is_invalid() { return; }
    let Some(tab) = app.tabs.get(app.active_tab) else { return; };
    let hwnd_re = tab.editor.hwnd_re;
    if hwnd_re.is_invalid() { return; }

    // EM_GETSEL via WPARAM/LPARAM out-pointers (UTF-16 code unit offsets).
    let mut sel_start: i32 = 0;
    let mut sel_end: i32 = 0;
    let _ = SendMessageW(
        hwnd_re,
        EM_GETSEL,
        Some(WPARAM(&mut sel_start as *mut _ as usize)),
        Some(LPARAM(&mut sel_end as *mut _ as isize)),
    );
    // Caret position is sel_end (the active end of the selection).
    let line0 = SendMessageW(
        hwnd_re,
        EM_LINEFROMCHAR,
        Some(WPARAM(sel_end as usize)),
        Some(LPARAM(0)),
    ).0 as i32;
    let line_start = SendMessageW(
        hwnd_re,
        EM_LINEINDEX,
        Some(WPARAM(line0 as usize)),
        Some(LPARAM(0)),
    ).0 as i32;
    let col0 = (sel_end - line_start).max(0);

    // 1-based for display per D-2.
    let label = format_row_col((line0 + 1) as u32, (col0 + 1) as u32);
    set_segment(app.hstatus, 0, &label);
}

/// Update segment 1 (encoding) — reads from active tab's editor.encoding.
pub unsafe fn update_encoding_segment(app: &App) {
    if app.hstatus.is_invalid() { return; }
    let Some(tab) = app.tabs.get(app.active_tab) else { return; };
    set_segment(app.hstatus, 1, tab.editor.encoding.label_for_status());
}

/// Update segment 2 (EOL) — reads from active tab's editor.eol.
pub unsafe fn update_eol_segment(app: &App) {
    if app.hstatus.is_invalid() { return; }
    let Some(tab) = app.tabs.get(app.active_tab) else { return; };
    set_segment(app.hstatus, 2, tab.editor.eol.label_for_status());
}

/// Update segment 3 (line count) — EM_GETLINECOUNT on active tab's RichEdit.
pub unsafe fn update_line_count_segment(app: &App) {
    if app.hstatus.is_invalid() { return; }
    let Some(tab) = app.tabs.get(app.active_tab) else { return; };
    let hwnd_re = tab.editor.hwnd_re;
    if hwnd_re.is_invalid() { return; }
    let n = SendMessageW(hwnd_re, EM_GETLINECOUNT, None, None).0 as u32;
    set_segment(app.hstatus, 3, &format!("{n} lines"));
}

/// Update ALL four segments. Called on tab switch, file open, file save, IDM_ENCODING/EOL_* commands.
pub unsafe fn update_all_segments(app: &App) {
    update_row_col_segment(app);
    update_encoding_segment(app);
    update_eol_segment(app);
    update_line_count_segment(app);
}
```
