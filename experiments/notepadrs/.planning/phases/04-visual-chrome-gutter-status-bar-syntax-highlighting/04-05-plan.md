- D-3: **In-source `&str` fixtures, NOT external files.** Each test inlines its input as a Rust string literal. Why: simpler than `tests/fixtures/syntax/...`; no fixture-loading helper needed; the test's input is right there for the reader. Source: 04-research.md Pattern V1 §"Where tests live".
- D-4: **Test what's CONTRACTUAL, not implementation details.** Assertions test:
  - **Token positions:** the byte offsets `start..end` slice the line correctly to the expected substring.
  - **Token classes:** `TokenClass::Keyword`, `TokenClass::String`, etc.
  - **LineEndState transitions:** for multi-line languages, that the state returned from line N is the right one to pass into line N+1.

  - Plain has no multi-line state — same.

//! Plan 04-05: per-language syntax tokenizer tests (TEST-07).
//!
//! Single-file integration test with sections per language. Tests assert:
//!   - Token positions (start..end byte offsets) slice the input correctly.
//!   - Token classes match expected.
//!   - LineEndState transitions correctly carry across line boundaries for
//!     stateful languages (JS block comment, Python triple string, MD fenced code).
//!
//! All tests run on every `cargo test` — no `#[ignore]` gating.
//!
//! Plan boundary: depends on Plan 04-04a (tokenizers) only — does NOT reference
//! Plan 04-04b's GUI/highlight pipeline. Runs in Wave 4 in parallel with 04-04b.

use notepadrs::syntax::{
    tokenize_line, Language, LineEndState, PyTripleKind, Token, TokenClass,

// ============================================================
// Helpers

/// Find the first token whose slice equals `expected_str`. Returns its index in the
/// token vec. Helps tests that don't care about other tokens (e.g., whitespace handling).
fn find_token<'a>(line: &str, tokens: &'a [Token], expected_str: &str) -> Option<&'a Token> {
    tokens
        .iter()
        .find(|t| &line[t.start as usize..t.end as usize] == expected_str)
}

/// Assert that a token spans `expected_str` and has the given class.
fn assert_token(line: &str, tok: &Token, expected_str: &str, expected_class: TokenClass) {
    let slice = &line[tok.start as usize..tok.end as usize];
    assert_eq!(slice, expected_str, "token slice mismatch");
    assert_eq!(
        tok.class, expected_class,
        "token class mismatch for slice {:?} (got {:?})",
        slice, tok.class
    );
}
