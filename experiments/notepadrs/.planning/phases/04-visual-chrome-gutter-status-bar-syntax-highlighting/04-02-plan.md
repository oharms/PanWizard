<objective>
Implement the per-tab line-number gutter: paint right-aligned 1-indexed line numbers in the same monospace font as the RichEdit editor, scroll the gutter in lockstep with RichEdit's vertical scroll via `EN_VSCROLL`, and grow the gutter's width automatically when the line count crosses a digit boundary (10, 100, 1000, 10000). Tab switch shows/hides the active tab's gutter alongside its RichEdit.

Purpose: Close VIEW-01 — every tab gets a gutter on the left edge that scrolls with the text and grows with line count, matching the canonical Win32 editor pattern (Notepad++ via Scintilla, every Win32 IDE that doesn't custom-render). This is also a small, focused validation of the sibling-child + scroll-lockstep + EM_NOTIFY pattern that Plan 04-04b will reuse for the highlight viewport.
Output: Full `src/gutter.rs` WM_PAINT implementation; `src/dispatch.rs` updates for EN_VSCROLL / EN_CHANGE / TCN_SELCHANGE handlers; `src/app.rs` updated to add `em_width_px: i32`; pure-logic tests in `tests/gutter_layout.rs`; one human-verify checkpoint after auto work completes. **This plan does NOT mutate `src/editor.rs` — Plan 04-01 already set the full Phase-4 EM_SETEVENTMASK mask (ENM_CHANGE | ENM_SCROLL | ENM_SELCHANGE), so EN_VSCROLL fires natively without any change here.**
</objective>

### Locked (executor MUST follow)

- D-1: **Gutter `WM_PAINT` reads from sibling RichEdit via GWLP_USERDATA.** Plan 04-01 stashed `editor.hwnd_re.0` on the gutter's GWLP_USERDATA at create time. WM_PAINT recovers it via `GetWindowLongPtrW(hwnd, GWLP_USERDATA)`, then queries `EM_GETFIRSTVISIBLELINE` and `EM_GETLINECOUNT` on it. Why: keeps the gutter self-contained (no App-pointer plumbing into a generic class WndProc); the GWLP_USERDATA round-trip is the standard Win32 idiom (project-research.md Pattern 1). Source: 04-research.md Code Examples §"Gutter WM_PAINT (sketch)".
- D-2: **Use the editor's font for gutter text.** Send `WM_GETFONT` to the sibling RichEdit, `SelectObject` it into the gutter's HDC for the paint pass, then restore. Why: monospace + same metrics = lines align pixel-perfect with the text; no font duplication. Source: 04-research.md "Don't Hand-Roll" — line height + 04-research.md Code Examples §gutter sketch.
- D-3: **Right-align line numbers with `DrawTextW(DT_RIGHT | DT_SINGLELINE | DT_VCENTER)`.** Each line's rect is `RECT { left: 0, top: i*line_height, right: client_width - GUTTER_PADDING_PX, bottom: (i+1)*line_height }`. Why: right alignment matches every editor convention (numbers grow leftward as line count grows); 4px right padding visually separates the gutter from the editor text. Source: 04-research.md Code Examples §gutter sketch.
- D-4: **Lockstep scrolling via EN_VSCROLL — Plan 04-01 already enabled ENM_SCROLL in `EM_SETEVENTMASK`.** This plan does NOT modify `src/editor.rs`. EN_VSCROLL is delivered to the parent via `WM_NOTIFY` because Plan 04-01 set the mask to `(ENM_CHANGE | ENM_SCROLL | ENM_SELCHANGE)`. The handler in `src/dispatch.rs` watches for `NMHDR.code == EN_VSCROLL` and `InvalidateRect(tab.gutter_hwnd, None, FALSE)` to repaint. Why: WM_VSCROLL alone misses keyboard scrolls (Pitfall P4-2); `EN_VSCROLL` is the documented RichEdit notification for any scroll source. Source: 04-research.md Pattern G1 §"Lockstep scrolling mechanism", Pitfall P4-2; plan-checker Blocker #2 fix (no editor.rs collision).
- D-5: **Dynamic gutter width — recompute on EN_CHANGE when digit count changes.** In the existing `EN_CHANGE` branch (Plan 03-02 wired this for dirty-flag), add: `let n = SendMessageW(editor.hwnd_re, EM_GETLINECOUNT, ...).0 as u32; let new_width = gutter_width_for_line_count(n, em_width); if new_width != tab.gutter_width { tab.gutter_width = new_width; tab.last_line_count = n; relayout_active_tab(app, hwnd); }`. Why: avoids paying the `MoveWindow` cost on every keystroke (only relayout when crossing a digit boundary, e.g., line count goes from 999 to 1000); pure-logic helper enables headless testing. Source: 04-research.md Pattern G1 §"Width-grows-with-line-count algorithm".
- D-6: **`gutter_width_for_line_count(n: u32, em_width_px: i32) -> i32` is the pure-logic test surface.**

//! Gutter — per-tab line-number child window.
//!
//! Plan 04-01 registered the class and provided a stub. Plan 04-02 fills in
//! WM_PAINT (right-aligned line numbers, monospace font from sibling RichEdit,
//! lockstep with EM_GETFIRSTVISIBLELINE).

#![deny(clippy::unwrap_used)]
#![deny(clippy::expect_used)]

use windows::core::*;
use windows::Win32::Foundation::*;
use windows::Win32::Graphics::Gdi::*;
use windows::Win32::System::LibraryLoader::*;
use windows::Win32::UI::Controls::RichEdit::EM_GETFIRSTVISIBLELINE;
use windows::Win32::UI::Controls::RichEdit::EM_GETLINECOUNT;
use windows::Win32::UI::WindowsAndMessaging::*;

use crate::error::Result;

pub const GUTTER_CLASS_NAME: PCWSTR = w!("NotepadrsGutter");
pub const GUTTER_PADDING_PX: i32 = 8;
pub const GUTTER_DEFAULT_WIDTH: i32 = 40;
pub const GUTTER_TEXT_COLOR: COLORREF = COLORREF(0x808080); // 50% grey

/// Pure-logic gutter width calculator. Returns the px width needed to display
/// line numbers up to `n` in a monospace font with `em_width_px` per character,
/// plus padding on both sides.
///
/// At least 1 digit is always rendered (n=0 → 1 digit space).
pub fn gutter_width_for_line_count(n: u32, em_width_px: i32) -> i32 {
    let digits: i32 = if n < 10 { 1 }
        else if n < 100 { 2 }
        else if n < 1_000 { 3 }
        else if n < 10_000 { 4 }
        else if n < 100_000 { 5 }
        else if n < 1_000_000 { 6 }
        else { 7 };
    digits * em_width_px + 2 * GUTTER_PADDING_PX
}

pub unsafe fn register_gutter_class() -> Result<u16> {
    let hinstance: HINSTANCE = GetModuleHandleW(None)?.into();
    let wc = WNDCLASSW {
        style: CS_HREDRAW | CS_VREDRAW,
        lpfnWndProc: Some(gutter_wnd_proc),
        cbClsExtra: 0,
        cbWndExtra: 0,
        hInstance: hinstance,
        hIcon: HICON::default(),
        hCursor: LoadCursorW(None, IDC_ARROW)?,
        hbrBackground: HBRUSH((COLOR_BTNFACE.0 + 1) as *mut _),
        lpszMenuName: PCWSTR::null(),
        lpszClassName: GUTTER_CLASS_NAME,
    };
    let atom = RegisterClassW(&wc);
    if atom == 0 {
        return Err(Error::from_win32().into());
    }
    Ok(atom)
}

unsafe extern "system" fn gutter_wnd_proc(
    hwnd: HWND,
    msg: u32,
    wparam: WPARAM,
    lparam: LPARAM,
) -> LRESULT {
    if msg == WM_PAINT {
        paint_gutter(hwnd);
        return LRESULT(0);
    }
    DefWindowProcW(hwnd, msg, wparam, lparam)
}

/// WM_PAINT handler for the gutter.
unsafe fn paint_gutter(hwnd: HWND) {
    let mut ps = PAINTSTRUCT::default();
    let hdc = BeginPaint(hwnd, &mut ps);
    if hdc.is_invalid() {
        let _ = EndPaint(hwnd, &ps);
        return;
    }

    // Recover the sibling RichEdit's HWND from GWLP_USERDATA (set in Tab::new_empty).
    let hwnd_re_raw = GetWindowLongPtrW(hwnd, GWLP_USERDATA);
    if hwnd_re_raw == 0 {
        // Not yet wired (could happen in early WM_PAINT before the userdata is set).
        let _ = EndPaint(hwnd, &ps);
        return;
    }
    let hwnd_re = HWND(hwnd_re_raw as *mut _);

    // Pull the editor font and select it into our DC.
    let hfont = SendMessageW(hwnd_re, WM_GETFONT, None, None).0;
    let prev_font = if hfont != 0 {
        SelectObject(hdc, HGDIOBJ(hfont as *mut _))
    } else {
        HGDIOBJ::default()
    };

    // Compute line-height from the selected font.
    let mut tm = TEXTMETRICW::default();
    let line_h = if GetTextMetricsW(hdc, &mut tm).is_ok() {
        (tm.tmHeight + tm.tmExternalLeading).max(1)
    } else {
        16
    };

    // First visible line in the sibling RichEdit (EM_GETFIRSTVISIBLELINE: O(1)).
    let first = SendMessageW(hwnd_re, EM_GETFIRSTVISIBLELINE, None, None).0 as i32;
    let total = SendMessageW(hwnd_re, EM_GETLINECOUNT, None, None).0 as i32;

    // How many rows fit in our client rect.
    let mut rc = RECT::default();
    let _ = GetClientRect(hwnd, &mut rc);
    let visible_rows = ((rc.bottom - rc.top) / line_h) + 1;

    // Paint setup.
    let _ = SetTextColor(hdc, GUTTER_TEXT_COLOR);
    let _ = SetBkMode(hdc, TRANSPARENT);

    for i in 0..visible_rows {
        let line_no = first + i + 1; // 1-indexed display
        if line_no > total {
            break;
        }
        let mut line_rc = RECT {
            left: rc.left,
            top: rc.top + i * line_h,
            right: rc.right - 4, // 4px right margin between number and editor edge
            bottom: rc.top + (i + 1) * line_h,
        };
        let s = format!("{}", line_no);
        let mut s_w: Vec<u16> = s.encode_utf16().collect();
        DrawTextW(
            hdc,
            &mut s_w,
            &mut line_rc,
            DT_RIGHT | DT_SINGLELINE | DT_VCENTER | DT_NOPREFIX,
        );
    }

    // Restore font.
    if !prev_font.is_invalid() {
        SelectObject(hdc, prev_font);
    }
    let _ = EndPaint(hwnd, &ps);
}

fn em_width_scales_linearly() {
