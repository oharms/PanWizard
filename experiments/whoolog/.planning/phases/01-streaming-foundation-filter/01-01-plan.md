  <name>Task 2: Implement lib/source.js (multi-file + glob + stdin reader)</name>
  <files>lib/source.js</files>
  <action>
Create `lib/source.js` exporting four functions per the `<interfaces>` contract:

1. **`toPosix(p)`** — replace all backslashes with forward slashes. One-liner. Used for output messages on Windows. Do NOT use `path.posix` for actual fs operations.

2. **`async function* expandGlob(pattern)`** — handles single-segment globs only (`logs/*.jsonl`, NOT `**/*.jsonl`):
   - If `pattern` contains no `*`, yield it verbatim (literal file path).
   - Otherwise: split into `dir = path.dirname(pattern) || '.'` and `base = path.basename(pattern)`.
   - Build a regex from `base` by replacing `*` with `[^/\\]*` and escaping other regex metacharacters. Anchor with `^...$`.
   - Call `fs.promises.readdir(dir, { withFileTypes: true })`. Do NOT mix with `recursive: true` (research/stack.md: known bug in Node 18.x).
   - For each entry that is a file and matches the regex, yield `path.join(dir, ent.name)`.
   - On readdir failure (ENOENT, EACCES): write `whoolog: cannot read directory ${toPosix(dir)}: ${err.message}\n` to stderr and exit 1.

3. **`async function* sources({ files })`** — top-level source iterator:
   - If `files` is a non-empty array: for each pattern, `for await (const file of expandGlob(pattern))`:
     - `await fs.promises.stat(file)` first — if it throws ENOENT/EACCES, write `whoolog: file not found: ${toPosix(file)}\n` to stderr and `process.exit(1)`. (Pitfall 24.)
     - yield `{ stream: fs.createReadStream(file), label: toPosix(file) }`.
     - Attach `stream.on('error', ...)` to convert any subsequent ENOENT/EACCES during read to the same exit-1 message (research/stack.md: errors on the underlying stream are NOT forwarded to the readline async iterator — they get swallowed if uncaught).
   - If `files` is empty/undefined and `process.stdin.isTTY` is falsy (i.e., stdin is piped or redirected): yield `{ stream: process.stdin, label: '<stdin>' }`. Do NOT yield if stdin is a TTY — caller (filter.js) handles the help-on-empty-TTY case (SRC-05).

4. **`async function* lines(source)`** — per-source line iterator:
   - `const rl = readline.createInterface({ input: source.stream, crlfDelay: Infinity })`. The `crlfDelay: Infinity` is non-negotiable (Pitfall 3 + research/stack.md).
   - CRITICAL: enter the `for await (const line of rl)` loop in the same synchronous tick as `createInterface` (research/stack.md issue #33463). Do NOT `await` between create and loop entry.
   - Maintain a per-call `lineno` counter (1-indexed, increments BEFORE yielding so the user sees `:1` for the first line).
   - Yield `{ line, lineno, file: source.label }` for every line (do NOT skip blanks here — `decode.js` skips them, keeping the line counter accurate for malformed-line messages).

**Anti-pattern check:**
- NO `fs.readFileSync`, NO `fs.readFile`, NO buffering. The reader must be 100% streaming (Pitfall 1 + research/architecture.md anti-pattern 1).
- NO `Promise.all(files.map(...))` — process files sequentially per Pitfall 5 (fd exhaustion).
- NO `path.posix.join` for actual fs work — only `toPosix(p)` for output.
- NO `glob`/`fast-glob`/etc — manual `fs.readdir` only (zero-dep constraint).

Module exports: `module.exports = { sources, lines, toPosix, expandGlob };`
  </action>

<task type="auto">
  <name>Task 3: Implement lib/decode.js (JSON parse + malformed-line policy)</name>
  <files>lib/decode.js</files>
  <action>
Create `lib/decode.js` with one exported async generator:

```js
'use strict';

async function* decode(linesIter, opts = {}) {
  for await (const { line, lineno, file } of linesIter) {
    if (line.trim() === '') continue;       // skip blank lines silently
    try {
      yield { obj: JSON.parse(line), file, lineno };
    } catch (err) {
      const action = opts.strict ? 'malformed' : 'skipping malformed';
      process.stderr.write(`whoolog: ${action} line at ${file}:${lineno}\n`);
      if (opts.strict) process.exit(1);
      // else: continue
    }
  }
}

module.exports = { decode };
```

**Locked details:**
- The exact stderr message format `whoolog: skipping malformed line at <file>:<lineno>` is locked by requirements.md DEC-02 — copy verbatim.
- The strict-mode message format must include the same `<file>:<lineno>` so users can locate the bad line (Pitfall 20).
- Empty/blank lines are skipped silently — they are not malformed (research/architecture.md decode example does the same).
- Line counter passes through from `source.lines()` — do NOT re-count.

**Anti-pattern check:**
- Do NOT include a `--strict` exception path that re-throws and unwinds the for-await — `process.exit(1)` is correct (Pitfall 20).
- Do NOT swallow non-SyntaxError exceptions from JSON.parse — but JSON.parse only throws SyntaxError, so a single try/catch is correct.
- Do NOT include the parser error message in the stderr line — requirements.md locks the format. (You can include it as additional context on a SECOND line if desired, but the first line must match the exact format.)

Module exports: `module.exports = { decode };`
  </action>
