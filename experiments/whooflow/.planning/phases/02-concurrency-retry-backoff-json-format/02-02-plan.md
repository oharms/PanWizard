<objective>
Add retry-with-bounded-jittered-backoff as a thin wrapper around `runTask`. The retry loop lives in a new `src/retry.js` (~50 LOC) so its math is unit-testable in isolation. The scheduler swaps its single `runTask(task)` call for `runTaskWithRetry(task, opts)` and supplies a `persistAttempt(n)` closure that captures `state`, `id`, and `stateFile` — keeping retry.js free of state.js and scheduler.js imports (clean test seam, no circular deps).

Purpose: Phase 2's retry layer must satisfy three subtle constraints: (1) `attempts: N` means N total runs, not 1+N (off-by-one defended at the loop bound); (2) the incremented attempts counter must be persisted to state BEFORE any backoff sleep so a crash during sleep doesn't enable infinite retries on resume (EXEC-05); (3) at `--concurrency 1`, the slot must be HELD during backoff sleep, otherwise the scheduler sees "0/1 slots used, no work running" — a deadlock-shaped semantic (pitfall M2 deadlock-at-N=1 corollary, EXEC-04). The slot-hold is naturally satisfied because `runTaskWithRetry` is awaited inside the scheduler's existing `.then` continuation chain — `running.add(id)` (Plan 01's scheduler line 86) doesn't get freed until the outer `.then` fires after the FINAL retry resolves.

Output: `src/retry.js` exporting `runTaskWithRetry`, scheduler now uses it, and a 5-task linear chain at concurrency=1 with a middle retry-twice-then-succeed task completes correctly with `attempts: 3` in the final state.
</objective>

Create `src/retry.js` with the contract specified in the interfaces block. The implementation must satisfy:

1. **Off-by-one defense (EXEC-03 + pitfall C7).** Loop bound is `for (let attempt = 1; attempt <= attempts; attempt++)`. `attempts: 1` (default) → 1 spawn, 0 sleeps. `attempts: 2` → up to 2 spawns, up to 1 sleep. `attempts: 3` → up to 3 spawns, up to 2 sleeps. **Sleep happens BETWEEN attempts, never after the last** — the line `if (attempt === attempts) return lastResult;` defends this.

2. **Capped jittered backoff (pitfall C7).** Formula: `delay = min(MAX_BACKOFF_MS, base * 2^(attempt-1)) * (0.5 + random() * 0.5)`. Equal jitter (range [0.5x, 1.0x]) chosen over full jitter — preserves meaningful spread without pushing some retries to instant. `MAX_BACKOFF_MS = 30_000` constant. Source: AWS Architecture Blog on Exponential Backoff and Jitter.

3. **Persist-before-sleep (EXEC-05 + pitfall C7).** `opts.persistAttempt(attempt)` is called BEFORE `runTask` and BEFORE any sleep, on EVERY iteration including the first. This means `markRunning` is called once per attempt and the state file shows the attempt count BEFORE any sleep starts. A crash during sleep is recoverable (Phase 3 resume sees the right `attempts` value).

4. **Slot-hold semantic (EXEC-04 + pitfall M2).** `runTaskWithRetry` returns ONE Promise that resolves with the final result. The scheduler awaits this Promise inside its existing `.then` continuation. `running.add(id)` happens before the awaited call (scheduler line 86); `running.delete(id)` happens after (scheduler line 102). Slot is held throughout the entire retry sequence including all sleeps. **The retry loop must NOT have any code path that releases the slot to the scheduler between attempts.** Architecturally, this is structural — retry.js doesn't even know about the running Set.

5. **Injectable timer + RNG (test determinism, pitfall M4).** `opts.sleepFn` defaults to a real-timer-based promise; tests pass a fake that resolves immediately and records call args. `opts.randomFn` defaults to `Math.random`; tests pass `() => 0.5` for deterministic jitter midpoint.

6. **AbortSignal scaffold (Phase 3 prep).** `opts.signal` is accepted as a parameter; if provided, the default sleep checks `signal.aborted` and rejects. Phase 2's CLI does NOT pass an active signal — the parameter exists for shape only. Phase 3 will wire the actual signal in.

7. **No state.js / scheduler.js imports.** retry.js only imports `runTask` from `./executor.js`. State persistence happens via the `persistAttempt` callback supplied by the scheduler.

```js
// Phase 2 Plan 02 — retry-with-bounded-jittered-backoff wrapper around runTask.
//
// Sits between scheduler and executor: scheduler.js calls runTaskWithRetry, which calls
// runTask N times (where N <= task.retry.attempts). The scheduler sees ONE ok|fail outcome
// per task — per-attempt failures never reach the scheduler's event stream. This keeps the
// scheduler's DAG/topology code free of retry concerns (architecture.md Pattern 4) and
// retry.js free of state.js / scheduler.js imports (clean test seam).
//
// Pitfall defenses:
//   C7 (DoS retry storm): capped at MAX_BACKOFF_MS=30_000 + equal-jitter (0.5..1.0x).
//   C7 (off-by-one):      attempts: N means N TOTAL runs. Loop is for(1..N), sleep skipped
//                         after the last attempt.
//   C7 (attempts not persisted): opts.persistAttempt(n) is called BEFORE runTask AND BEFORE
//                                any sleep on every iteration. A crash during sleep is
//                                recoverable on Phase 3 resume.
//   M2 (slot-hold at concurrency=1): runTaskWithRetry is a SINGLE awaited Promise to the
//                                    scheduler. running.add/delete bracket the await.
//                                    No code path here releases the slot — the slot-hold
//                                    is structural, not policy.
//   M4 (test flakiness): opts.sleepFn and opts.randomFn injectable. Tests pin both.

import { runTask } from './executor.js';

export const MAX_BACKOFF_MS = 30_000;

/**
 * runTaskWithRetry(task, opts) -> Promise<{ ok, exit_code, signal, error? }>
 *
 * @param {Object} task - normalized task with { id, cmd, retry: {attempts, backoff_ms} }
 * @param {Object} opts
 * @param {(attemptNumber: number) => void} opts.persistAttempt
 *        Scheduler-supplied callback that increments task.attempts in state and persists
 *        atomically. Called BEFORE each runTask invocation (including the first) — see
 *        EXEC-05 / pitfall C7.
 * @param {(ms: number, signal?: AbortSignal) => Promise<void>} [opts.sleepFn=defaultSleep]
 *        Injectable for tests; real impl uses setTimeout with optional AbortSignal.
 * @param {() => number} [opts.randomFn=Math.random]
 *        Injectable for tests; real impl uses Math.random for equal-jitter.
 * @param {AbortSignal} [opts.signal]
 *        P2: scaffold-only, no CLI wires it. P3: will be wired to a SIGINT-triggered
 *        AbortController in cli.js.
 * @returns {Promise<{ok, exit_code, signal, error?}>} same shape as runTask
 */
export async function runTaskWithRetry(task, opts) {
  const attempts = task.retry?.attempts ?? 1;
  const baseMs = task.retry?.backoff_ms ?? 0;
  const persistAttempt = opts.persistAttempt;
  const sleep = opts.sleepFn ?? defaultSleep;
  const random = opts.randomFn ?? Math.random;

  let lastResult = null;
  for (let attempt = 1; attempt <= attempts; attempt++) {
    // EXEC-05 / pitfall C7: persist BEFORE runTask AND BEFORE any sleep.
    // markRunning (called inside persistAttempt) increments attempts to `attempt`.
    persistAttempt(attempt);

    lastResult = await runTask(task);

    if (lastResult.ok) return lastResult;             // success short-circuit

    // Off-by-one defense: don't sleep after the LAST attempt — return the failure.
    if (attempt === attempts) return lastResult;

    // Capped exponential backoff with equal jitter.
    // delay = min(MAX, base * 2^(attempt-1)) * (0.5 + random*0.5)
    const capped = Math.min(MAX_BACKOFF_MS, baseMs * Math.pow(2, attempt - 1));
    const jittered = Math.floor(capped * (0.5 + random() * 0.5));
    await sleep(jittered, opts.signal);
  }

  // Unreachable in normal flow — the loop body always returns.
  return lastResult;
}

function defaultSleep(ms, signal) {
  return new Promise((resolve, reject) => {
    if (signal?.aborted) return reject(new Error('aborted'));
    const id = setTimeout(resolve, ms);
    signal?.addEventListener(
      'abort',
      () => {
        clearTimeout(id);
        reject(new Error('aborted'));
      },
      { once: true },
    );
  });
}
```

```js
function pump() {
  while (running.size < cap && ready.length > 0) {
    const id = ready.shift();
    running.add(id);

    // Per-task wall-clock timer spans ALL retries.
    const startedNs = process.hrtime.bigint();

    const task = flow.tasks[id];

    // Closure captures state, id, stateFile. Called by retry.js BEFORE each runTask
    // (including the first attempt). markRunning increments attempts by 1 each call.
    // EXEC-05 / pitfall C7: this MUST run before any sleep in retry.js.
    function persistAttempt(attemptNumber) {
      // markRunning increments attempts internally (+1 each call). After this call,
      // state.tasks[id].attempts === attemptNumber. We do NOT pass attemptNumber to
      // markRunning; the helper increments by 1 per call, and retry.js calls us once
      // per attempt in order 1, 2, ..., N.
      markRunning(state, id);
      atomicWriteJson(stateFile, state);

      // Emit a 'running' event ONLY on the first attempt — subsequent attempts are
      // internal retry transitions that the scheduler should not surface. This matches
      // architecture.md Pattern 4 (scheduler sees one ok|fail per task; retries are
      // transparent). The 'running' event for attempt 1 marks the task's overall start.
      if (attemptNumber === 1) {
        onEvent({
          type: 'task',
          id,
          status: 'running',
          attempts: state.tasks[id].attempts,
          seq: nextSeq(),
        });
      }
    }

    runTaskWithRetry(task, { persistAttempt })
      .then((result) => {
        running.delete(id);
        const durationNs = process.hrtime.bigint() - startedNs;

        if (result.ok) {
          markSuccess(state, id, result.exit_code);
          atomicWriteJson(stateFile, state);
          summary.success++;
          onEvent({
            type: 'task',
            id,
            status: 'success',
            attempts: state.tasks[id].attempts,
            exit_code: result.exit_code,
            duration_ns: durationNs,
            seq: nextSeq(),
          });
          for (const c of flow.children[id]) {
            if (--indegree[c] === 0) ready.push(c);
          }
        } else {
          markFailed(state, id, result.exit_code, result.signal);
          atomicWriteJson(stateFile, state);
          summary.failed++;
          onEvent({
            type: 'task',
            id,
            status: 'failed',
            attempts: state.tasks[id].attempts,
            exit_code: result.exit_code,
            duration_ns: durationNs,
            seq: nextSeq(),
          });
        }

        pump();
        maybeFinish();
      })
      .catch((err) => {
        // runTaskWithRetry should not reject — defensive fallback.
        running.delete(id);
        markFailed(state, id, -1, null);
        atomicWriteJson(stateFile, state);
        summary.failed++;
        onEvent({
          type: 'task',
          id,
          status: 'failed',
          attempts: state.tasks[id].attempts,
          exit_code: -1,
          error: err && err.message,
          seq: nextSeq(),
        });
        pump();
        maybeFinish();
      });
  }
  if (running.size === 0 && ready.length === 0) {
    maybeFinish();
  }
}
```
