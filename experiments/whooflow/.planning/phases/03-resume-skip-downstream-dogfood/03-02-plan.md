<objective>
Create `src/skip.js` exporting a single function `propagateSkip(failedId, flow, state, indegree, onEvent, nextSeq)` that performs a BFS from a terminally-failed task over the readiness graph, marking each transitive descendant as `skipped` and decrementing grandchildren's indegrees so independent branches don't deadlock. The function is a pure mutation over its arguments — no I/O, no scheduler imports, no spawn. Returns the count of newly-skipped tasks so the caller can update its summary.

Purpose: SCHED-06 says skip-downstream propagates only on terminal failure (not on individual attempt failures). By isolating the BFS in its own module that takes the state + indegree map by reference, we keep `scheduler.js`'s failure branch as a one-line call — no inline graph walk, no embedded queue. The C2 deadlock prevention is structural: every time we mark a task `skipped`, we MUST decrement its children's indegrees in the same loop iteration, and `propagateSkip` is the single function that owns this contract.

The M1 ordering guarantee (skip events fire AFTER the parent's terminal fail event) is achieved by the caller: Plan 04 will call `propagateSkip` inside the `result.ok === false` branch of `runTaskWithRetry.then()` AFTER `markFailed + atomicWriteJson + onEvent({type:'task', status:'failed'})` have already executed. Since `propagateSkip` runs synchronously within that `.then()` continuation and emits its events synchronously, the JS microtask discipline guarantees the fail event precedes any skip events for the same parent.

Output: A new pure module `src/skip.js` (~50 LOC) exporting `propagateSkip` and a comprehensive test file `test/skip.test.js` with ≥5 unit tests using a fake flow + state + indegree + a captureEvents callback. No changes to scheduler.js — that's Plan 04's domain.

```js
// Phase 3 Plan 02 — skip-downstream BFS over reverse edges.
//
// SCHED-06: when a task reaches TERMINAL failed (after all retries exhausted),
// the scheduler propagates 'skipped' status to every transitive descendant.
// Independent branches keep running.
//
// Pitfall C2: each newly-skipped task's children's indegrees MUST be decremented
// here — without this, an independent task whose depends_on includes a skipped
// task would never become ready, and the scheduler would deadlock.
//
// Pitfall M1 ordering: skip events fire AFTER the parent's terminal 'failed'
// event. This is the caller's responsibility — propagateSkip is invoked inside
// the failure branch of runTaskWithRetry.then() AFTER markFailed + the fail
// event have been emitted. JS microtask discipline keeps the order intact.
//
// Diamond-convergence safety: tasks already in 'success', 'failed', or 'skipped'
// status are NOT re-marked. Only 'pending' tasks become 'skipped'. (A diamond
// node reached from one path that succeeded and one that failed keeps its
// 'success' state.)

import { markSkipped } from './state.js';

/**
 * propagateSkip(failedId, flow, state, indegree, onEvent, nextSeq) -> number
 *
 * BFS from failedId over flow.children (forward DAG edges). Marks each
 * pending descendant as 'skipped' and decrements grandchildren's indegrees.
 *
 * @param {string}   failedId   BFS root — the task that just terminally failed
 * @param {Object}   flow       validatedFlow with flow.children
 * @param {Object}   state      state object (state.tasks mutated in place)
 * @param {Object}   indegree   indegree map (mutated: grandchildren decremented)
 * @param {Function} onEvent    (event) => void — fires once per skipped task
 * @param {Function} nextSeq    () => number — monotonic seq counter
 * @returns {number}            count of newly-skipped tasks
 */
export function propagateSkip(failedId, flow, state, indegree, onEvent, nextSeq) {
  const queue = [failedId];
  const visited = new Set([failedId]);
  let skippedCount = 0;
  const reason = `upstream ${failedId} failed`;

  while (queue.length > 0) {
    const parent = queue.shift();
    const children = flow.children[parent] ?? [];

    for (const child of children) {
      if (visited.has(child)) continue;
      visited.add(child);

      const childTask = state.tasks[child];
      if (!childTask) continue; // defensive — flow/state shape mismatch

      // Diamond-convergence safety: only skip 'pending' tasks. Tasks already in
      // 'success' (a path through a non-failing parent succeeded), 'failed', or
      // 'skipped' are left alone — but we still walk through them so their
      // descendants get visited. EXCEPT: if we don't enqueue, an independent
      // descendant past a 'success' diamond node won't be re-skipped. That's
      // CORRECT — a 'success' node satisfies its children's depends_on, so
      // those children should NOT be skipped on this path. The BFS terminates
      // at 'success' / 'failed' / 'skipped' nodes by NOT enqueuing them.
      if (childTask.status !== 'pending') continue;

      // Mark skipped + persist reason. Note: state.js markSkipped sets ended_at
      // via Date.now() — that's fine; propagateSkip is called from runFlow where
      // wall-clock timestamps are already in use (see markFailed).
      markSkipped(state, child, reason);
      skippedCount++;

      onEvent({
        type: 'task',
        id: child,
        status: 'skipped',
        skip_reason: reason,
        seq: nextSeq(),
      });

      // Pitfall C2: decrement grandchildren's indegrees so independent branches
      // don't deadlock waiting on a dep that will never finish.
      for (const grandchild of flow.children[child] ?? []) {
        indegree[grandchild]--;
      }

      queue.push(child);
    }
  }

  return skippedCount;
}
```

**DO NOT** import from `./scheduler.js` — propagateSkip is a pure mutation function. The scheduler imports propagateSkip, not the other way around.

**DO NOT** call `atomicWriteJson` inside propagateSkip — the caller (scheduler.js, Plan 04) is the single writer that decides when to persist (typically once after the BFS completes, OR per-step if the caller prefers).

**DO NOT** spawn or do any I/O. propagateSkip is synchronous and pure (apart from mutating its arguments and emitting events via the callback).

**DO NOT** start the BFS from the failedId's *children* — start from failedId itself. The visited set guards against the BFS visiting failedId in a self-loop (which would be a cycle and would have failed validation); the iteration over `flow.children[failedId]` correctly enqueues the direct downstream tasks.

**DO NOT** treat 'running' as a terminal state inside the BFS — 'running' should not appear in propagateSkip's input because the function is called inside the failure branch where the failing task has already been markFailed'd, and other running tasks are independent. (If a 'running' descendant somehow appears, leaving it alone is correct — the scheduler will resolve it normally.)

```js
import { test } from 'node:test';
import assert from 'node:assert/strict';
import { propagateSkip } from '../src/skip.js';

// Helper: build a fake state map with all tasks pending.
function pendingState(taskIds) {
  const tasks = {};
  for (const id of taskIds) {
    tasks[id] = {
      status: 'pending', attempts: 0,
      started_at: null, ended_at: null, exit_code: null,
    };
  }
  return { flow_file: '/tmp/flow.json', started_at: 't0', ended_at: null, tasks };
}

// Helper: build flow.children map from edges array.
function flowOf(edges) {
  const children = {};
  const ids = new Set();
  for (const [parent, child] of edges) {
    ids.add(parent); ids.add(child);
    if (!children[parent]) children[parent] = [];
    children[parent].push(child);
  }
  // Initialize children arrays for leaf nodes too.
  for (const id of ids) if (!children[id]) children[id] = [];
  return { children };
}

// Helper: capture events fired during propagateSkip.
function captureEvents() {
  const events = [];
  let seqCounter = 0;
  return {
    events,
    onEvent: (e) => events.push(e),
    nextSeq: () => ++seqCounter,
  };
}

test('SCHED-06 S-1: linear A→B→C, A fails, propagateSkip from A skips B and C', () => {
  const flow = flowOf([['A', 'B'], ['B', 'C']]);
  const state = pendingState(['A', 'B', 'C']);
  // Indegrees as the scheduler would have them at the moment A fails:
  // A=0 (already running), B=1, C=1
  const indegree = { A: 0, B: 1, C: 1 };
  const cap = captureEvents();

  // Mark A failed manually (caller would do this via markFailed before propagateSkip).
  state.tasks.A.status = 'failed';

  const count = propagateSkip('A', flow, state, indegree, cap.onEvent, cap.nextSeq);

  assert.equal(count, 2);
  assert.equal(state.tasks.B.status, 'skipped');
  assert.equal(state.tasks.C.status, 'skipped');
  assert.equal(state.tasks.B.skip_reason, 'upstream A failed');
  assert.equal(state.tasks.C.skip_reason, 'upstream A failed');
  // Events: B and C should each have a skipped event with monotonic seq.
  assert.equal(cap.events.length, 2);
  assert.deepEqual(
    cap.events.map((e) => ({ id: e.id, status: e.status })),
    [{ id: 'B', status: 'skipped' }, { id: 'C', status: 'skipped' }],
  );
  assert.ok(cap.events[0].seq < cap.events[1].seq, 'seq must be monotonic');
});

test('SCHED-06 S-2: diamond A→B, A→C, B→D, C→D — A fails → B,C,D all skipped (D once, not twice)', () => {
  const flow = flowOf([['A', 'B'], ['A', 'C'], ['B', 'D'], ['C', 'D']]);
  const state = pendingState(['A', 'B', 'C', 'D']);
  const indegree = { A: 0, B: 1, C: 1, D: 2 };
  const cap = captureEvents();
  state.tasks.A.status = 'failed';

  const count = propagateSkip('A', flow, state, indegree, cap.onEvent, cap.nextSeq);

  assert.equal(count, 3, 'B, C, D each marked skipped exactly once');
  assert.equal(state.tasks.B.status, 'skipped');
  assert.equal(state.tasks.C.status, 'skipped');
  assert.equal(state.tasks.D.status, 'skipped');
  // Exactly 3 events (no double-event for D).
  const dEvents = cap.events.filter((e) => e.id === 'D');
  assert.equal(dEvents.length, 1, 'D emits exactly one skipped event');
});

test('SCHED-06: indegrees of skipped tasks\' children are decremented (C2 deadlock prevention)', () => {
  // Setup: A → B → D, X → D (independent root X). A fails → B should be skipped.
  // D depends on B and X. After B is skipped, D's indegree must drop by 1 so when X
  // succeeds, D can run.
  const flow = flowOf([['A', 'B'], ['B', 'D'], ['X', 'D']]);
  const state = pendingState(['A', 'B', 'D', 'X']);
  const indegree = { A: 0, B: 1, D: 2, X: 0 };
  const cap = captureEvents();
  state.tasks.A.status = 'failed';

  const count = propagateSkip('A', flow, state, indegree, cap.onEvent, cap.nextSeq);

  // B is skipped (depends on A); D's indegree drops because B was skipped.
  // D is also visited via B's children, but D depends on X too — and X is still
  // pending (not skipped, because it's not downstream of A). The BFS reaches D
  // from B and skips D (because D is pending and we don't know X will succeed).
  //
  // CORRECTNESS CALL: per architecture.md and pitfalls.md, the BFS-skip semantic
  // is "any task that has at least one upstream dependency in failed state gets
  // skipped." So D is correctly skipped here. The C2-deadlock-prevention indegree
  // decrement is for tasks deeper than D — e.g., if Y depends on D, Y's indegree
  // would be decremented when D is skipped.
  assert.equal(state.tasks.B.status, 'skipped');
  assert.equal(state.tasks.D.status, 'skipped');
  // X is independent — must not be touched.
  assert.equal(state.tasks.X.status, 'pending');
  // Indegrees: D was decremented when B was skipped (2 → 1).
  assert.equal(indegree.D, 1);
});

test('SCHED-06: diamond convergence node already in success → NOT re-marked (diamond-convergence safety)', () => {
  // A → B, A → C, B → D, C → D. D ran successfully (reached from a parallel path
  // before A's failure became terminal). A then fails terminally. propagateSkip
  // from A walks to B (skipped), then would visit D — but D is already success,
  // so propagateSkip leaves D alone.
  //
  // (NOTE: in real flow, D wouldn't run before A finishes because D depends on B
  // which depends on A. But propagateSkip's diamond-convergence safety should
  // still hold defensively.)
  const flow = flowOf([['A', 'B'], ['A', 'C'], ['B', 'D'], ['C', 'D']]);
  const state = pendingState(['A', 'B', 'C', 'D']);
  state.tasks.A.status = 'failed';
  state.tasks.D.status = 'success';
  state.tasks.D.exit_code = 0;
  const indegree = { A: 0, B: 1, C: 1, D: 0 };
  const cap = captureEvents();

  const count = propagateSkip('A', flow, state, indegree, cap.onEvent, cap.nextSeq);

  // B and C are pending → skipped (count = 2). D stays success.
  assert.equal(count, 2);
  assert.equal(state.tasks.B.status, 'skipped');
  assert.equal(state.tasks.C.status, 'skipped');
  assert.equal(state.tasks.D.status, 'success', 'success node MUST NOT be re-marked');
  // No skipped event emitted for D.
  assert.equal(cap.events.filter((e) => e.id === 'D').length, 0);
});

test('SCHED-06 S-3: independent branches not affected — A→B (A fails), C→D (independent) — D unaffected', () => {
  const flow = flowOf([['A', 'B'], ['C', 'D']]);
  const state = pendingState(['A', 'B', 'C', 'D']);
  state.tasks.A.status = 'failed';
  const indegree = { A: 0, B: 1, C: 0, D: 1 };
  const cap = captureEvents();

  const count = propagateSkip('A', flow, state, indegree, cap.onEvent, cap.nextSeq);

  assert.equal(count, 1, 'only B is skipped');
  assert.equal(state.tasks.B.status, 'skipped');
  // C and D must remain pending — they are not downstream of A.
  assert.equal(state.tasks.C.status, 'pending');
  assert.equal(state.tasks.D.status, 'pending');
  // Indegrees of independent branch unchanged.
  assert.equal(indegree.C, 0);
  assert.equal(indegree.D, 1);
});

test('SCHED-06: events emit with monotonic seq and skip_reason references the failed parent', () => {
  // 4-node chain. The seq numbers must strictly increase as we walk the BFS.
  const flow = flowOf([['A', 'B'], ['B', 'C'], ['C', 'D']]);
  const state = pendingState(['A', 'B', 'C', 'D']);
  state.tasks.A.status = 'failed';
  const indegree = { A: 0, B: 1, C: 1, D: 1 };
  const cap = captureEvents();

  propagateSkip('A', flow, state, indegree, cap.onEvent, cap.nextSeq);

  assert.equal(cap.events.length, 3);
  for (const e of cap.events) {
    assert.equal(e.skip_reason, 'upstream A failed');
    assert.equal(e.status, 'skipped');
    assert.equal(e.type, 'task');
  }
  // Strictly monotonic seq across all emitted events.
  for (let i = 1; i < cap.events.length; i++) {
    assert.ok(cap.events[i].seq > cap.events[i - 1].seq, 'seq must be strictly increasing');
  }
});
```
