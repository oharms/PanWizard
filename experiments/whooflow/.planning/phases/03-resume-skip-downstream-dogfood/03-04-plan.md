// mergeState) have their children's indegrees pre-decremented. This is the mechanism
// that makes the scheduler oblivious to fresh-vs-resume mode (STATE-05): the seed
// loop's only knowledge is the current status of each task in the supplied state.
// For a fresh run, every task is 'pending' and the pre-decrement branch never fires.
for (const id of flow.topoOrder) {

  if (status === 'success') {
    // Pre-decrement children's indegrees — the work is already done.
    for (const c of flow.children[id]) indegree[c]--;

    ready.push(id);

  // 'running' is impossible at this point — mergeState resets running to pending.

**Change 4: Add a resume-startup BFS pass for already-failed tasks.**

Right after the seed loop, BEFORE `pump()` is called, walk every task that arrived already in 'failed' status (from mergeState terminal-failed branch) and fire propagateSkip on it. This ensures downstream tasks of a previously-failed task are marked skipped before any new dispatch.

```js
// Resume-startup skip-downstream pass: any task already in 'failed' status on entry
// (from mergeState terminal-failed branch — a prior run's task that exhausted retries)
// must trigger skip-downstream BFS now, before pump dispatches anything new. SCHED-06
// "propagation only on terminal fail" still holds — terminal fail happened in the prior
// run, recorded in the state file, surfaced by mergeState.
for (const id of flow.topoOrder) {
  if (state.tasks[id].status === 'failed') {
    // Optionally emit a synthetic 'failed' event here so the JSON event stream shows it
    // — but the prior run already emitted that event. Don't double-emit. Just propagate.
    summary.failed++;  // count it in this run's summary so exit code reflects final state
    summary.skipped += propagateSkip(id, flow, state, indegree, onEvent, nextSeq);
  } else if (state.tasks[id].status === 'skipped') {
    // Pre-existing skipped tasks (from prior run) — count them in summary.
    // mergeState resets these to pending though, so this branch is currently dead;
    // safe to leave for forward compatibility.
    summary.skipped++;
  } else if (state.tasks[id].status === 'success') {
    // Already counted via the seed loop's pre-decrement; we don't bump summary.success
    // here because the prior run already accounted for it. The CURRENT run's summary
    // reflects "what changed in this run." Exit code mapping: summary.failed > 0 → 1,
    // else → 0. Already-failed tasks in summary.failed correctly produce exit 1 on resume.
  }
}

// Persist after the resume-startup BFS so the on-disk state reflects the post-skip view.
atomicWriteJson(stateFile, state);
```
