<task type="auto">
  <name>Task 1: Implement fence-aware heading extractor</name>
  <files>src/lib/extract.js</files>
  <action>
Create `src/lib/extract.js` exporting a single function `extract(markdown, maxDepth = 3)`.

Implementation:
1. Split markdown string by newlines
2. Maintain fence state: `inFence` boolean and `fenceLen` integer
3. For each line:
   - Check for fence opening/closing: match `^(`{3,})` pattern
   - If not in fence and match found: set `inFence = true`, store `fenceLen = match length`
   - If in fence and match found with length >= fenceLen: set `inFence = false`, reset fenceLen
   - If in fence: skip line entirely (continue)
   - If not in fence: check for heading match `^(#{2,})\s+(.+)$`
   - If heading matched and `level <= maxDepth`: push `{ level, text: stripFormatting(rawText.trim()) }`
4. Return array of `{ level, text }` objects

The `stripFormatting(text)` helper (not exported) must process in this order:
- Links: `[text](url)` -> `text` (regex: `/\[([^\]]+)\]\([^)]+\)/g`)
- Bold `**...**`: strip markers (regex: `/\*\*(.+?)\*\*/g`)
- Bold `__...__`: strip markers (regex: `/__(.+?)__/g`)
- Italic `*...*`: strip markers (regex: `/\*(.+?)\*/g`)
- Italic `_..._`: strip markers (regex: `/_(.+?)_/g`)
- Code backticks: strip markers (regex: `/`(.+?)`/g`)

IMPORTANT: The function must be pure — no `fs`, no `process`, no side effects. String in, array out.
  </action>
  <verify>
    node -e "import { extract } from './src/lib/extract.js'; const h = extract('## Hello\n### World\n## **Bold**'); console.log(JSON.stringify(h)); process.exit(h.length === 3 && h[2].text === 'Bold' ? 0 : 1)"
  </verify>
  <done>
    - `extract('## A\n### B')` returns `[{level:2, text:'A'}, {level:3, text:'B'}]`
    - Headings inside fences are excluded
    - 4-backtick fences only close with 4+ backticks
    - `## **Bold** text` extracts text `Bold text` (formatting stripped)
    - `extract(md, 2)` excludes `###` headings
    - Function is pure: no imports of `fs` or `process`
  </done>
</task>

<task type="auto">
  <name>Task 2: Implement GitHub-style slug generator</name>
  <files>src/lib/slugify.js</files>
  <action>
Create `src/lib/slugify.js` exporting a single factory function `createSlugger()`.

Implementation:
1. `createSlugger()` returns a closure `slugify(text)` that:
   - Maintains a `seen` Map in the closure scope (not module-level)
   - Converts text to lowercase
   - Replaces all characters that are NOT alphanumeric, space, or hyphen with empty string
   - Replaces spaces with hyphens
   - Collapses consecutive hyphens to single hyphen
   - Trims leading/trailing hyphens
   - Checks `seen` Map for the base slug:
     - If not seen: set `seen[base] = 1`, return `base`
     - If seen N times: set `seen[base] = N + 1`, return `base-N`
   - This matches GitHub behavior: first `## Foo` -> `foo`, second `## Foo` -> `foo-1`, third -> `foo-2`

IMPORTANT:
- Factory pattern (not module-level state) so each document gets its own dedup context
- The function is pure (given the same sequence of inputs, same outputs)
- No `fs`, no `process`, no side effects beyond closure state
  </action>
  <verify>
    node -e "import { createSlugger } from './src/lib/slugify.js'; const s = createSlugger(); const r = [s('Hello World'), s('Hello World'), s('Test!')]; console.log(JSON.stringify(r)); process.exit(r[0]==='hello-world' && r[1]==='hello-world-1' && r[2]==='test' ? 0 : 1)"
  </verify>
  <done>
    - `slugify('Hello World')` returns `'hello-world'`
    - `slugify('Test!')` returns `'test'` (special chars stripped)
    - Duplicate calls: first returns `foo`, second returns `foo-1`, third returns `foo-2`
    - Each `createSlugger()` call produces an independent instance
    - Function is pure: no imports of `fs` or `process`
  </done>
</task>
