<task type="auto">
  <name>Task 1: Implement TOC renderer and wire CLI</name>
  <files>src/lib/render.js, src/cli.js</files>
  <action>
**1. Create `src/lib/render.js`** exporting `render(headings, slugify)`:

- Takes array of `{ level, text }` objects and a `slugify` function
- Base level is 2 (i.e., `##` gets zero indent)
- For each heading: compute indent as `'  '.repeat(level - 2)`, format as `${indent}- [${text}](#${slugify(text)})`
- Join lines with `\n` and append trailing `\n`
- Return the full TOC string
- Pure function: no I/O, no side effects

**2. Rewrite `src/cli.js`** (replace existing stub) as the I/O shell:

The existing cli.js is a stub that only prints usage. Replace it entirely with:

- Shebang: `#!/usr/bin/env node`
- Imports: `extract` from `./lib/extract.js`, `createSlugger` from `./lib/slugify.js`, `render` from `./lib/render.js`
- Import `readFileSync` from `node:fs` and `resolve` from `node:path`
- Parse `process.argv.slice(2)`:
  - `--input <file>`: next arg is the file path
  - `--from-stdin`: flag to read from stdin
  - `--max-depth <N>`: next arg parsed as integer (default 3)
- Validation:
  - No arguments at all -> `process.stderr.write('Usage: whootoc --input <file> [--from-stdin] [--max-depth N]\n')`, `process.exit(1)`
  - Both `--input` and `--from-stdin` -> `process.stderr.write('Error: --input and --from-stdin are mutually exclusive\n')`, `process.exit(1)`
- If `--input`:
  - `readFileSync(resolve(filePath), 'utf8')` wrapped in try/catch
  - On ENOENT: `process.stderr.write('Error: file not found: ${filePath}\n')`, `process.exit(1)`
- If `--from-stdin`:
  - Collect all stdin chunks into a string (use `process.stdin` readable with `for await...of` or manual buffering)
  - Process after stdin closes
- Pipeline: `const headings = extract(content, maxDepth); const slugify = createSlugger(); const toc = render(headings, slugify); process.stdout.write(toc);`
- Exit 0 on success (implicit)

IMPORTANT: Use `process.stdout.write` for output (not console.log). Use `process.stderr.write` for errors. All lib imports, no business logic in cli.js itself.
  </action>
  <verify>
    echo -e "## Hello\n### World" | node src/cli.js --from-stdin
  </verify>
  <done>
    - `whootoc --input <file>` reads file and prints TOC to stdout

    - `--max-depth 2` excludes `###` headings
    - `--input` and `--from-stdin` together -> error on stderr, exit 1
    - Missing file -> error on stderr, exit 1
    - No args -> usage on stderr, exit 1

    - Output ends with trailing newline
  </done>
