# LOCKED DECISIONS (from 02-research.md — NON-NEGOTIABLE)

1. **Pipeline order:** `WHERE → group/finalize → AGGR-07 default sort on synth rows (only when no user ORDER BY) → project → user ORDER BY (if any) → slice`. NEVER project before group, NEVER slice before sort, NEVER user sort before project. (Research Pattern 5 + Anti-Patterns table.) **Important:** the AGGR-07 default sort runs on SYNTH ROWS (pre-projection) so it works regardless of whether the user projected the group columns. The user's ORDER BY runs on POST-PROJECTION rows (so SORT-03 alias support works).
2. **Routing rule:** Streaming path is taken iff `plan.groupBy === null && plan.sort === null && plan.implicitGroupBy === false`. Otherwise buffered path. The streaming path optionally honors `plan.limit` via `for await ... break` early termination (PAGE-03).
3. **AGGR-07 default emission order:** When `plan.groupBy !== null` AND `plan.sort === null` AND `plan.implicitGroupBy !== true`, the buffered path sorts SYNTH ROWS by group-key columns in lex order BEFORE projection. When `plan.sort !== null`, the user's ORDER BY wins (skip the AGGR-07 default sort; user sort runs post-projection).
4. **PAGE-03 early-termination contract:** the streaming path MUST `break` out of `for await` after `offset + count` rows have passed WHERE. Breaking the loop triggers `rl.return()` which closes the underlying file descriptor. The TEST gate uses an `onLineRead` callback to count JSON.parse invocations and asserts the counter does NOT exceed (offset + count + small_overhead) — proving the file was NOT fully read.
5. **Synth rows from finalizeBuckets:** flat keys for group columns (dotted-name as literal key) AND flat keys for aggregate output slots. The projection step over a synth row uses `opts.synthRow = true` to skip dot-walking.
6. **Implicit GROUP BY () routing:** `plan.implicitGroupBy === true` (with `plan.groupBy === []`) routes to buffered path. Single bucket; one output row. NO AGGR-07 default sort applied (a single row needs no sorting).
7. **Aggregate-projection output keys:** When the user writes `SUM(x) AS total`, the output key is `total` (the alias). When the user writes `COUNT(*)` with NO alias, the synthesized output key is `'COUNT(*)'` (literal string with parens). When the user writes `SUM(x)` with no alias, the synthesized key is `'SUM(x)'`. The accumulator's `aggregateDefs` map uses these output keys.
8. **No external imports.** All imports are `node:` builtins or local `./` modules.

```javascript
/**
 * Synthesize an output key name for a bare Aggregate (no alias).
 * Format: FUNC(*) for COUNT_STAR, FUNC(field.path) otherwise.
 *
 * @param {{ type: 'Aggregate', func: string, arg: any }} agg
 * @returns {string}
 */
export function aggregateOutputName(agg) {
  if (agg.arg === 'STAR') return `${agg.func}(*)`;
  // arg is a ColumnRef; use its name (the dotted form).
  return `${agg.func}(${agg.arg.name})`;
}

/**
 * @param {any[]} projectionList
 * @param {Record<string, any>} row
 * @param {{ synthRow?: boolean }} [opts]
 * @returns {Record<string, any>}
 */
export function project(projectionList, row, opts = {}) {
  const synth = opts.synthRow === true;

  // Wildcard short-circuit (Phase 1 behavior preserved)
  if (projectionList.length === 1 && projectionList[0].type === 'Wildcard') {
    return { ...row };
  }

  const out = {};
  for (const item of projectionList) {
    if (item.type === 'ColumnRef') {
      // Phase 1: dot-walk via resolvePath. Phase 2 synth-row path: literal flat-key read.
      out[item.name] = synth ? row[item.name] : resolvePath(row, item.path);
    } else if (item.type === 'Alias') {
      const inner = item.expr;
      if (inner.type === 'ColumnRef') {
        out[item.alias] = synth ? row[inner.name] : resolvePath(row, inner.path);
      } else if (inner.type === 'Aggregate') {
        // Synth row stores the aggregate output under the alias name as a flat key.
        out[item.alias] = row[item.alias];
      } else {
        throw new Error(`project: unsupported alias inner type ${inner.type}`);
      }
    } else if (item.type === 'Aggregate') {
      // Bare aggregate without alias — use synthesized output name.
      const outKey = aggregateOutputName(item);
      out[outKey] = row[outKey];
    } else {
      throw new Error(`project: unsupported projection item type ${item.type}`);
    }
  }
  return out;
}
```

The new `aggregateOutputName` export is needed by the evaluator to build the `aggregateDefs` map keys consistently with what `project()` later reads. Both code paths use the same name — that's the contract.

This change DOES preserve Phase 1 behavior for all existing tests because:
- Phase 1 callers don't pass `opts.synthRow` → defaults to false → existing dot-walk via resolvePath continues for ColumnRef and Alias-of-ColumnRef.
- Phase 1 never had Aggregate or Alias-of-Aggregate projection items (analyzer rejected them). Phase 2's parser produces them, and Phase 2 code paths handle them via the new branches.

//          regardless of whether the user projected them.)
//       4. Project synthRows → projected rows.
//       5. User ORDER BY: IF plan.sort !== null
//          THEN projected.sort(buildComparator(plan.sort)).
//          (Runs AFTER projection because SORT-03 requires alias support.)
//       6. Slice: IF plan.limit !== null THEN projected.slice(offset, offset+count).
//       7. Yield each.
//
// Pipeline order is LOCKED — see research.md Pattern 5 + Plan 04 LOCKED DECISION 1.
//
// Malformed JSONL: emit a single stderr warning, skip the row, continue.
// Blank lines: silently skipped.
//
// opts.onLineRead?: (lineNo: number) => void — fired AFTER successful JSON.parse
//   (test hook for PAGE-03 early-termination verification).
// =============================================================================

import { createReadStream } from 'node:fs';
import { createInterface } from 'node:readline';
import { evalWhere } from './where.js';
import { project, aggregateOutputName } from './project.js';
import { accumulateGroups, finalizeBuckets } from './accumulator.js';
import { buildComparator } from './sort.js';

/**
 * @param {object} plan — ExecutionPlan from analyze()
 * @param {string} filePath
 * @param {{ onLineRead?: (lineNo: number) => void }} [opts]
 */
export async function* execute(plan, filePath, opts = {}) {
  const isBuffered =
    plan.groupBy !== null || plan.sort !== null || plan.implicitGroupBy === true;

  if (!isBuffered) {
    yield* runStreaming(plan, filePath, opts);
    return;
  }

  // BUFFERED PATH — pipeline order is LOCKED.

  // Step 1: Collect WHERE-passing rows.
  const rows = [];
  for await (const row of streamRows(filePath, plan, opts)) {
    rows.push(row);
  }

  const aggregateDefs = collectAggregateDefs(plan.projection);
  const isGrouped = plan.groupBy !== null || plan.implicitGroupBy === true;
  const groupCols = plan.groupBy || [];

  // Step 2: Group + finalize → synth rows. (Or pass through raw rows for non-grouped buffered.)
  let synthRows;
  let synthRowFlag;
  if (isGrouped) {
    const buckets = accumulateGroups(rows, groupCols, aggregateDefs);
    synthRows = finalizeBuckets(buckets, groupCols);
    synthRowFlag = true;
  } else {
    synthRows = rows;
    synthRowFlag = false;
  }

  // Step 3: AGGR-07 default sort on SYNTH ROWS — runs BEFORE projection.
  // Skipped when implicit GROUP BY () (single bucket, nothing to sort) or
  // when the user supplied their own ORDER BY (handled at step 5).
  if (
    isGrouped &&
    plan.implicitGroupBy !== true &&
    plan.sort === null &&
    groupCols.length > 0
  ) {
    synthRows.sort(buildGroupKeyComparator(groupCols));
  }

  // Step 4: Project synth rows.
  let projected = synthRows.map((r) => project(plan.projection, r, { synthRow: synthRowFlag }));

  // Step 5: User ORDER BY (post-projection — required for SORT-03 alias support).
  if (plan.sort !== null) {
    projected.sort(buildComparator(plan.sort));
  }

  // Step 6: Slice (LIMIT/OFFSET).
  if (plan.limit !== null) {
    const off = plan.limit.offset;
    const cnt = plan.limit.count;
    projected = projected.slice(off, off + cnt);
  }

  // Step 7: Yield.
  for (const r of projected) yield r;
}

// ---------------------------------------------------------------------------
// STREAMING PATH (Phase 1 behavior + PAGE-01/02/03 early termination)
// ---------------------------------------------------------------------------

async function* runStreaming(plan, filePath, opts) {
  const limit = plan.limit;
  const count = limit ? limit.count : Infinity;
  const offset = limit ? limit.offset : 0;

  let passed = 0;
  let emitted = 0;

  for await (const row of streamRows(filePath, plan, opts)) {
    passed++;
    if (passed <= offset) continue;
    yield project(plan.projection, row);
    emitted++;
    if (emitted >= count) break; // PAGE-03 early termination — closes readline
  }
}

// ---------------------------------------------------------------------------
// Shared row producer — JSONL parse + WHERE filter
// ---------------------------------------------------------------------------

async function* streamRows(filePath, plan, opts) {
  const rl = createInterface({
    input: createReadStream(filePath, { encoding: 'utf8' }),
    crlfDelay: Infinity,
  });
  let lineNo = 0;
  for await (const rawLine of rl) {
    lineNo++;
    const line = rawLine.trim();
    if (line === '') continue;
    let row;
    try {
      row = JSON.parse(line);
    } catch (err) {
      process.stderr.write(
        `whoodb: skipping malformed JSON line ${lineNo}: ${err.message}\n`,
      );
      continue;
    }
    if (opts && typeof opts.onLineRead === 'function') opts.onLineRead(lineNo);
    if (!evalWhere(plan.filter, row)) continue;
    yield row;
  }
}

// Helpers

/**
 * Build the aggregateDefs map from a projection: { [outputKey]: AggregateNode }.
 * outputKey is the alias if Alias-wrapped, else the synthesized aggregate name.
 */
function collectAggregateDefs(projection) {
  const defs = {};
  for (const item of projection) {
    if (item.type === 'Aggregate') {
      defs[aggregateOutputName(item)] = item;
    } else if (item.type === 'Alias' && item.expr.type === 'Aggregate') {
      defs[item.alias] = item.expr;
    }
  }
  return defs;
}

/**
 * AGGR-07: build a comparator that sorts SYNTH ROWS by group-key column order
 * lexicographically. groupBy is the ColumnRef[] from the plan; for each row,
 * read row[col.name] LITERALLY (synth rows store dotted-name as flat key).
 *
 * NOTE: this comparator runs on synth rows BEFORE projection. Synth rows
 * ALWAYS carry the group columns (because finalizeBuckets writes them under
 * their dotted names as flat keys), so this comparator works regardless of
 * whether the user projected those columns into the SELECT list.
 */
function buildGroupKeyComparator(groupBy) {
  return (rowA, rowB) => {
    for (const col of groupBy) {
      const a = rowA[col.name];
      const b = rowB[col.name];
      // Use lex string compare (deterministic across types).
      const sa = a === null || a === undefined ? '' : String(a);
      const sb = b === null || b === undefined ? '' : String(b);
      if (sa < sb) return -1;
      if (sa > sb) return 1;
    }
    return 0;
  };
}

// Phase 2 helper: install a stderr capture, return { text getter, restore }.
// Use this for ASYNC tests where the captured stderr must persist through awaits.
function captureStderr() {
  const original = process.stderr.write.bind(process.stderr);
  const captured = [];
  process.stderr.write = (chunk) => {
    captured.push(typeof chunk === 'string' ? chunk : chunk.toString('utf8'));
    return true;
  };
  return {
    get text() { return captured.join(''); },
    restore() { process.stderr.write = original; },
  };
}
