## Executive Summary

This experiment is the v3.7.0 self-improvement loop's first non-trivial end-to-end run, and it overwhelmingly validates the hypothesis: building a real tool with a non-optional dogfood gate produces orders of magnitude more learning than a synthetic toy. Of the 12 trace events, 6 directly raise promote-worthy patterns (P-201..P-205 universal, P-301 internal), 4 confirm prior patterns (P-001, P-002, P-101, P-102), and 1 explicitly generalizes a prior pattern (P-001 timestamp-shape → P-204 violation-shape). The single critical event — 252 errors when the parser hit PAN's real files despite passing all 8 unit tests — is the highest-value finding of the run and crystallizes a universal rule: **synthetic fixtures lie; real-world corpus fixtures don't.**

## Findings

### F1 — P-201 — Test parsers against real-world fixtures from day one

- **Evidence:** `trace.jsonl` 11:50:00Z (surprise, critical): "Dogfood gate produced 252 errors against PAN's commands/pan/. Cause: parser supports flow-style lists but rejects block-style lists." 39/39 unit tests passed *before* dogfood ran — they gave a false green light.
- **Rule:** When building any parser, validator, or schema-checker for a structured format, include at least one fixture sampled directly from the real-world corpus before declaring v1 done. The dogfood gate runs *before* asserting "passed", not after.
- **Scope:** universal · **Topic:** test-strategy · **Priority:** must

### F2 — P-202 — Schemas should be inferred from real files, not authored from imagination

- **Evidence:** `trace.jsonl` 11:52:00Z (gap, major): name pattern rejected 44/52 PAN files. 11:53:00Z (gap, minor): missing `argument-hint` field caused 30+ unknown-field warnings.
- **Rule:** When authoring a schema for an existing corpus, run a sampling pass first. Manual schemas guarantee a mismatch with reality.
- **Scope:** universal · **Topic:** schema-design · **Priority:** should

### F3 — P-203 — Document explicit Out-of-Scope cuts in DESIGN_SPEC

- **Evidence:** `trace.jsonl` 11:32:00Z + 11:51:00Z: block-list scope cut documented up front; dogfood reversal then read as a spec correction, not a bug fix.
- **Rule:** Feature specs and DESIGN_SPECs should include an explicit "Out of Scope" table. When dogfood reveals a cut was wrong, the deviation is self-documenting.
- **Scope:** universal · **Topic:** design-process · **Priority:** must

### F4 — P-204 — Test the violation/error CONTRACT, not message prose

- **Evidence:** `trace.jsonl` 11:40:00Z (decision, references P-001): "Tests assert violation SHAPE, not exact string messages."
- **Rule:** When testing structured error/violation output, assert on stable contract fields (`code`, `field`, `severity`) — never on `message` text. Generalizes P-001 (timestamp shape).
- **Scope:** universal · **Topic:** test-patterns (append to existing) · **Priority:** must

### F5 — P-205 — globToRegex root-level convention

- **Evidence:** `trace.jsonl` 11:45:00Z (error, major): "globToRegex bug: '**/foo.md' regex required at least one '/' in the path." Fix: emit `(?:.*/)?` so slash is optional.
- **Rule:** Translate `**/X` to allow zero-segment match (root-level X), per gitignore/minimatch convention.
- **Scope:** universal · **Topic:** glob-semantics · **Priority:** should

### F6 — P-301 — PAN's commands/pan/*.md has 9 real frontmatter consistency bugs

- **Evidence:** Final dogfood: 9 errors + 4 warnings across 52 files in 51ms. Specifics: optimize.md missing frontmatter; patches.md missing `name` and has `description` as array; phase-tests.md uses multi-line block-scalar; todo-add.md and todo-check.md have `description` as array.
